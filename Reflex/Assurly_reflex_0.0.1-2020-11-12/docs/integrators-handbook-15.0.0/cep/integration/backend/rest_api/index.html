CEP REST API

The CEP services provide the following REST endpoints:

  • CEP Core

The CEP Core (cep-core) is the main back end service. It is used by the integrating system to manage the lifecycle of insurance applications/assessments and by the CEP Questionnaire UI component (reflex-cep-questionnaire). It can optionally be used by to deliver documents based on a previously created assessment (e.g. PDF documents).

The description of the returned values from each endpoint can be found here: Media-Types

Common error responses codes

All error responses created by the cep have the Content-Type application/json; charset=utf-8. Depending on the configured system verbosity they contain an ErrorModel that provides further information about the error.

Code Name Description
400 Bad Request The request was invalid. The error model response contains further information.
401 Unauthorized The request to the specific resource requires an authorization but the request does not contain valid authorization information.
403 Forbidden The request to the specific resource is due to authentication restriction not permitted.
404 Not Found Specified resource not found, URL invalid or does not exist.
500 Internal Server Error Request is not processed due to internal server error. Please check the log files of the accessed application server.
502 Bad Gateway An downstream system replied with an unexpected error or is unavailable. Please refer to the upstream section of the error model response or check the log files of the accessed application server.
