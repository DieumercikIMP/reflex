Pagination

Pagination is splitting the application form in smaller parts so that each part can be displayed as a single page.

Special KB design

To enable pagination, you have to specify top-level question groups in the knowledge base. Please refer to chapter Questionnaire Pagination of the ReFlex RKE Handbook for details on how to prepare the knowledge base for pagination.

Questionnaire configuration

After the KB is prepared the questionnaire front end component must be configured to use pagination. The configuration is a variable in the parent scope of the questionnaire because it may theoretically change during runtime. Possible values are 'Questionnaire' (default) or 'QuestionGroup'. For a fixed value, add the pagination-mode and the current-page-id attribute to your assessment.html:

<cep-questionnaire
	application="application"
	assessment-id="assessmentId"
	assessment-role="Applicant"
	pagination-mode="'QuestionGroup'"
	current-page-id="currentPageId">
</cep-questionnaire>

You will find more details on this in section Questionnaire Composition #Pagination

Pagination bars

When you start the application now, you will see the first page of the questionnaire but no possibility to continue to the next page. You have to add one or more pagination directives to your assessment.html file. A simple pager with only “previous” and “next” buttons is included like in the following example:

<cep-questionnaire ... pagination-mode="'QuestionGroup'" current-page-id="currentPageId">
	<after>
		<cep-pagination>
			<cep-prev-next-pagination />
		</cep-pagination>
	</after>
</cep-questionnaire>

You will find more details and pagination control alternatives in section Questionnaire Composition #Pagination

Translation and configuration

In your configuration, please add the pagination object:

pagination: {
	freePagination: false,
	ignoreValidityOnBackwardsPagination: true
},

You will find more information on this in section Questionnaire Configuration #Pagination.

The pagination bars use translations from components.pagination.

pagination: {
	errors: {...},
	hints: {...},
	texts: {
		previous: 'previous',
		next: 'next'
	},
	pageTitle: {
		...
	}
},

Please refer to section Translation for details. The key pageTitle is used for translating the breadcrumb pagination bar. Refer to Questionnaire Composition #Pagination for details.

Customizing the submit button — optional

The cep-submit-button directive comes with built-in support for hiding and disabling. This can be used for optimal user experience during pagination.

To hide the button as long as the user did not see all pages and to disable the submit button as long as the current page is invalid, add this to your assessment.html:

<cep-questionnaire ...>
	<after>
		<cep-pagination>...</cep-pagination>

		<cep-submit-button
			submit-function="$parent.submitIfValid"
			should-be-disabled="$parent.questionForm.$invalid"
			should-be-hidden="userHasNotSeenAllPages">
		</cep-submit-button>

	</after>
</cep-questionnaire>

You will find more details on this in section Questionnaire Composition #Pagination
