CEP front end configuration

The CEP front end is configured by constants for the different AngularJS modules provided by the CEP:

angular.module('reflexCepAngularApp')
       .constant('reflexCepAngularAppConfig', {
    ...
       });
    ...
angular.module('reflexCepQuestionnaire')
       .constant('questionnaireConfig', {
    ...
        });
    ...

These module configurations must be declared and included after the questionnaire and its dependencies in the index.html.

See the following section for a detailed description of the supported configuration parameters.

Configuring the cep-questionnaire (and cep-angular-app) component

(via the “questionnaireConfig” JSON Structure)

The questionnaireConfig constant configures certain behaviours of the CEP questionnaire and defines URL paths to the CEP back end.

Example:

  angular.module('reflexCepAngularApp').constant('reflexCepAngularAppConfig', {
			assessmentViewPath: 'views/assessment.html', // file path to the assessment view
			assessmentViewUrlPath: '/assessment'         // route to the assessment view
	});

  angular.module('reflexCepQuestionnaire').constant('questionnaireConfig', {
		defaultTranslation: 'en',
		autoSubmit: 'reflexive',
		autoSaveEnabled: true,
		autoSaveInterval: 60000,
		backendTimeoutWarning: 5000,
		backendTimeout: 20000,
		minLettersForSearch: 2,
		showCheckboxKey: false,
		showParentValidations: true,

		api: {
			urlPrefix: '/cep-core',
			assessmentService: {
				assessmentPath: '/assessment',
				search: '/search'
			},
			documentServicePath: '/documents'
		},

		questionAnnotation: {
			required: false,
			optional: 'denote'
		},

		pagination: {
			freePagination: false,
			ignoreValidityOnBackwardsPagination: true,
			dismissAlertDelay: 0
		},

		blockingQuestionnaire: {
			backdrop: 'fullpage',
			spinner: true
		},

		assessmentFactorGroups: {
			collapsiblesForSearchChildren: true
		}

	})
Key Value Type Meaning Example Mandatory
assessmentViewPath String File location of the HTML template containing the reflex-cep-questionnaire directive that will be rendered, when the assessment route is called (see below). Because it is only used by the reflexCepAngularApp module, it is now separate from the questionnaire config. views/assessment.html yes
assessmentViewUrlPath String The URL path parameter for the assessment route as shown in the browser’s address field, i.e. the location of the questionnaire directive. Enables the integrator to choose another name, e.g. in another language. /assessment yes
defaultTranslation String Specifies the default language code matching the JavaScript key of your translation. See Translation for details. en yes
autoSubmit String Specifies when a server round-trip should happen if an answer is given. Must be either reflexive to trigger a roundtrip only on reflexive answers or always to trigger a roundtrip every time an answer is given. reflexive yes
autoSaveEnabled Boolean Enables the auto save feature of CEP. true no (defaults to false)
autoSaveInterval Number Time interval in milliseconds for the auto save to happen. 60000 if autoSaveEnabled = true
backendTimeoutWarning Number A back end request-timeout in milliseconds that causes a warning that a request is unnaturally slow. 5000 yes
backendTimeout Number A back end request-timeout in milliseconds that causes a request to be aborted. 20000 yes
minLettersForSearch Number Number of characters to type before a search request is triggered against the CEP search API. 2 no (defaults to 1)
showCheckboxKey Boolean Whether to show the checkbox and radio button index for fast keyboard use. false no (defaults to false)
showParentValidations Boolean Whether to show the additional validation messages of struct or list parent questions (e.g. “This struct has errors.”) false no (defaults to false)
api Object Defines the paths to the public CEP API used by the questionnaire. See section ‘api’ for more information. See example given above. yes
questionAnnotation Object Sets required and optional annotations or legends. See section ‘questionAnnotation’ for more information. See example given above and below. no (shows no annotations at all when missing)
pagination Object Allows or disallows certain types of pagination. Pagination means the partial rendering of question groups to divide large application forms into smaller chunks. See section ‘pagination’ for more information. See example given above. no
blockingQuestionnaire Object Configures or disables the loading spinner. See section ‘blockingQuestionnaire’ for more information. See example given above. no
assessmentFactorGroups Object Enables or disables collapsible assessment factor groups for children of multi-term searches. See section ‘assessmentFactorGroups’ for more information. See example given above. no
initialAnswerByProductChangeMarked Boolean Enables or disables setting input-label css class (initialAnswerByProductChangeMarked) if answers taken over by product change. Fallback value is ‘false’. initialAnswerByProductChangeMarked: true no

(Note on date formats)

API

The API defines paths to the CEP public API used by the questionnaire.

Key Value Type Meaning Example Mandatory
urlPrefix String The base URL/location for all CEP REST-APIs. It will be used as a prefix for all service paths, if set.
For example, if set to cep/api/ the path to the assessment service would be cep/api/{assessmentService}. 		cep/api/ yes
assessmentService Object Object containing the base path to the Assessment Endpoint. see assessmentService yes
documentServicePath String Base path to the Document Endpoint. /documents if the Document Endpoint is used (e.g. for PDF generation).

assessmentService

Key Value Type Meaning Example Mandatory
assessmentPath String Path to the Assessment Endpoint /assessment yes
search String Path to the Assessment-factor search endpoint /search yes

questionAnnotation

The questionAnnotation object contains settings for annotation of required and/or optional questions.

The object has two keys: required and optional. Each one accepts:

  • 'denote': (Recommended) Question text is followed by a dash and the translation for components.questionAnnotation.optional or components.questionAnnotation.required.
  • 'legend': Question is marked with the annotation marker from the translation (‘*’ by default) and the annotation appears in the legend below the questionnaire. The legend is an optional directive. More information can be found in the front end integration documentation.
  • false: Questions have no marker at all.

Pagination

Whether pagination is active or not has to be configured when integrating the reflex-cep-questionnaire directive. Once active, the following optional parameters are evaluated.

Mode Fallback Value Description
freePagination false If set to true, navigation to any page of the questionnaire is permitted at any time, even if there are validation errors on the current page. In this case, the ignoreValidityOnBackwardsPagination parameter is ignored.
ignoreValidityOnBackwardsPagination false If set to true, navigation to any page prior to the current page is permitted at any time, even if there are validation errors on the current page. However, forward navigation is only possible if there are no validation errors on the current page.
dismissAlertDelay 0 The delay in milliseconds after which the pagination blocked notification alert (the <cep-blocked-pagination-alert /> component) should disappear. The value 0 means that the user has to dismiss the field manually.

Please note: Pagination is based on top level question groups as configured in the application form via form groups / sections in the knowledge base. CEP does not support pagination with other elements than top level question groups.

Blocking the questionnaire (loading spinner)

The built-in loading spinner and/or the backdrop can be removed completely. The predefined styling can be customized or disabled, too. (See: customization/theming for details).

If there is no configuration, neither spinner nor backdrop will be shown.

Mode Fallback Value Value Type Description
backdrop false 'fullpage', 'questionnaire' or false The backdrop is a semi-transparent glass pane which visually blocks the questionnaire during round trips. The setting 'questionnaire' lets the backdrop cover only the questionnaire DOM element. 'fullpage' covers the complete document, false shows no backdrop at all.
spinner false boolean Whether to show a (vertically fixed, rotating) loading spinner or not.

Collapsible Assessment Factor Groups

If there is no configuration, all assessment factor groups will be rendered as static panels.

Mode Fallback Value Value Type Description
collapsiblesForSearchChildren false boolean If true, all assessment factor groups that are direct children of a multi-term search are displayed as collapsible panels.

Date formats

Since the date format depends on the current language, they are translation keys instead of configuration keys although they are never visible in clear text to the user. Please have a look at customization/translation/date formats

Additionally, certain definitions in the underlying ReFlex Knowledge Base can change / influence the visualization of the application form / questionnaire in the CEP questionnaire component. Please refer to chapter Designing the Questionnaire of the ReFlex RKE Handbook for more details on how to prepare the knowledge base with regards to the CEP Questionnaire.
