Theming the questionnaire

Basics

The CEP Questionnaire is Bootstrap 3 conform. This means you only have to include the reflex-cep-questionnaire Yarn component and Bootstrap 3 to get a working and good-looking questionnaire. There are two possibilities to use the questionnaire styles: compiling the SASS (which makes the styles configurable) or using the pre-compiled CSS (which makes it super-easy to integrate). If you compile the CEP-SASS, you may want to overwrite SASS variables to adjust the questionnaire’s style.

However, if you would like to adapt the Questionnaire to your look and feel, follow this guide.

Including the styles

You have to include the questionnaire styles. - Even though it mainly uses Bootstrap 3 standard components, there are some components that need special styling (like the date-picker, the search component or structures).

Compile and @import SASS

The uncompiled SASS is located under src/scss/build/cepQuestionnare.scss. To compile it, you need to specify where the yarn-dependencies are located. With the Ruby SASS shell toolkit, you may use the -I option.

cep-questionnaire$ sass -I '../../node_modules/' src/scss/build/cepQuestionnaire.scss dist/css/compiled.css

The grunt-compass task uses the option: importPath: 'app/bower_components/'.

For import, a minimal custom-SASS file would look like this:

@import 'bootstrap-sass-official/assets/stylesheets/bootstrap'; //has to be before the questionnaire
@import 'reflex-cep-questionnaire/src/scss/build/cepQuestionnaire';

Variable overwrites would be located before those two lines, since all variable declarations use the !default-flag. Selector overwrites have to be located after those two lines.

Examples

The following examples need to be compiled with SASS as described above.

Overwrite all variables (Bootstrap theme)

To get a different look and feel, you could import a complete Bootstrap theme (which is a set of variables), e.g. from https://bootswatch.com/ .

@import 'themes/simplex' //has to be first
@import 'bootstrap-sass-official/assets/stylesheets/bootstrap'; //has to be before the questionnaire
@import 'reflex-cep-questionnaire/src/scss/build/cepQuestionnaire';

Overwriting some variables

All those example snippets belong at the very beginning of your myPortal.scss-file.

Fonts

@import url("https://fonts.googleapis.com/css?family=Open+Sans:400,300,700|Domine:400,700");
$font-family-sans-serif:  "Open Sans", Helvetica, Arial, sans-serif;
$font-family-serif:       "Domine", Georgia, "Times New Roman", Times, serif;
$font-family-monospace:   Menlo, Monaco, Consolas, "Courier New", monospace;
$font-family-base:        $font-family-sans-serif;
$headings-font-family:    $font-family-serif;

Colors

$gray-base:             #000;
$gray-darker:           #222;
$gray-dark:             #444;
$gray:                  #777;
$gray-light:            #808080;
$gray-lighter:          #ddd;
$brand-primary:         #D9230F;
$brand-success:         #469408;
$brand-info:            #029ACF;
$brand-warning:         #9B479F;
$brand-danger:          #D9831F;

Border Radius

$border-radius-base:        4px;
$border-radius-large:       6px;
$border-radius-small:       3px;

Input size

$input-height-base: 20px;

Customizing the backdrop and loading spinner

The spinner is customizable with the following variables (the values show the fallback value):

//the color of the backdrop, should have not more that 0.8 opacity
$questionnaire-backdrop-color: rgba(255, 255, 255, 0.8);
$questionnaire-spinner-color: $brand-primary;
$questionnaire-spinner-color-alt: $gray-lighter;

//the time for one rotation
$questionnaire-spinner-speed: 1s;
//the border width is calculated automatically (1/8 of the spinner size)
$questionnaire-spinner-size: 75px;
$questionnaire-spinner-z-index: 100;

To completely turn of the spinner and backdrop styling, set the following flags to true:

$questionnaire-suppress-spinner-styling: true;
$questionnaire-suppress-backdrop-styling: true;

To use the HRE-Bootstrap theme spinner, also add this SASS-snippet to your sources:

.loader {
    @extend .spinner-backdrop;
    .spinner {
        @extend .spinner-lg;
    }
}

Use pre-compiled CSS

The pre-compiled CSS is located under dist/css/standalone.css and is linked in the package.json file. So if you have a task that automagically injects the yarn components (like grunt-bower-install) then there is nothing more to do. If you include the sources by hand, you have to link to the standalone.css (and Bootstrap):

<link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.6/css/bootstrap.min.css">
<link rel="stylesheet" href="bower_components/reflex-cep-questionnaire/dist/standalone.css" />

Please keep in mind that you cannot configure the SASS variables with this approach. Any customization has to be done by overwriting CSS-selectors.

Applying the HRE Bootstrap Theme

This can only be done with SASS. The HRE Bootstrap theme is a common shared component which includes Bootstrap 3 itself. Therefore the only thing you have to change in your myPortal.scss main SASS file, is the Bootstrap include:

@import "hre-bootstrap-theme/src/scss/hre-bootstrap-theme"; //instead of official Bootstrap
@import 'reflex-cep-questionnaire/src/scss/build/cepQuestionnaire';

That’s all. Bootstrap and the questionnaire themes are now available.

Customizing main styling properties

Some inputs are considered “small inputs” (text, numbers, dates, …). Those have a maximum width which can be overridden.

//small inputs (eg. text, number, etc)
$short-input-width: 320px !default;

The position of validation of those short inputs can be configured with the $validation-position variable.

//position of default question validations,
//- does not apply to lists and structs or special widgets
//- does not apply for screens < screen-md
//- possible values: "below", "above" or "aside"
$validation-position: "below" !default;

The space between questions is now globally set via the following variable. It depends on the Bootstrap standard padding by default:

$space-between-questions: $padding-large-vertical * 2 !default;

Using AngularJS in CSP (Content Security Policy) mode

When using the ngCsp directive that suppresses inline Javascript and styles, the bower_components/angular-bootstrap/ui-bootstrap-csp.css and bower_components/angular/angular-csp.css files have to be included to make the questionnaire work properly.

Right-to-left layout support

The questionnaire supports RTL-layouts for languages that are read from right to left. To activate RTL-support, the translation table for a language must override the top-level dir key in its translation table with rtl:

angular
	.module('questionnaireTranslationAr', ['reflexCepTranslate'])
	.config(function(cepTranslateProvider) {
		cepTranslateProvider.registerLanguage('ar', translationTable);
	});

	var translationTable = {
		dir: 'rtl',
		...
	}

Animations

Since the questionnaire is reflexive, there might come new questions at any time. It is a good advice to add some sort of styling or even animations to those new questions because they might be before the currently active question.

New question-container that are caused by a reflexive question have the class new. Animating them with SCSS is easy:

.question-form.question-container.new {
    animation-name: fadeInFromRight;
    animation-duration: 0.5s;
    animation-iteration-count: 1;
    animation-timing-function: ease-out;
    animation-delay: 0;

    @keyframes fadeInFromRight {
        from {
            opacity: 0;
            transform: translate3d(100%, 0, 0);
        }

        to {
            opacity: 1;
            transform: none;
        }
    }
}

It is also possible to add more animations using the Angular Module ngAnimate.
