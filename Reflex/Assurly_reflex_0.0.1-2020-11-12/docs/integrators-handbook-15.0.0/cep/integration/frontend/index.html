CEP front end integration

The CEP Front End supports three integration scenarios:

  1. CEP UI as a standalone web application
  2. CEP UI embedded into an existing AngularJS Application
  3. CEP UI embedded into a Web Application without AngularJS

The following sub-chapters will describe a detailed integration guide for each scenario. Please note: these chapters will refer to the “questionnaire”, which is, in fact, the CEP front end/UI rendered in the browser, therefore these terms can be treated as synonyms.

CEP Questionnaire

The CEP front end will be delivered as a yarn component called reflex-cep-questionnaire (See yarn documentation). It has the following structure:

  • dist/ contains a compiled version of the CEP front end
  • src/ contains a standalone.css, a minimal stylesheet for the questionnaire as well as SCSS files that can be used to create a different styling

The root of this release artefact contains the following files:

  • package.json the description of the component and its functional dependencies
  • README.md the components general description and an installation guide

Configuration

For the available configuration parameters, please refer to the Configuration chapter.

Routing

The same routing and navigation mechanism applies, regardless of the used questionnaire integration scenario. The questionnaire has a configurable mandatory hash route for internal purposes which by default is /assessment. This route must not be used for routes in the surrounding portal, because this page is used for rendering the questions.

Start the questionnaire

After an assessment has been initialized by the integration service, the portal application should redirect the user to the directUrl.
