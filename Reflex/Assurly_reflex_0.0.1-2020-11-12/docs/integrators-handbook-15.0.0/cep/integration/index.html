Overview

Title: ReFlex CEP Integration Developer’s Guide
Classification: Technical document
Target group: Development team commissioned with the integration of ReFlex into the insurer’s software system(s)

Introduction

ReFlex is Hannover Re’s solution for supporting the automatic and manual underwriting process at the primary insurer. ReFlex CEP consists of several technical components which can be used to enhance the insurance sales / application process. The main ReFlex CEP component is the Questionnaire, which enables the user (e.g. end user, call centre agent, bank teller, etc.) to fill in the insurance application or parts of it (e.g. only the health declaration) and get an immediate risk assessment and business decision. ReFlex CEP also offers additional components supporting the insurance sales process around the questionnaire and automated decision process. Most of the ReFlex CEP components will be integrated on the UI level, i.e. they already provide a white-label user interface which can be customized to match the primary insurer’s corporate design (theming). For an integration of the automatic risk assessment feature using only back end services, please refer to the documentation of the ReFlex Middleware component RAS/REST (RESTful API).

The following diagram illustrates a common scenario where ReFlex components have been integrated in the insurer’s portal application.

  • The insurer’s portal application initializes a new assessment using the integration endpoint of the ReFlex CEP Core service via its REST API.
  • The integration endpoint will return a URL, which can be used to open the ReFlex CEP Questionnaire in the browser.
  • When handing back the control to the Insurer’s portal application, the result of the automatic risk assessment is provided and can be used to continue in the insurance application process.

About this document

This document contains all the necessary information for the integrator planing to integrate ReFlex CEP components into an existing software system. For example, the ReFlex CEP Questionnaire component may be integrated into a client portal to be able to provide a decision for an insurance application directly at the point-of-sale. In another example, the ReFlex CEP Questionnaire may be integrated into an underwriting system to support manual underwriting or adjust claims. Other useful resources:

  • ReFlex CEP uses the ReFlex RAS REST API for automatic risk assessment.
  • ReFlex CEP uses the ReFlex DCS REST API for template-based document generation (e.g. PDF)
