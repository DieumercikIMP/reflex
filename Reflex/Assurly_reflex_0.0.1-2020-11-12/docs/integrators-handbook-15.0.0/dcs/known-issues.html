Known issues

Currently issues regarding the following topics are known.

No fonts found error

This error occurs in situations where a template uses a font that is not installed on the system or embedded in the template. For example this error occurs when the DCS is installed in an environment comparable to the following

  • A minimal operating system is used without additional fonts installed (e.g. an Ubuntu docker image),
  • OpenJDK is used, because unlike Oracle Java, OpenJDK does not provide proprietary fonts by itself and
  • the fonts are not provided together with the template (see here).

This error may also occur if no font at all is used in the template. In this case Apache FOP automatically falls back to the font Times which may itself also not exist under these circumstances.

It is generally recommended to embed fonts that are used in templates into the templates in order to make the template system-independent. Instructions for embedding fonts in a template can be found here.

Unsupported fonts

There are fonts e.g. Google’s Noto fonts that are not supported by DCS and the underlying rendering engine Apache FOP. If a font is not supported by DCS then a warning similar to this

Unable to load font file: file:/Users/stse/Projects/hr/reflex-new/workspaces/dcs/lab/config/dcs/common/fonts/NotoSansHebrew-Regular.ttf. Reason:    java.lang.UnsupportedOperationException: coverage set class table not yet supported

is rendered into its log file. As long as these fonts are not supported by the underlying rendering engine, we recommend to use another font instead that is supported.
