REST API

The document creation service provides a REST interface to create documents from input data. When a caller sends a GET request, the document is created by the DCS and provided as a byte stream in the response. Although the DCS is capable of processing arbitrary input data, the most common input within the ReFlex environment will be application/json that is converted to a PDF or XML (or any other format) depending on the specified type.

Interface description

title: DCS REST Service API
baseUri: http://{host}:{port}/{dcsContext}
baseUriParameters:
  host:
    description: The name or IP address of the host on which the DCS is deployed.
  port:
    description: The TCP port under which the service is reachable
  dcsContext:
    description: The name of the deployed webapp (e.g. 'dcs').

/status:
  get:
    description: Retrieves the system status of the DCS.
  responses:
    200:
      description: The status of the DCS, indicating the DCS version and the
                   currently deployed document creation pipelines.
      body:
        application/json:
          example: |
            {
              "version": "1.21.0",
              "pipelines": ["hrlhxml","sample-application"]
            }
    404:
      description: The DCS is not available.
    500: 
      description: An internal error occurred when trying to determine the status.

/document:
  post:
    description: Return a document created from the posted data. The body and
                 response media types depend on the document type specified
                 via the 'type' parameter.
    queryParameters:
      type:
        displayName: Document Type
        type: string
        description: Defines the document type (template) from which the
                     document will be generated.
        example: sample-application
        required: true
      language:
        displayName: Language Code
        type: string
        description: Language code that will be used to translate text
                     within the document.
        example:  en
        required: true
      exitAfter:
        displayName: Exit After
        type: string
        description: The document creation pipeline is stopped after the processor
                     with the specified name and the intermediate result is returned
        example:  myProcessorName
        required: false
    body:
      application/json: !!null
      application/xml: !!null
    responses:
      200:
        body:
          "*/*":
            description: The byte stream of the created document. The media
                         type depends on the document type specified via
                         'type' parameter.
      400:
        body:
          text/html:
            description: When the specified 'exitAfter' processor name is undefined.
      404:
        body:
          text/html:
            description: When the specified 'type' is unknown or is missing.
      415:
        body:
          text/html:
            description: When the sent media type is not supported by the specified 'type'.
      500:
        body:
          text/html:
            description: When the requested type (template) has been wrongly configured or 
                         an unexpected error occurred during document creation.
