Testing and debugging pipelines

It is possible to debug the processing steps of a pipeline by stopping the pipeline earlier at a specific point. This feature can also be used to do automatic tests on intermediate processing steps like on FO xml content before it is converted to a binary format like PDF.

Enabling debugging

In order to debug or test a template it is necessary to specify named processor elements by setting the processor’s name attribute. You can use an arbitrary name as long as the name is unique within the pipeline.

Given you have configured the following pipeline:

<dcs:pipeline id="myPipeline">
  <dcs:jsonToXml name="0"/>
  <dcs:xslTransformation name="1" styleSheet="cspJsonXmlToIntermediateXml.xslt"/>
  <dcs:i18NXmlProcessor name="2"/>
  <dcs:xslTransformation name="3" styleSheet="intermediateXmlToFoXml.xslt"/>
  <dcs:fopProcessor name="4" mimeType="application/pdf"/>
</dcs:pipeline>

You can retrieve the FO xml by specififying the optional query parameter exitAfter:

dcs/document?type=myPipeline&language=en&exitAfter=3
