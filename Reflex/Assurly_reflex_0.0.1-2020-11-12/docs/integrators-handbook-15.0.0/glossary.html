Glossary

ReFlex Related Terms

Term Definition
hr | ReFlex Hannover Re system for automated risk assessment.
3PS Third Party Data Services - ReFlex component which provides an interchange mechanism for communication with external data providers.
ATT Assessment Test Tool - ReFlex component, part of the ReFlex Knowledge Editor (RKE). The Assessment Test Tool enables the user to test changes he has made on the knowledge base.
CEP Customer Experience Platform - ReFlex Component that provides a graphical interface between the end user and ReFlex RAS as well as many convenience features.
CEP integration scenario Combination of CEP, RAS and KB used on the Client side.
CEP UI User interface (frontend) component of ReFlex CEP, running in the user’s browser. It contains basic presentation logic and uses the ReFlex functionality via the (RESTful) CEP Backend Services.
DCS Document Creation Service - ReFlex component that can be used to create documents based on the ReFlex application JSON.
KB ReFlex Knowledge base - ReFlex core component. The knowledge base contains the medical and non-medical knowledge of ReFlex.
Knowledge base Compare Tool ReFlex Component, part of the ReFlex Knowledge Editor. The Compare Tool enables the user to compare different knowledge bases.
MI Management Information System - ReFlex Component that allows the creation of reports and statistical analysis based on de-personalised assessed applications. The system is hosted by Hannover Re. Those application need to be sent from from the integrating client system to the Hannover Re MI server.
UWB The Underwriting Workbench is a tool for the Underwriter and enables the Underwriter to manually underwrite applications which could not be assessed automatically at the point-of-sale (e.g. by ReFlex RAS / CEP). This includes capturing relevant information from the application and additional documents (e.g. lab results, APS/GPR, financial records, etc.), incorporating external data received by third-party data providers / systems and assessing all the information in order to decide whether and under which terms to accept or decline the application.
RAS Risk Assessment Services - ReFlex component that integrates the knowledge base and performs the risk assessment.
RAS integration scenario Combination of RAS and KB used on the Client side.
ReFlex: The Product The product ReFlex contains all software components that are not client specific. Product components can be used potentially for each and every client without being changed. Client specific aspects are solved via integration code (part of client integration project) and especially via KB. A KB is always client specific since it contains an individual rule set.
Risk profile Contains the information of one applicant (provided by the applicant and possibly 3rd party systems).
RKE ReFlex Knowledge Editor - ReFlex component that enables the user to edit the knowledge base, former WAT (for Web Admin Tool).
RMW RAS Middleware - deprecated name for the RAS REST API, part of the RAS component.
RSP ReFlex Standalone Portal - Component designed for clients whose state of digitalization does not allow for the RAS or CEP integration scenarios. RSP provides a web portal that is embedded into an existing sales process. It handles all of the steps required for a risk assessment in order to make it easy for a cedent to quickly replace a paper-based health declaration.
RTT Regression Test Tool - ReFlex Component, part of the ReFlex Knowledge Editor. The Regression Test Tool enables the user to perform regression tests on a knowlegde base after having made changes.
TED Cucumber Test Driver - Part of the ReFlex environment. The Cucumber Test Driver facilitates tests for the ReFlex components.
Translation Manager ReFlex Component, part of the ReFlex Knowledge Editor. The Translation Manager enables the user to import and export translation lists as well as to edit the translations of a knowledge base.

Knowledge base Terms and Objects

Term Definition
Accept Assessment result that declares that the insurance application can be accepted without any conditions.
AF Assessment Factor - Assessable element or property which triggers a specific rule set when selected. An assessment factor can be a disease, an occupation, a country, an avocation, or an other factor which usually carries a certain risk.
AFG Assessment Factor Group - Group of assessment factors which contains one rule set for each product, leading to one result representing the risk assessment.
BaseResult Drools results based on the KB solely.
CIC Critical Illness Cover - Cover: Critical Illness.
Cover Knowledge base term. The risk an insurance product covers (a product can have more than one cover). A cover is represented by a TORC in the DSL. The term cover is used to describe the composition of a product from the business perspective.
Decline Result that declares the assessment is not acceptable.
FlexRule Superordinate rule which aggregates or changes previous assessment results. Flexrules operate in layered stacks and create the final assessment for a product.
FlexRuleResult Results generated by FlexRules, based on BaseResults, risk profile and domain logic.
GPR General practitioner's result - Assessment result that declares that the insurance application can be accepted with the condition that a general practitioners report is submitted.
HospitalCash Cover: Hospital Cash.
Life Life Term / Life Endowment - Cover: Life Insurance.
LTC Long Term Care - Cover: Long Term Care.
Offer Result that declares the assessment is acceptable, only if certain additional conditions are met, for example exclusion of a cover, or at a higher rate.
PHI Permanent Health Insurance - Cover: Physical Health Insurance.
RTC Refer to client - Result that declares the applicant has to provide more specific information, for example on his earning capacity.
RTE Refer to expert - Result that declares the assessment has to be decided by an expert.
RTU Refer to underwriter Result that declares the assessment has to be decided by an underwriter.
TORC Type of risk covered - Knowledge base term. One key property of a product is the type of risk covered (it can have more than one TORC). A TORC as a DSL object is related to the cover of a product.
TPD Total and Permanent Disability - Cover: Total and Permanent Disability Insurance.
TPDany Cover: Total and Permanent Disability, any occupation.
TPDown Cover: Total and Permanent Disability, own occupation only.

Technical Terms

Term Definition
API Application Programming Interface - A set of routine definitions, protocols, and tools for building software and applications.
Drools Business rule management system written in Java.
DSL Domain Specific Language - The language used to specify products, questions, assessment rules, and more in the knowledge base.
PoS Point of Sale - Systems used at time and place where retail transactions are completed.
REST Representational State Transfer - Architectural style consisting of a coordinated set of components, connectors, and data elements within a distributed system typically communicating over HTTP.
UI User Interface - The area where interactions between humans and machines occur, which is in this case the web application platform.
URL Uniform Resource Locator - Reference to a web resource that specifies its location on a computer network and a mechanism for retrieving it. A URL is a specific type of Uniform Resource Identifier (URI).
XML Extensible Markup Language - A markup language used to transfer documents between information processing systems.
