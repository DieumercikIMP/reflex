Integrator’s Handbook

Welcome to the ReFlex Integrator’s Handbook. This document will provide you with detailed information about the integration of the delivered ReFlex software components. Furthermore, it will also focus on the installation and configuration of those artefacts.

This document is meant to be used by anyone who is commissioned in the technical procedure of installing and upgrading ReFlex components as well as integrating them with one another and the insurer’s software systems. Business use cases will be discussed only as far as necessary for these topics. The Integrator’s Handbook specifically does not provide details about the design and compilation of ReFlex knowledgeBases (KBs).

Structure of the document and how to read it

This document is divided into several parts: The first part will give a technical overview about ReFlex including its architecture and underlying concepts. It discusses update scenarios and provides a step-by-step installation guide for ReFlex components that are provided by Hannover Re. This first part is designed to be read top to bottom and finishes with some use cases that highlight some integration topics in detail.

The following parts contain detailed technical information about the integration of the ReFlex components. The reader will want to use it as reference material during the actual process of integrating ReFlex into the client software stack.

The document concludes with an appendix including a glossary and version specific update, migration and release notes.

Roles used in this document

  • Client / Customer: A client is typically an insurance company running ReFlex as part of their sales process. The client receives a license to be allowed to run ReFlex.
  • Integrator / Integration Company: An integrator does the integration work to get ReFlex integrated into the environment of a client. The integrator can be the IT organisation unit of a client (if existing) or a separate consulting company.
  • End user: End user is a person using ReFlex. This can be a call centre agent, an employee of the client or a person to be insured (e.g. via the internet).
  • ReFlex software vendor: Hannover Re is the software vendor of ReFlex providing the software and services needed to run ReFlex in a production mode (e.g. support). Hannover Re also provides the rule sets to run the questionnaire (KB).
  • Client Project Coordination (CPC): This role is responsible for defining best practices to run client projects as well as for coordinating currently running client projects. Additionally, the role provides a single point of contact to each market in order to answer questions coming from local offices, integrators or clients. The role is fulfilled by Hannover Re’s CPC team.

Integrator, client and Hannover Re are running the “client integration project” in order to gather all of the requirements concerning ReFlex integration from the client and to finally do a successful integration of ReFlex on the client side.

Getting help

In the case of any questions or problems concerning the installation, integration or update of ReFlex components, please do not hesitate to contact Hannover Re via reflex-integration-support@hannover-re.com.
