Multiple knowledge bases

Although RAS-REST is - potentially - capable of managing multiple knowledge bases with different IDs (possibly specifying different cedents) by appropriately setting ...ras.knowledgeBaseIdValidation configuration property (see sample configuration), doing so may raise one of the following issues. Therefore, it is strongly recommended to not enable the deployment of multiple knowledge bases with different IDs or cedents in production settings. RAS-REST supports the deployment of multiple versions of a knowledge base with the same ID and for a single cedent without any problems.

[MKB-01] Ambiguous knowledge bases

Currently, it is permitted to deploy two knowledge bases with the same ID and version but specifying different cedents. Both knowledge bases are grouped correctly by their cedents when issuing the following request:

GET /rs/knowledgebases/

The knowledge base ID and version are identical and both knowledge bases are listed:

{
    "knowledgeBases": [{
            "knowledgeBaseId": "AKnowledgeBase",
            "cedentId": "ACedent",
            "versions": {
                "1.0.0": {
                    "version": "1.0.0",
                    "creationDate": "...",
                    "creator": "...",
                    "links": [...]
                },
                "latest": {
                    "version": "1.0.0",
                    "creationDate": "...",
                    "creator": "...",
                    "links": [...]
                }
            }
        }, {
            "knowledgeBaseId": "AKnowledgeBase",
            "cedentId": "AnotherCedent",
            "versions": {
                "1.0.0": {
                    "version": "1.0.0",
                    "creationDate": "...",
                    "creator": ".",
                    "links": [...]
                },
                "latest": {
                    "version": "1.0.0",
                    "creationDate": "...",
                    "creator": ".",
                    "links": [...]
                }
            }
        }
    ]
}

However, RAS-REST provides no means to differentiate those knowledge bases, i.e. the URLs to access these two knowledge bases are the same. When, for example, retrieving the products for the knowledge base “AKnowledgeBase” in version 1.0.0 by issuing the following request:

GET /rs/knowledgebases/AKnowledgeBase/versions/1-0-0/products?language=en

it is not clear if the knowledge base for cedent “ACedent” or “AnotherCedent” should be used. If the knowledge base cannot be identified free of doubt by RAS-REST an error is raised. As the version can be specified by using the “latest” keyword this ambiguity will be even easier to create. It is strongly recommended - if you need to deploy multiple knowledge bases on one RAS-REST instance at all - to define unique knowledge base IDs to work around possible problems with the ambiguity.
