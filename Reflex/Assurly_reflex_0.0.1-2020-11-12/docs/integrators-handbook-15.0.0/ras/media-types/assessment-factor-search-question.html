Assessment factor search question

The assessment factor search question is used when the answer should be provided in form of one or more assessment factors. Typically, this question will trigger a search for assessment factors which then are added to the answer for the question.

The type specifier for a assessment factor search question is assessment-factor-search. For information about type specifiers see the description of media type questionnaire element.

The links part of the assessment factor search question’s JSON representation provides the corresponding URL for the assessment factor resource. That resource provides the facilities to search the knowledge base for assessment factors based on a query term and filters. More detailed examples are shown in the usage scenario section for assessment factor search.

Example (simplified JSON)

{
    "type": "assessment-factor-search",
    "id": "Diseases",
    "scope": "GLOBAL",
    "isNew": true,
    "caption": "Specify your diseases:",
    "meta": {...},
    "tags": {...},
    "hasReflexive": true,
    "answerState": "valid",
    "index": "1",
    "assignedTo": "Applicant",
    "constraints": {
        "required": true,
        "minNumberOfFactors": 1,
        "minNumberOfFactorsViolationMessage": "Too few factors"
        "maxNumberOfFactors": 3,
        "maxNumberOfFactorsViolationMessage": "Too many factors"
    },
    
    "currentSearchTerm": "diab",
    "searchId": "Diseases",
    "searchLanguage": "en",
    "searchContextQuestions": [...],
    "exposesNextUnusedFactor": true,
    "nextUnusedFactor": {
        "id": "Beneficiary01"
    },
    "links": [{
        "href": "...",
        "rel": ["search-assessmentfactors"],
        "method": "GET",
        "params": {
            "limit": null,
            "q": null,
            "questionId": "Diseases",
            "language": "en",
            "category": null,
            "questionnaire": null
        }
    }],
    "answer": [{
        "id": "AsthmaAttack",
        "label": "asthma attack",
        "category": "Medical"
    },
    ...
    {
        "id": "Diabetes",
        "label": "diabetes",
        "category": "Medical"
    }]
}

Properties

Note: Described here are only those properties that are directly related to an assessment factor search question. For a description of general question properties, see the description of media type question.

constraints (Object)

  • minNumberOfFactors (Number)
    The optional minimum number of answers which need to be provided.
    Further information on constraints can be found under general notes.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • maxNumberOfFactors (Number)
    The optional maximum number of answers which may be provided.
    Further information on constraints can be found under general notes.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • minNumberOfFactorsViolationMessage (String)
    The potential violation message if the number of given answers is smaller than the minNumberOfFactors.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • maxNumberOfFactorsViolationMessage (String)
    The potential violation message if the number of given answers is greater than the maxNumberOfFactors.
    This property is set by ReFlex/REST only and must not be modified by clients.

currentSearchTerm (String)

The optional search term that most recently has been used for the assessment factor search.

This property may be set by clients. It is neither interpreted nor modified by ReFlex/REST, and therefore remains unchanged until changed by a client.

searchId (String)

The ID that must be specified when searching for assessment factors that are to be added as answers to the question.

This property is set by ReFlex/REST only and must not be modified by clients. Clients do not need to evaluate this property. It is recommended to use the search-assessmentfactors link instead.

searchLanguage (String)

The language that must be specified when searching for assessment factors that are to be added as answers to the question.

This property is set by ReFlex/REST only and must not be modified by clients. Clients do not need to evaluate this property. It is recommended to use the search-assessmentfactors link instead.

searchContextQuestions (ArrayString)

The optional IDs of questions (aka search context questions) for which answers must be provided when searching for assessment factors that are to be used to answer this question.

This property is set by ReFlex/REST only and must not be modified by clients.

exposesNextUnusedFactor (Boolean)

States that this search question exposes next unused assessment factors if an unused assessment factor is available.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

nextUnusedFactor (Object)

The next factor that should be added as answer if an applicant requests a new section with questions.

  • id (String)
    The globally unique ID of the assessment factor.
    This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

  • label (String)
    The text to be displayed for the assessment factor.
    This property is set by ReFlex/REST only and must not be modified by clients. This property might be missing.

This property is set by ReFlex/REST only and must not be modified by clients. It is only present if the corresponding search variable is configured to expose the next unused factors. For more informaion see the corresponding usage scenario

links (ArrayLink)

The hypermedia links for performing further actions related to the assessment factor search question. Links with the following relation types are provided:

  • search-assessmentfactors: Search for assessment factors applicable to the assessment factor search question. Note, that the questionnaire parameter of this link is only present if the search questions requires further search context questions in order for an assessment factor search to be performed.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

The parameter questionId

answer (ArrayAssessmentFactor)

The assessment factors that have been given for an answer to the assessment factor search question.

This property may be added, removed, and modified by clients.

DSL mapping

The question shown in the example above maps to the following definition of an assessment variable in ReFlex/DSL.

AssessmentVariable Diseases {
    type AssessmentFactorSearch
    matches category==Medical
    minNumberOfFactors 1
    maxNumberOfFactors 3
    global
}

There are two additional use cases, that are supported by the assessment factor search:

  • the assessment factor search question is part of a structure
  • there is a list with elements of type structure, where an assessment factor search question is part of the structure

In these two cases, the questionId encoded in the JSON is different from the simple definition above. In the nested use the id is prefixed with the id of the containing element.

Given a assessment factor search as an element of a structure, as in the following example, the questionId used in the JSON is of the form Diseases.AssessmentVariableInStructure:

AssessmentVariable Diseases {
    type DiseasesStructure
}

Structure DiseasesStructure {
    AssessmentVariable AssessmentVariableInStructure {
        type AssessmentFactorSearch
    }
}

If the assessment factor search is an element of a structure that is part of a list,as illustrated by the following example, the questionId has the form Diseases.prototype.AssessmentVariableInStructure:

Structure DiseasesStructure {
    AssessmentVariable AssessmentVariableInStructure {
        type AssessmentFactorSearch
    }
}

List DiseasesListType {
    of DiseasesStructure
}

AssessmentVariable Diseases {
    type DiseasesListType
}

Minimised information set

The minimised information set for a assessment factor search question consists of the following properties:

  • type
  • id
  • scope
  • searchContextQuestions Only if present.

The minimised information set for the optional answer property is an array of assessment factor objects each consisting of the id and label properties only.

Example (JSON)

{
    "type": "assessment-factor-search",
    "id": "Diseases",
    "scope": "GLOBAL",
    "answer": [
        { "id": "AsthmaAttack", "label": "asthma attack" },
        { "id": "Diabetes", "label": "diabetes" }
    ]
}
