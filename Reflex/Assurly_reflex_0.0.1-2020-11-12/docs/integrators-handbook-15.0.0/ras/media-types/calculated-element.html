Calculated element

An element for which a value has been automatically calculated during a risk assessment. Calculated elements are exposed as part of a risk assessment’s calculated data section.

There are different types of calculated elements, which differ in the properties (and the data types) they support. Nevertheless, all calculated elements share a set of common properties.

Example (simplified JSON)

{
    "id": "MyId",
    "type": "...",
    "caption": "A caption",
    "tags": { "tag1":["value1", ..., "valueN"], ..., "tagN":[]},
    "scope": "GLOBAL",
    "value": ...
}

Properties

id (String)
The globally unique ID of the calculated element.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

type (String)
The specifier indicating of what type the calculated element is. A type specifier may have one of the following values:
  • date
  • number
  • string
  • singleselection
  • multiselection

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

caption (String)
The text to be displayed for the calculated element.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

tags (Object)

The optional tags that have been applied to the element. A tag is basically a key-value pair where the key is a string denoting a tag’s name and the value is a (possibly empty) list of strings.

This property is set by ReFlex/REST only and must not be modified by clients.

scope (String)
The logical scope that the element is defined in.

The information regarding scope is encoded in a compound string in the form <visibility> [ ":" <reference-id> ]. Visibility can have one of three possible values: GLOBAL, INSURANCEPRODUCT or ASSESSMENTFACTOR.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

  • In case of visibility GLOBAL, the element is present globally at most one time. In this case the scope has no reference-id.
  • For the visibility of INSURANCEPRODUCT the element is present per product at most one time. In this case the scope includes a reference-id. The reference-id is the ID of a product.
  • For the visibility of ASSESSMENTFACTOR the element is present per assessment factor at most one time. In this case the scope also includes a reference-id. The reference-id is the ID of an assessment factor.
value (any)
The value that has been calculated for the element during a risk assessment. The actual data type and format of the value depends upon the type of the element. See descriptions of the specific calculated elements documented in the sections below.

This property is set by ReFlex/REST only and must not be modified by clients. This property may be null if the calculated element has been requested during a risk assessment but could not (yet) be computed.

Calculated date

An element that has been calculated during a risk assessment with a date value.

The type specifier for a calculated date is date.

Example (JSON)

{
    "id": "CalculatedDate",
    "type": "date",
    "caption": "A calculated date with precision day",
    "scope": "GLOBAL",
    "value": "2017-02-01"
}

Properties

value (String)
The value that has been calculated for the element during a risk assessment. This value is a textual representation of a date specified in ISO 8601 format. Depending on the precision with which the date has been calculated, this value conforms to the following patterns:
  • year: yyyy (example: 2015)
  • month: yyyy '-' MM (example: 2015-01)
  • day: yyyy '-' MM '-' dd (example: 2015-01-30)
  • hour: yyyy '-' MM '-' dd 'T' HH (('+' | '-') HH ':' mm | 'Z')? (examples: 2015-01-30T12, 2015-01-30T12+01:00)
  • minute: yyyy '-' MM '-' dd 'T' HH ':' mm (('+' | '-') HH ':' mm | 'Z')? (examples: 2015-01-30T12:05, 2015-01-30T12:05+01:00)

Thereby,

  • yyyy is the year of the date (4 digits).
  • MM is the numerical month of within the year (2 digits, ranging from 01 to 12).
  • dd is the numerical day within the month (2 digits, ranging from 01 to 31).
  • HH is the hour within a day (2 digits, ranging from 00 to 23).
  • mm is the minute within an hour (2 digits, ranging from 00 to 59).

DSL mapping

The calculated date shown in the example above maps to the following definition of a calculated assessment variable in ReFlex/DSL.

AssessmentVariable CalculatedDate {
    type Date
    precision day
    global
    calculate {
        CalculatedDate = ReferenceDate + 1 month
    }
}

Calculated number

An element that has been calculated during a risk assessment with a numeric value.

The type specifier for a calculated date is number.

Example (JSON)

{
    "id": "CalculatedNumber",
    "type": "number",
    "caption": "A calculated integer number",
    "scope": "GLOBAL",
    "value": 21
}

Properties

value (Number)
The value that has been calculated for the element during a risk assessment. This value is a number that, depending on the precision with which the number has been calculated, may have decimal places (e.g. 10.01).

DSL mapping

The calculated number shown in the example above maps to the following definition of a calculated assessment variable in ReFlex/DSL.

AssessmentVariable CalculatedNumber {
    type Integer
    global
    calculate {
        CalculatedNumber = year(ReferenceDate) - year(InsuredPerson.DateOfBirth)
    }
}

Calculated string

An element that has been calculated during a risk assessment with a string value.

The type specifier for a calculated string is string.

Example (JSON)

{
    "id": "CalculatedString",
    "type": "string",
    "caption": "A calculated string",
    "scope": "GLOBAL",
    "value": "Hello world!"
}

Properties

value (String)
The value that has been calculated for the element during a risk assessment. This value is a string.

DSL mapping

The calculated string shown in the example above maps to the following definition of a calculated assessment variable in ReFlex/DSL.

AssessmentVariable CalculatedString {
    type String
    global
    calculate {
        CalculatedString = "Hello World"
    }
}

Calculated single selection

An element that has been calculated during a risk assessment with a single selection option. Note, that values which have been calculated to a boolean are considered being “special” single selection values (i.e. either a “Yes” or a “No” option that may be selected) and therefore are represented by a calculated single selection.

The type specifier for a calculated date is singleselection.

Example (JSON)

{
    "id": "CalculatedBoolean",
    "type": "singleselection",
    "caption": "A single option calculated for a boolean",
    "scope": "GLOBAL",
    "value": {
        "id": "Yes",
        "label": "Yes"
    }
}

Properties

value (ObjectOption)
The value that has been calculated for the element during a risk assessment. This value is an object that represents the single option selected from a set of possible options, having the following properties:

  • id (String)
    The ID of the selected option. In case the value has been calculated from a boolean value, the id is either Yes (for a boolean value of true) or No (for a boolean value of false).

  • label (String)
    The text to be displayed for the selected option.

DSL mapping

AssessmentVariable CalculatedBoolean {
    type Boolean
    global
    calculate {
        if Age >= 18 then CalculatedBoolean = true
        if Age < 18 then CalculatedBoolean = false
    }
}

Calculated multi selection

An element that has been calculated during a risk assessment with a multi selection option.

The type specifier for a calculated multi selection is multiselection.

Example (JSON)

{
    "id": "CalculatedMultiSelection",
    "type": "multiselection",
    "caption": "Multiple options calculated for a multi selection",
    "scope": "GLOBAL",
    "value": [ 
        {
            "id": "SmokeTypeSelection.Cigars",
            "label": "Cigars"
        },
        {
            "id": "SmokeTypeSelection.Pipes",
            "label": "Pipes"
        }
    ]
}

Properties

value (ObjectOption)
The value that has been calculated for the element during a risk assessment. This value is an array containing objects that represents the options selected from a set of possible options, having the following properties:

  • id (String)
    The ID of the selected option. In case the value has been calculated from a boolean value, the id is either Yes (for a boolean value of true) or No (for a boolean value of false).

  • label (String)
    The text to be displayed for the selected option.

DSL mapping

AssessmentVariable CalculatedMultiSelection {
    type SmokeTypeSelection
    global
    calculate {
        if Age >= 18 then CalculatedMultiSelection = {SmokeTypeSelection.Pipes, SmokeTypeSelection.Cigars}
        if Age < 18 then CalculatedMultiSelection = {SmokeTypeSelection.Cigarettes}
    }
}

Thereby, the type SmokeTypeSelection is defined as follows:

Selection SmokeTypeSelection {
    Cigarettes, Cigars, Pipes, None
}
