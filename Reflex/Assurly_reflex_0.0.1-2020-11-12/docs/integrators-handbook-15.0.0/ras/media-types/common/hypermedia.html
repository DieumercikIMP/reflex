Hypermedia links

Hypermedia links are used to provide navigation options: For instance, the /knowledgebases resource provides links to subresources of available knowledge base versions. The client can navigate to these subresources in order to search for products or assessment factors and to process risk assessments. The links are also embedded in the risk assessment JSON.

    "links": [
        {
            "href": "/rs/knowledgebases/{kb}/versions/{version}/riskassessments",
            "rel": [
                "process-riskassessments",
                "self"
            ],
            "method": "POST"
        }
    ]

The shown link can be invoked with HTTP POST to perform a risk assessment (rel = process-riskassessment). The link also identifies the resoruce itself (rel = self).

The links have the following structure:

Element Comment
href Specifies a relative URI path that identifies a subresource.
rel This array of strings holds relation types that define the semantic of the link.
method This HTTP method can be used to access the subresource.
params Some links also have a map of query params that are used to call the resource.

Selecting the latest knowledge base version

A client that wants to work with the latest version can use the URI alias ‘latest’. The relative URI path would be for instance: /rs/knowledgebases/{kb}/versions/latest/riskassessments

However, we strongly recommend using the /knowledgebases resource to find out the links of the latest version.

Clients should not construct the URIs. This is a well-known anitpattern. Therefore, a client should only use the URIs that are provided by the API.

Link semantics

The semantics of the links are specified by relation types.

Relation Type Comment
self Identifies the resource itself
next Retrieve the next page of a paged result
prev Retrieve the previous page of a paged result
index Retrieve basic interaction links and status information for RAS-REST
status Retrieve general status information about RAS-REST
knowledgebases-list Retrieve a list of deployed knowledge base versions and corresponding interaction links
knowledgebase-status Retrieve detailed status information of a deployed knowledge base
knowledgebase-deploy Trigger the deployment of a knowledge base
knowledgebase-undeploy Trigger the undeployment of a knowledge base
process-riskassessments Process risk assessment data using a knowledge base
change-products Perform a change of product configuration for previously assessed risk assessment data, using a knowledge base
search-products Search for all insurance products that are defined in a knowledge base
search-assessmentfactors Search for assessment factors defined in a knowledge base
translations Translate the risk assessment into another language

Assessment factor search questions

Assessment factor search questions used in the risk assessment JSON documents also have links. Each question provides a link to the assessment factor search resource. The link can be used by the client to answer the question.

    {
      "type": "assessment-factor-search",
      "id": "LifeProduct.PreviousDiseases",
      ...
      "links": [
        {
          "href": "/rs/knowledgebases/knowledgebase/versions/1-13-0-snapshot-20150101-1000/assessmentfactors",
          "rel": [
            "search-assessmentfactors"
          ],
          "method": "GET",
          "params": {
            "q": null,
            "questionId": "LifeProduct.PreviousDiseases",
            "language": "en"
          }
        }
      ]
    }
