Meta data

General

Every object of the JSON documents interchanged between clients and resources of RAS-REST is potentially capable of conveying so-called meta data blocks. The purpose of a meta data block is to give clients of RAS-REST the possibility to transport additional information needed for their functionality. Further, RAS-REST itself uses meta data blocks for additional information that is needed to ensure proper functionality.

Generally, a meta data block is an object for which the following applies:

  • The name of the property under which a meta data block may be present is always "meta".
  • The properties directly contained in a meta data block act as namespaces into which different clients of RAS-REST may incorporate their information, without the risk of naming conflicts.
  • The information contained in a namespace must be an object. The structure of that object may be arbitrarily chosen - as long as it is valid JSON.

Summarizing, each meta data block must conform to the following scheme:

"meta": {
    "namespace_1": {...},
    ...
    "namespace_N": {...}
}

Note: The namespace "ras" is reserved by RAS-REST. Its content must not be modified by clients, however clients are allowed to read the content. Further, clients must not rely on information being present in the "ras" namespace, except for those that are explicitly documented in the description of the media type objects.

Note: Although clients are allowed to apply meta data blocks to nearly all objects of the interchanged JSON documents, their content is not always retained when invoking a method of a REST resource. The detailed description of the JSON documents for the different resources specify which objects may be safely annotated with meta data blocks.

Examples

RAS knowledge base information applied to a risk assessment at top level

"meta": {
    "ras": {
        "knowledgebase": {
            "id": "MyKnowledgeBase",
            "version": "1.2.0",
            "cedent": "MyCedent"
        },
        "version": "12.0.0"
    }
}

Internal RAS mapping information applied to a question

"meta": {
    "ras": {
        "questionMapping": {
            "sourceAssessmentVariableType": "DateVariable"
        }
    }
}

UI related rendering information applied to a question

"meta": {
    "ui": {
        "level": 1,
        "isActive": true,
        "autoSubmit": false
    }
}
