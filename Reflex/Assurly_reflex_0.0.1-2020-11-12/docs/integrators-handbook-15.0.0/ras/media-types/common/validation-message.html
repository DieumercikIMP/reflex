Validation message

A message caused by ReFlex/RAS Runtime as a result of validating some data relevant for a risk assessment using constraints or validation rules defined in a knowledge base.

Example (JSON)

{
    "message": "The text of this message", 
    "type": "error",
    "isNew": true
}

Properties

message (String)
The text of the message to be displayed.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

type (String)
The type of the message, specifying the severity. Possible values are:
  • "error": The data given is invalid and will not be accepted by the knowledge base.
  • "warning": The data will be accepted by the knowledge base, but there is something about the data that the user should be warned about.
  • "info": The data given is valid, but there is something about the data that the user should be notified about.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

isNew (boolean)
A flag indicating if the message has been newly added, compared to the preceding assessment.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.
