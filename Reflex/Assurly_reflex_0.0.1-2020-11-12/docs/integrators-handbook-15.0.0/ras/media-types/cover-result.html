Cover result

A cover result is an assessment result that contains the ratings for a single insurance cover. Each cover result is contained in the assessment result of a specific insurance product or insurance rider.

Note that there is no notion of an aggregated overall (“roll-up”) rating for a cover - at least not in general. See description of the ratings object for further details.

Example (simplified JSON)

{
    "id": "LifeCover",
    "caption": "Life Cover",
    "ratings": {...},
    "flexRules": [{...}, ..., {...}],
    "baseRatings": {...}
}

Properties

id (String)
The globally unique ID of the insurance cover associated with this cover result.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

caption (String)
The text to be displayed for the cover associated with this cover result.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

ratings (ObjectRatings)
The ratings for the insurance cover associated with this cover result. These ratings represent the result after the FlexRules specified in the definition of the insurance cover have been executed. Note that an arbitrary number of ratings may be present for a cover result.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

flexRules (ArrayFlexRuleResult)
The execution information for the FlexRules specified in the definition of the insurance cover that is associated with this cover result.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can only expect this property to be present if FlexRule results are enabled in the configuration.

baseRatings (ObjectRatings)
The base ratings for the insurance cover associated with this cover result. These ratings represent the result before the FlexRules specified in the definition of the insurance cover have been executed. Note that an arbitrary number of ratings may be present for a cover result.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can only expect this property to be present if base ratings are enabled in the configuration.
