Date question

A question that is expecting a date for an answer.

The type specifier for a date question is date. For information about type specifiers see description of media type questionnaire element.

Example (simplified JSON)

{
    "type": "date",
    "id": "InsuredPerson.DateOfBirth",
    "scope": "ASSESSMENTFACTOR:InsuredPerson",
    "isNew": true,
    "meta": {...},
    "tags": {...},
    "hasReflexive": false,
    "answerState": "valid",
    "index": "1",
    "assignedTo": "Applicant",
    
    "constraints": {
        "required": true,
        "minValue": "1900-01-01",
        "maxValue": "2015-09-01",
        "minValueViolationMessage": "Date must be later than 01.01.1900",
        "maxValueViolationMessage": "Date must be earlier than 01.09.2015"
    },
    "precision": "day",
    "answer": "1970-01-01"
}

Properties

Note: Described here are only those properties that are directly related to a date question. For a description of general question properties, see the description of media type question.

constraints (Object)

  • minValue (String)
    The optional minimum date with which the question may be answered. The same syntax as described for the property answer applies for this property.
    This property is set by ReFlex/REST only and must not be modified by clients.
    Note, that in case no explicit time zone offset is specified for this constraint, the date is to be interpreted as being in UTC time zone (see also general notes about time zones).
    Further information on constraints can be found under general notes.

  • maxValue (String)
    The optional maximum date with which the question may be answered. The same syntax as described for the property answer applies for this property.
    This property is set by ReFlex/REST only and must not be modified by clients.
    Note, that in case no explicit time zone offset is specified for this constraint, the date is to be interpreted as being in UTC time zone (see also general notes about time zones).
    Further information on constraints can be found under general notes.

  • minValueViolationMessage (String)
    The potential violation message if the value would be set to a value earlier than the minValue.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • maxValueViolationMessage (String)
    The potential violation message if the value would be set to a value later than the maxValue.
    This property is set by ReFlex/REST only and must not be modified by clients.

precision (String)
The precision with which the date answer has to be be provided. Possible values are:
  • year: A date specifying a year only, e.g. 1999.
  • month: A date precise up to a month in a year, e.g. 1999-12.
  • day: A date precise up to a single day, e.g. 1999-12-31.
  • hour: A date precise up to the hour of a single day, e.g. 1999-12-31T23.
  • minute: A date precise up to the minute of an hour of a single day, e.g. 1999-12-31T23:59.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

answer (String)
The date answer given to the question. The date has to be specified in ISO 8601 format, according to the following patterns (depending upon the precision of the date):
  • year: yyyy (example: 2015)
  • month: yyyy '-' MM (example: 2015-01)
  • day: yyyy '-' MM '-' dd (example: 2015-01-30)
  • hour: yyyy '-' MM '-' dd 'T' HH (('+' | '-') HH ':' mm | 'Z')? (examples: 2015-01-30T12, 2015-01-30T12+01:00)
  • minute: yyyy '-' MM '-' dd 'T' HH ':' mm (('+' | '-') HH ':' mm | 'Z')? (examples: 2015-01-30T12:05, 2015-01-30T12:05+01:00)

Thereby,

  • yyyy is the year of the date (4 digits).
  • MM is the numerical month of within the year (2 digits, ranging from 01 to 12).
  • dd is the numerical day within the month (2 digits, ranging from 01 to 31).
  • HH is the hour within a day (2 digits, ranging from 00 to 23).
  • mm is the minute within an hour (2 digits, ranging from 00 to 59).

Note, that when specifying a date for an answer, the precision must be taken into account, as described above.

Also note, that in case no explicit time zone offset is specified, the date will be interpreted by RAS-REST as being in UTC time zone (see also general notes about time zones).

This property may be added, removed, and modified by clients.

DSL mapping

The question shown in the example above maps to the following definition of an assessment variable in ReFlex/DSL if required in the context of the assessment factor InsuredPerson.

AssessmentVariable DateOfBirth {
  type Date
  precision day
  maxValue #2015-09-01#
  minValue #1900-01-01#
}

Translation:

DateOfBirth/MinValueNotReached/en = Date must be later than 01.01.1900
DateOfBirth/MaxValueExceeded/en = Date must be earlier than 01.09.2015

Minimised information set

The minimised information set for a date question consists of the following properties:

  • type
  • id
  • scope
  • precision

For the optional answer property there is no minimised information set available.

Example (JSON)

{
    "type": "date",
    "id": "InsuredPerson.DateOfBirth",
    "scope": "ASSESSMENTFACTOR:InsuredPerson",
    "precision": "day",
    "answer": "1970-01-01"
}
