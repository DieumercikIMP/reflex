List question

A question that is expecting a list of answered questions for an answer. Thereby, the answered questions all must conform to a so-called prototype question, defined by the list question.

The type specifier for a list question is list. For information about type specifiers see description of media type questionnaire element.

Examples (simplified JSON)

List of text

{
    "type": "list",
    "id": "InsuredPerson.DrugsTaken",
    "scope": "ASSESSMENTFACTOR:InsuredPerson",
    "isNew": true,
    "caption": "Which drugs are you taking?",
    "meta": {...},
    "tags": {...},
    "hasReflexive": false,
    "answerState": "valid",
    "index": "1",
    "assignedTo": "Applicant",
    "constraints": {
        "required": true,
        "minSize": 2,
        "minSizeViolationMessage":"Must have at least 2 answers.",
        "maxSize": 5,
        "maxSizeViolationMessage":"Can have at most 5 answers.",
    },
    
    "prototype": {
        "type": "text",
        "id": "prototype",
        ...
    },
    "answer": [
        {
            "type": "text",
            "id": "prototype",
            ...,
            "answer": "Aspirin"
        },
        {
            "type": "text",
            "id": "prototype",
            ...,
            "answer": "Insulin"
        }
    ]
}

Properties

Note: Described here are only those properties that are directly related to a list question. For a description of general question properties, see the description of media type question.

constraints

  • minSize (Number)
    The optional minimal number of elements that the answer given to the question must have.
    Further information on constraints can be found under general notes.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • minSizeViolationMessage (String)
    The potential violation message if an answer for the list question contains too few elements.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • maxSize (Number)
    The optional maximal number of elements that the answer given to the question must not exceed.
    Further information on constraints can be found under general notes.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • maxSizeViolationMessage (String)
    The potential violation message if an answer for the list question contains too many elements.
    This property is set by ReFlex/REST only and must not be modified by clients.

prototype (ObjectQuestion)
The prototype question that all questions given as an answer must conform to.

The prototype question can be of any question type. It will have the same scope as the list question within which it is defined. The index of a prototype question will always be "0" (this may be changed in later releases). Further, the prototype question (or any further question contained in the prototype) does not have an isNew or answerState attribute.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

answer (ArrayQuestion)
The list of answered questions given for an answer to the list question. Each question given as an answer must conform to the type of the list’s prototype question, and must contain an answer property.

A typical strategy for adding an answer to a list question is to “copy” the prototype question into the answer array and then set the answer property of the copied prototype question. Thereby, the ID of the prototype question may - but does not have to - be retained. It is thus possible for clients to encode additional information into the ID. For example, a client may specify IDs like InsuredPerson.DrugsTaken_0, InsuredPerson.DrugsTaken_1 etc, in order to encode the row indices of the answers.

Note that for a question to be accepted as an answer to a list question not all information of the question is necessary. For the question to be accepted as an answer the data defined by the question’s minimised information set set is required in order for the question to be accepted as an answer. In case a reduced information set is used for an answer, the missing information will be restored when an assessment is performed.

This property may be added, removed, and modified by clients.

DSL mapping

The question shown in the List of text example above maps to the following definition of an assessment variable in ReFlex/DSL if required in the context of the assessment factor InsuredPerson.

AssessmentVariable DrugsTaken {
    type DrugTakenList
    minSize 2
    maxSize 5 
}

Thereby, the type DrugTakenList is defined as follows:

List DrugTakenList {
    of String
}

The question shown in the List of structure example above maps to the following definition of an assessment variable in ReFlex/DSL if required in the context of the assessment factor InsuredPerson.

AssessmentVariable DrugsTaken {
    type DrugTakenList
    minSize 2
    maxSize 5
}

Thereby, the type DrugTakenList is defined as follows:

List DrugTakenList {
    of DrugTaken
}

Where the DrugTaken structure is defined as follows:

Structure DrugTaken {
    AssessmentVariable DrugName { 
        type String 
    }
    AssessmentVariable DrugTreatedDisease { 
        type AssessmentFactorSearch
        matches category==Medical  
    }
}

Minimised information set

The minimised information set for a list question consists of the following properties:

  • type
  • id
  • scope

The minimised information set for the optional answer property is an array of answered questions using the minimised information set appropriate for the corresponding question type.

Example (JSON)

{
    "type": "list",
    "id": "InsuredPerson.DrugsTaken",
    "scope": "ASSESSMENTFACTOR:InsuredPerson",
    "answer": [
        {
            "type": "text",
            "id": "prototype",
            "scope": "ASSESSMENTFACTOR:InsuredPerson",
            "answer": "Aspirin"
        },
        {
            "type": "text",
            "id": "prototype",
            "scope": "ASSESSMENTFACTOR:InsuredPerson",
            "answer": "Insulin"
        }
    ]
}
