Number question

A question that is expecting a numerical answer.

The type specifier for a number question is number. For information about type specifiers see description of media type questionnaire element.

Example (simplified JSON)

{
  "type": "number",
  "id": "InsuredPerson.Weight",
  "scope": "ASSESSMENTFACTOR:InsuredPerson",
  "isNew": true,
  "caption": "Weight [kg]?",
  "meta": { ... },
  "tags": { ... },
  "hasReflexive": false,
  "answerState": "valid",
  "index": "1",
  "assignedTo": "Applicant",
  
  "constraints": {
    "required": true,
    "step": "0.1",
    "minValue": 30.0,
    "maxValue": 200.0,
    "minValueViolationMessage": "Value must be greater than 30.0",
    "maxValueViolationMessage": "Value must be smaller than 200.0"
  },
  "precision": 1,
  "unit": "[kg]",
  "defaultUnit": {
    "id": "[kg]",
    "label": "kg",
    "helpText": "The unit is kg."
  },
  "availableUnits": [
    {
      "id": "[kg]",
      "label": "kg",
      "helpText": "The unit is kg."
    },
    {
      "id": "[lb]",
      "label": "lb",
      "helpText": "The unit is lb."
    },
    {
      "id": "[st]",
      "label": "st",
      "helpText": "The unit is st."
    }
  ],
  "answer": 75.5
}

Properties

Note: Described here are only those properties that are directly related to a number question. For a description of general question properties, see the description of media type question.

constraints (Object)

  • step (String)
    The step by which an answer to the number question may be increased or decreased. The value for the step can be directly derived from the precision using the following formula:
    step := 10-precision The step is therefore just a convenience for clients.
    This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

  • minValue (Number)
    The optional minimum number with which the question may be answered.
    Further information on constraints can be found under general notes.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • maxValue (Number)
    The optional maximum number with which the question may be answered.
    Further information on constraints can be found under general notes.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • minValueViolationMessage (String)
    The potential violation message if the value would be set to a value smaller than the minValue.
    This property is set by ReFlex/REST only and must not be modified by clients.

  • maxValueViolationMessage (String)
    The potential violation message if the value would be set to a value greater than the maxValue.
    This property is set by ReFlex/REST only and must not be modified by clients.

precision (Number)
The number of relevant decimal places for the question’s numerical answer.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

unit (String)
The id of the selected unit.

This property can be set by the client as part of the answer. The given unit id must correspond with one of the availableUnits. If the given unit id is not part of availableUnits, then the request is not valid and an error is returned by ReFlex/REST.

If the client does not provide an unit id, then ReFlex/REST automatically sets the defaultUnit. If the client provides an unit id without valid numeric answer, then the unit will be ignored silently. If for instance the client removes the anwser, then ReFlex/REST will remove the unit as well.

defaultUnit (ObjectUnit)
The default unit defined for this question in the knowledge base.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be present if there is at least one available unit.

availableUnits (ArrayUnit)
All units that are supported to answer this question.

This property is set by ReFlex/REST only and must not be modified by clients.

answer (Number)
The numerical answer given to the question. If a number with higher precision than defined by the precision property is specified, rounding will occur. Thereby, the rounding mode specified in the knowledge base will be applied by ReFlex/RAS.

The numerical answer must not have more than 101 integer digits and not more than 100 fractional digits.

This property may be added, removed, and modified by clients.

DSL mapping

The question shown in the example above maps to the following definition of an assessment variable in ReFlex/DSL if required in the context of the assessment factor InsuredPerson.

AssessmentVariable Weight {
    type Decimal
    minValue 30.0
    maxValue 200.0
}

Thereby, the type Decimal is defined as follows:

NumberType Decimal {
    internalPrecision 1
}

Translation:

Weight/MinValueNotReached/en = Value must be greater than 30.0
Weight/MaxValueExceeded/en = Value must be smaller than 200.0

Minimised information set

The minimised information set for a number question consists of the following properties:

  • type
  • id
  • scope
  • precision

For the optional answer property there is no minimised information set available. The unit property is also optional and should be part of the minimised information set if the numeric question supports units and the client wants to specify a specific unit. In case of a missing unit property the default unit is used.

Example (JSON)

{
  "type": "number",
  "id": "InsuredPerson.Weight",
  "scope": "ASSESSMENTFACTOR:InsuredPerson",
  "precision": 1,
  "unit": "[kg]",
  "answer": 75.5
}
