Product configuration

The configuration of products and riders contained in product-change data.

Example (JSON)

{
    "LifeProduct": {
        "attributes": {
            "InsuranceSum": 100000,
            "PolicyDuration": 10
        },
        "riders": {
            "CICRider": {},
            "ADBRider": {
                "attributes": {
                    "InsuranceSum": 200000
                }
            }
        }
    },
    "TPDownProduct": {
        "riders": {
            "TPDownRider": {}
        }
    }
}

Properties

product-ID0..∗ (ObjectProductData)

A product configuration object may contain an arbitrary number of properties. The name of each property must be the ID of a product defined in the knowledge base. The value must either be an empty object, or an object of additional product configuration data as described below (see “Product data”).

Product data

Properties

attributes (Object)

The optional attributes for the product that should be considered when applying the product configuration.

The attributes must be specified as an object containing an arbitrary number of properties. Each property must correspond to a product-scoped question defined in the knowledge base. The name of a property must be the un-scoped ID of the corresponding question (e.g. “InsuranceSum” and not “LifeProduct.InsuranceSum”).

The value of a property depends on the type of the question corresponding to the product attribute. Thereby, the value must be of the same format as the value of the answer property defined for answer data objects.

Note, that only a subset of answer types are supported for product attributes. The following answer types are being supported for product attributes:

  • Text
  • Number
  • Date
  • Single-Selection
  • Multi-Selection
riders (ObjectRiderConfiguration)

The optional configuration of a product’s riders that should be considered when applying the product configuration.

Rider configuration

Properties

rider-ID0..∗ (ObjectRiderData)

A rider configuration object may contain an arbitrary number of properties. The name of each property must be the ID of a rider defined in the knowledge base that is a supported rider of the product corresponding to the product data in which it is contained. The value must either be an empty object, or an object of additional rider configuration data as described below (see “Rider data”).

Rider data

Properties

attributes (Object)

The optional attributes for the rider that should be considered when applying the product configuration.

The attributes must be specified as an object containing an arbitrary number of properties. Each property must correspond to a product-scoped question defined in the knowledge base. The name of a property must be the un-scoped ID of the corresponding question (e.g. “InsuranceSum” and not “ADBRider.InsuranceSum”).

The value of a property depends on the type of the question corresponding to the rider attribute. Thereby, the value must be of the same format as the value of the answer property defined for answer data objects.

Note, that only a subset of answer types are supported for rider attributes. The following answer types are being supported for rider attributes:

  • Text
  • Number
  • Date
  • Single-Selection
  • Multi-Selection
