Provided assessment factor

An assessment factor contained in the assessmentFactors section of a provided data block, that should be added automatically to a risk profile.

Examples

Provided assessment factor without processing info (JSON)

{
    "id": "HeartAttack",
    "category": "Medical"
}

Processing info attached to a provided assessment factor (JSON)

{
    "meta": {
        "ras": {
            "processing": {
                "error": "Assessment factor could not be found",
            }
        }
    },
    "id": "HeartAttack",
    "category": "Medical"
}

Properties

meta (ObjectMetaData)

The optional meta data applied to the provided answer.

Meta data in the “ras” namespace may contain processing information, disclosing warning and error conditions that occurred when processing the assessment factor.

id (String)
The ID of the assessment factor that is to be added.

This is a mandatory property that must be specified by clients.

category (String)
The ID of the assessment category that the assessment factor belongs to.

This is an optional property. Nevertheless, we strongly advice clients to specify the category.

Automatic processing

  • Automatic processing of provided assessment factors is done every time a risk assessment is performed.
  • A provided assessment factor is added to a risk profile only if there exists an assessment factor in the knowledge base having the specified id and category (if any) and no assessment factor with the specified id is already contained in the risk profile.
  • Adding a provided assessment factor to the risk profile does not cause a question to be answered. This is a substantial difference compared to adding an assessment factor by automatically answering a question of type assessment-factor-search.
  • If no assessment factor with the specified id and category (if any) exists in the knowledge base, then it is silently ignored.
