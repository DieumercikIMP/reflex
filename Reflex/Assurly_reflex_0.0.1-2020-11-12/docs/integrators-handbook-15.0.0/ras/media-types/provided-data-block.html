Provided data block

A block of data provided by an external source which has been added to the provided data section of a risk profile.

Example (simplified JSON)

{
    "timestamp": "2015-01-01T13:59:59+01:00",
    "meta": {...},
    "source": {...},
    "products": {...},
    "questionnaire": {...},
    "assessmentFactors": [{...}, ..., {...}],
    "data": {...},
    "links": [{...}, ..., {...}]
}

Properties

timestamp (String)
The timestamp at which the data were provided (i.e. added to the risk profile). The timestamp should be specified in ISO 8601 format, using a calendar date part, a time part (specified in ‘extended’ time format) and an optional timezone designator part (specified as an offset from UTC).

Note: The format of the timestamp will NOT be checked by RAS-REST. Nevertheless, clients should use the following following pattern:

  • yyyy '-' MM '-' dd 'T' HH ':' mm ':' ss (('+' | '-') HH ':' mm | 'Z')?

Clients are not required to set a timestamp - but are encouraged to do so, since this will enhance the traceability about answers given to questions in a risk profile’s questionnaire. In case clients specify a timestamp without timezone designator part, it is recommended to specify the timestamp in UTC time zone, since this is consistent with timestamps produced by RAS-REST (see also general notes about time zones).

meta (ObjectMetaData)
The optional meta data applied to the provided data block. Note, that the meta data namespace “ras” is reserved by RAS-REST. Its content must not be modified by clients, however clients are allowed to read the content.
source (ObjectSource)
Information specifying the source that provided the data-block, as well as additional meta data related to the source. The value is provided by the client to keep track of the data’s origin. This data will be transferred into the answerModification of the question if the question is answered.

This is a mandatory property that must be specified by clients.

products (ObjectProvidedProducts)
The data for products (and riders) to be automatically selected for risk assessment, optionally also containing values for the attributes of those products.

This is an optional property that may be specified by any client.

For more details about how this data are structured and processed also see:

  • Media type provided products
  • Usage scenario “Pre-selecting products”
questionnaire (ObjectQuestionAnswerMapping)
The data that are to be used to automatically answer questions of a risk profile’s questionnaire when performing a risk assessment.

This is an optional property that may be specified by any client.

For more details about how this data are processed see:

  • Media type provided questionnaire
  • Usage scenario “Automatically answer general questions”
  • Usage scenario “Automatically answer AFS questions”
assessmentFactors (ArrayProvidedAssessmentFactor)
The data that are to be used to automatically add assessment factors to a risk profile when performing a risk assessment.

This is an optional property that may be specified by any client. Nevertheless, clients are strongly advised to NOT use this mechanism for adding assessment factors to a risk profile. Instead, the mechanism for automatically answering questions (see property questionnaire) should be used, thereby specifying answers for questions of type assessment-factor-search.

For more details about how this data are structured and processed also see:

  • Media type provided assessment factor
  • Usage scenario “Adding assessment factors”
data (Object)
Uninterpreted (aka “raw”) data of any type. This data will not be inspected or processed by RAS-REST. The only restriction imposed by RAS-REST is, that the data is a valid JSON object.

This is an optional property that may be specified by any client.

links (ArrayLink)
Optional hypermedia links that may be specified by clients in order to by used in certain integration scenarios. The links will not be inspected or processed by RAS-REST. They must conform to the structure described for the media-type “hypermedia link”.

This property is NOT intended to be used by “regular” clients. There are rare integration scenarios where a contract between two clients is made up based upon provided links. This is, for example, the case for the ATT (Automated Test Tool), instructing the CEP integration-service which knowledge base version is to be used for the risk assessment.
