Provided products

Products (and riders) contained in a block of provided data, that should be automatically selected for risk assessment, along with optional attributes to be added to the risk profile.

Example (simplified JSON)

{
    "meta": {...},
    "LifeProduct": {
        "attributes": {
            "InsuranceSum": 100000,
            "PolicyDuration": 10
        },
        "riders": {
            "CICRider": {},
            "ADBRider": {
                "attributes": {
                    "InsuranceSum": 200000,
                    "PolicyDuration": 20
                }
            }
        }
    },
    "TPDownProduct": {
        "riders": {
            "TPDownRider": {}
        }
    }
}

Properties

meta (ObjectMetaData)
The meta data applied to the provided products.

This is an optional property that may be specified by any client. The meta data will not be inspected or processed by RAS-REST.

product-ID[0..*] ObjectProvidedProductData

A provided products object may contain an arbitrary number of additional properties. The name of each of these properties must be the ID of a product defined in the knowledge base, that should be selected for risk assessment. The value must be either an empty object, or an object of additional product properties as described below (see “Provided product data”).

Provided product data

Properties

meta (ObjectMetaData)
The meta data applied to the provided product data.

This is an optional property that may be specified by any client. The meta data will not be inspected or processed by RAS-REST.

attributes (Object)
The optional attributes for the provided product.

Note: Product attributes are currently not processed. They may be specified, but this will have no effect.

The attributes must be specified as an object where each property name must correspond to the name of one of the product’s attribute as defined in the knowledge base. The value of a property depends on the type of the product’s attribute. As a rule of thumb you can assume that the same data types as for providing answers for questions can be used.

riders (ObjectProvidedRiders)
The riders that should be automatically selected for the product. The provided riders object has basically the same structure as the provided products object, with the following exceptions:
  • Each name of the object’s properties (apart from the meta property) must be the ID of a rider defined for the product in the knowledge base.
  • The value of such a property must either by an empty object, or an object of additional rider properties as described below (see “Provided rider data”).

This is an optional property that may be specified by any client.

Provided rider data

Properties

meta (ObjectMetaData)
The meta data applied to the provided rider data.

This is an optional property that may be specified by any client. The meta data will not be inspected or processed by RAS-REST.

attributes (Object)
The optional attributes for the provided rider.

Note: Rider attributes are currently not processed. They may be specified, but this will have no effect.

The attributes must be specified as an object where each property name must correspond to the name of one of the riders attribute as defined in the knowledge base. The value of a property depends on the type of the rider’s attribute. As a rule of thumb you can assume that the same data types as for providing answers for questions can be used.

Automatic processing

  • Automatic processing of provided products occurs at most once.
  • Automatic processing of provided products will not be performed if there is already a product selected in the risk profile for risk assessment.
  • Re-processing of provided products may be accomplished by removing all selected products from the risk profile and performing a risk assessment again.
  • If IDs of products or riders are provided that are not defined in the knowledge base, then performing a risk assessment will result in a “Bad Request” (HTP status 400).
  • Attributes of provided products and riders will be added as answered questions to the risk profile. Thereby, the same mechanism as for automatically answering questions is used.
