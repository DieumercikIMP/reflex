Provided questionnaire

Answers for questions contained in a block of provided data that should be automatically processed and thereby added to a risk assessment data’s questionnaire.

A provided questionnaire is specified using a question answer mapping. See documentation of that mapping for examples and further descriptions.

Example (JSON)

{
    "InsuredPerson.Name": {"answer": "John Doe"},
    "LifeProduct.InsuranceSum": {"answer": 500000},
    "ApplicationDate": {"answer": "2015-01-01"},
    "InsuredPerson.Gender": {"answer": "Male"},
    "Smoker.TypesOfSmoke": {"answer": ["Cigarettes", "Pipes"]},
    "InsuredPerson.PreDiseases": {"answer": ["HeartAttack", "Diabetes"]},
    "InsuredPerson.Address": {
        "answer": {
            "StreetNameAndNumber": {
                "StreetName": "Rotherstr.",
                "StreetNumber": "19"
            },
            "ZipCodeAndCity": {
                "ZipCode": 10243,
                "CityName": "Berlin"
            }
        }
    },
    "HighBloodPressure.BlooadPressureReadings": {
        "answer": [
            {"ReadingSystolic": 120, "ReadingDiastolic": 80, "ReadingDate": "2015-01-01"},
            {"ReadingSystolic": 130, "ReadingDiastolic": 85, "ReadingDate": "2015-01-02"}
        ]
    }
}

Automatic processing

  • Processing of provided questionnaires is done every time a risk assessment is performed.
  • The question answers must fit to the expected answer type. Otherwise the question is not answered automatically. Answers that are considered being erroneous are ignored and a processing information disclosing an error message is attached to the answer data. See description of answer value formats for the different question types about when an answer is considered being erroneous.
  • Reflexive questions are automatically answered.
  • If more than one answer for the same question is provided, the first match with correct type is used.
  • If a question is already (partially) answered, the answer will not be updated - as long as no overwrite mode has been specified that forces the answer to be overwritten.
  • If a client removes an answer from a question in the risk assessment’s questionnaire, then RAS REST tries to answer the question again with a provided answer. If however the question still has answer modification information, then the question will not be answered again - as long as no overwrite mode has been specified that forces the answer to be overwritten.
