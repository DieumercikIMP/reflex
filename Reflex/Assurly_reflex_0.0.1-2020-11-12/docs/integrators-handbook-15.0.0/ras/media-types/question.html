Question

A questionnaire element for which an answer may be given. There are different types of questions, with different semantics and structure. Nevertheless, all questions share a common set of properties.

Example (simplified JSON)

{
    "type": "...",
    "id": "foo",
    "scope": "GLOBAL",
    "isNew": true,
    "caption": "bar",
    "index": "1.1",
    "meta": {...},
    "tags": { "tag1":["value1", ..., "valueN"], ..., "tagN":[]},
    "questions": [{...}, ..., {...}],
    
    "answerState": "missing",
    "hasReflexive": true,
    "reflexiveOn": ["...", ..., "..."],
    "constraints": {
        "required": true
    },
    "hint": "hint text",
    "subText": "sub text",
    "infoText": "info text",
    "declarativeStatements": ["statement text 1", ..., "statement text N"]
    "messages": [
        {"message": "error message", "type": "error", "isNew": true},
        {"message": "warning message", "type": "warning", "isNew": true},
        {"message": "info message", "type": "info", "isNew": true}
    ],
    "answerModification": {
        "source": "Demo:CustomerPortal",
        "timestamp": "2015-07-29T10:40:53"
    },
    "assignedTo": "Applicant",
    "initialAnswerByProductChange": "false",
}

Properties

Note:

  • Described here are only the common question properties. For a description of general questionnaire element properties, see description of media type questionnaire element.
  • The type property of a question may have one of the following values:
    • text
    • number
    • date
    • singleselection
    • multiselection
    • assessment-factor-search
    • struct
    • list
  • For each question, a so-called “minimised information set” is defined.
answerState (String)
The answering state of the question, specifying whether an answer was given - and if it was given whether it is valid or invalid. Possible states and transition between states apply according to the state model shown in the following diagram.

The states have the following meaning:

  • absent: No answer has been given for the question and the question does not require an answer.
  • missing: No answer has been given for the question but the question requires an answer.
  • valid: An answer has been given for the question and the answer is complete and correct (according to knowledge base validations).
  • invalid: An answer has been given for the question but the answer is not correct (according to knowledge base validations).
  • incomplete: An answer has been given for the question consisting of different parts (e.g. a structure), but there is at least one part that is required to be answered which has no answer.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present, except for the prototype of list questions (and any further question possibly contained in the prototype).

hasReflexive (Boolean)
A flag, indicating if answering the question will potentially trigger further reflexive questions to be asked. For further information about reflexive questions, see the description in the “Reflexive questions” section of the questionnaire media type .

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

reflexiveOn (ArrayString)
The optional information about which answers given to the question will trigger further reflexive questions to be asked. The content of this property strongly depends on the type of the question. Please find further information in the documentation of the particular question types.

Note, that this property may only be available if the hasReflexive property is set to true. Also, if this property is absent, it does not mean that no answer will trigger reflexive questions. It means that there is no detailed information available on which answer will trigger reflexive questions.

This property is set by ReFlex/REST only and must not be modified by clients.

constraints (Object)
The constraints defined for the question. The constraints for all types of questions specify at least the required property. Certain types of questions may provide additional constraints, specifying restrictions on how the question may be answered.

* required (Boolean)
A flag, indicating if the question is required to be answered.
This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

hint (String)
The optional hint text that may be displayed for the question. A hint text typically gives advice to users about how to answer the question.

This property is set by ReFlex/REST only and must not be modified by clients.

subText (String)
The optional sub text that may be displayed for the question. A sub text gives additional advice to users about how to answer the question.

This property is set by ReFlex/REST only and must not be modified by clients.

infoText (String)
The optional info text that may be displayed for the question. An info text may be used, for example, to describe the expected value of a question’s answer.

This property is set by ReFlex/REST only and must not be modified by clients.

declarativeStatements (ArrayString)
The optional declarative statements for the question. A declarative statement is a descriptive text (typically formulated as a sentence in active form) of an answer given to the question.

For example, a question “Do you smoke?” that has been answered with “Yes” may have a declarative statement “I do smoke”. Or a question “What do you smoke?” that has been answered with “Cigarettes” and “Cigars” may have two declarative statements “I smoke cigarettes” and “I smoke cigars”.

Note, that declarative statements will only be present if the question has been answered and the answer is valid (i.e. the answerState is valid).

This property is set by ReFlex/REST only and must not be modified by clients.

messages (ArrayValidationMessage)
The optional validation messages for the question. Validation messages will be present, if the answer given to the question deviates in some kind of the expected answer as specified by validations or constraints formulated in the knowledge base. Please note that the answer property of list questions and the elements property of structure questions also refer to media type Question and can contain validation messages as well. The validation messages attached to list elements and structure members are created by validations or constraints formulated for lists and structures in the knowledge base.

This property is set by ReFlex/REST only and must not be modified by clients.

answerModification (ObjectAnswerModification)
The optional information about the modification of the answer given to the question. It basically describes who modified the answer and when the modification happened.

Even though this is an optional property, clients should always set this property every time the answer for a question is changed by them.

assignedTo (String)
The role to which the question is assigned to, indicating who is intended to answer this question.

This information may be used by clients to decided whether the question should be displayed or not.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

initialAnswerByProductChange (boolean)
A flag, indicating if the answer for the question has been initially set by performing a product change request. This property is set by ReFlex/REST only and must not be modified by clients. The value of this flag will not change when performing a risk assessment even when the answer to a question is removed.

Minimised information set

The minimised information set defines the minimal amount of information for a question that is required for RAS/REST being able to pass it to the RAS/Runtime without losing any information. It’s purpose is to have a representation of questions that is sufficient to be processed in some use-cases (e.g. when performing a context-aware assessment factor search), thereby minimising the amount of data being transferred to RAS/REST. In these uses-cases, clients will typically build up questions or complete questionnaires by themselves. The minimised information set is not intended to carry information being presented to an end-user.

Note that a question’s minimised information set may differ, depending on the type of the question. Please find the specifications in the “Minimised Information Set” section within the description of the following question types:

  • Text question
  • Number question
  • Date question
  • Single-selection question
  • Multi-selection question
  • Assessment factor search question
  • Structure question
  • List question
