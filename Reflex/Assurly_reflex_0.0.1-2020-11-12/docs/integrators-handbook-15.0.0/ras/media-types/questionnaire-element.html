Questionnaire element

An element contained in a questionnaire. There are different types of questionnaire elements, with different semantics and structure. Nevertheless, all questionnaire elements share a common set of properties.

Example (simplified JSON)

{
    "type": "...",
    "id": "foo",
    "scope": "GLOBAL",
    "isNew": true,
    "caption": "bar",
    "index": "1.1",
    "meta": {...},
    "tags": { "tag1":["value1", ..., "valueN"], ..., "tagN":[]}
    "questions": [{...}, ..., {...}]
}

Properties

type (String)
The specifier indicating of what type the element is. A type specifier may have one of the following values:
  • group
  • assessment-factor-group
  • label
  • text
  • number
  • date
  • singleselection
  • multiselection
  • assessment-factor-search
  • struct
  • list

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

id (String)
The globally unique ID of the element. This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.
scope (String)
The logical scope that the element is defined in. The scope is a string that conforms to the following pattern:

<visibility> [ ":" <reference-id> ]

Thereby,

  • visibility specifies the scope’s visibility, which may be one of GLOBAL, INSURANCEPRODUCT, or ASSESSMENTFACTOR. GLOBAL means that the element is present globally at most one time. In this case, the scope has no reference-id. INSURANCEPRODUCT means that the element is present per product at most one time. In this case, the scope also includes a reference-id. ASSESSMENTFACTOR means that the element is present per assessment factor at most one time. In this case, the scope also inlcudes a reference-id.
  • reference-id is optional ID of the object in which context the element is relevant. In case the visibility is INSURANCEPRODUCT, the reference-id is the ID of a product. In case the visibility is ASSESSMENTFACTOR, the reference-id is the ID of a assessment factor.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

isNew (boolean)
A flag indicating if the element has been newly added to a questionnaire, compared to the preceding assessment. This flag is only present on elements that have a meaning on their own. For example, a member of a structure question or the prototype of a list question will not have this flag.

This property is set by ReFlex/REST only and must not be modified by clients.

caption (String)
The text to be displayed for the element. This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.
index (String)
The optional index of the element within its containing questionnaire. An index specifies hierarchical information, as well as positional information for a questionnaire element, e.g. “1.2.5”. The index is intended to specify some kind of reference number for a questionnaire element, that may be displayed on a questionnaire. Note, that not all questionnaire elements have an index.

This property is set by ReFlex/REST only and must not be modified by clients.

meta (ObjectMetaData)
The optional meta data applied to the element.
tags (Object)

The optional tags that have been applied to the element. A tag is basically a key-value pair where the key is a string denoting a tag’s name and the value is a (possibly empty) list of strings.

This property is set by ReFlex/REST only and must not be modified by clients.

questions (ArrayQuestionnaireElement)
The optional child elements contained in the element.
