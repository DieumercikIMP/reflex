Questionnaire

A questionnaire contains questions to be asked and answered during a risk assessment, as well as elements used for structuring the questions.

Example (simplified JSON)

    {
        "meta": {...},
        "questions": [{...}, ..., {...}],
        "hasNewData": true
    }

Properties

meta (ObjectMetaData)

The optional meta data applied to the questionnaire.

questions (ArrayQuestionnaireElement)

The elements contained in the questionnaire. These elements are provided by ReFlex/REST only. Clients must not add or remove elements. Nevertheless, clients may add, remove or modify the answers of questions contained in a questionnaire.

hasNewData (boolean)

A flag indicating if new “relevant” data has been added to this questionnaire, compared to the preceding assessment. The following data is considered to be relevant for this flag:

  • All elements contained in the questionnaire.
  • All validation messages attached to questions contained in the questionnaire.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

Hierarchical structure

A questionnaire is hierarchically structured. It has an arbitrary number of child elements (so called questionnaire elements), each of which may in turn have child elements, also. Thus, a questionnaire forms a tree of elements, where the questionnaire itself is the root of the tree. With this hierarchical structure of questionnaire elements, two aspects are expressed: structural containment (i.e. grouping of elements) and logical dependencies between questions.

Groups

Grouping of questionnaire elements means, that there are elements that logically belong together. These related elements are then provided as child elements of a grouping element. A simplified example to demonstrate this:

"questionnaire": {
    "questions": [
        {
            "type": "group",
            "id": "LifeProduct.PersonalDetails",
            "caption": "Personal details",
            ...
            "questions": [
                {
                    "type": "date",
                    "id": "InsuredPerson.DateOfBirth",
                    "caption": "Date of birth",
                    ...
                },
                {
                    "type": "singleselection",
                    "id": "InsuredPerson.Gender",
                    "caption": "Gender",
                    ...
                }
            ]
        }
    ]
}

Here we have a grouping questionnaire element PersonalDetails. Within this group there are two questions DateOfBirth and Gender. These questions are contained as child elements in the questions array.

Reflexive questions

Logical dependencies between questionnaire elements means that an element is present in a questionnaire only if a higher-level question has been given a certain answer. Thus, the lower-level question depends on the answer given to the higher-level question. The dependant question is also called reflexive question. A simplified example to demonstrate this:

"questionnaire": {
    "questions": [
        {
            "type": "singleselection",
            "id": "WasTreatedInHospital",
            "caption": "Have you been treated in hospital?",
            "answer": {"id": "Yes", "label": "Yes"},
            ...
            "questions": [
                {
                    "type": "assessment-factor-search",
                    "id": "TreatedInHospitalDisease",
                    "caption": "Please specify the disease ...",
                    ...
                }
            ]
        }
    ]
}

Here the question TreatedInHospitalDisease is a reflexive question, which is contained as a child question of the question WasTreatedInHospital. It is present in the questionnaire only if WasTreatedInHospital is answered with “Yes”.

Traversal order

The elements contained in a questionnaire are ordered. To process the elements in the correct order, clients are strongly advised to traverse the questionnaire tree in a “depth-first” fashion. Given the following simplified example:

"questionnaire": {
    "questions": [{
        "id": "InsuredPerson.DateOfBirth", ...
    },
    {
        "id": "PersonalDetails",
        "questions": [{
            "id": "InsuredPerson.IsSmoker", ...
            "questions": [{
                "id": "InsuredPerson.IsSmoker:Smoker", ...
                "questions": [{
                    "id": "Smoker.SmokeType", ...
                },
                {
                    "id": "Smoker.NumberPerDay", ...
                }]
            }],
        }],
    {
        "id": "Medical", ...
        "questions": [{
            "id": "RespiratoryDiseases", ...
            "questions": [{
                "id": "RespiratoryDisease:Asthma", ...
                "questions": [{
                    "id": "Asthma.HasComplaintsOrTreatment", ...
                },
                {
                    "id": "Asthma.HasLongerHospitalStayInLast3Years", ...
                }]
            }]
        },
        {
            "id": "DiabetesDiseases", ...
        }]
    }]
}
}

The correct traversal order would then be:

InsuredPerson.DateOfBirth → PersonalDetails → InsuredPerson.IsSmoker → InsuredPerson.IsSmoker:Smoker → Smoker.SmokeType → Smoker.NumberPerDay → Medical → RespiratoryDiseases → RespiratoryDisease:Asthma → Asthma.HasComplaintsOrTreatment → Asthma.HasLongerHospitalStayInLast3Years → DiabetesDiseases

Minimised information set

The minimised information set for a questionnaire is actually a simple object, optionally consisting of the questions property where for each question the corresponding minimised information set is used. The questions contained in the questionnaire do not necessarily need to be hierarchically structured. Instead, all questions may be contained on the same level within the array.

See also the description of the minimised information set for the following question types:

  • Text question
  • Number question
  • Date question
  • Single-selection question
  • Multi-selection question
  • Assessment factor search question
  • Structure question
  • List question
