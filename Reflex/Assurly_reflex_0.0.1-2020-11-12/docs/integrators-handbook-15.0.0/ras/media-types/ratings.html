Ratings

A ratings object contains all ratings that are available for a specific part of an assessment result. For example, it may contain all ratings of an insurance rider (i.e. after the FlexRules have been executed), all base ratings of an insurance cover (i.e. before the FlexRules have been executed), all ratings that have been added during the execution of a specific FlexRule for an insurance product, etc.

The actual ratings contained in a ratings object are sorted according to their assigned assessment category (see properties, below). Thereby, the category named “Global” has a special meaning. It simply acts as a container for ratings that cannot be uniquely assigned to a assessment category (either because a rating is not assigned to any category at all, or it is assigned to multiple different categories). There is no notion of an aggregated overall (“roll-up”) rating - at least not in general. Actually, the number of ratings, and how they may be aggregated, strongly depends on the design of a knowledge base, as well as the defined FlexRules. So, the “Global” category must not be confused with something that is referencing an aggregated overall result.

Example (simplified JSON)

        "caption" : "medical (en)",
        "ratings" : [

                "caption": "Offer",

                    {
                        "id": "EMLoading",
                        "caption": "EM Loading",
                        "value": 200
                    },
                    {
                        "id": "FlatExtraDuration",
                        "caption": "Flat Extra Duration",
                        "value": 3
                    },
                    {
                        "id": "FlatExtraLoading",
                        "caption": "Flat Extra Loading",
                        "value": 1
                    }
                ],
                "reason": {
                    "assessmentFactors": [
                        {
                            "id": "OverWeight",
                            "label": "OverWeight (en)"
                        }
                    ]
                }
            },
            {
                "id": "Offer",
                "caption": "Offer",
                "attributes": [
                    {
                        "id": "EMLoading",
                        "caption": "EM Loading",
                        "value": 100
                    },
                    {
                        "id": "FlatExtraDuration",
                        "caption": "Flat Extra Duration",
                        "value": 3
                    },
                    {
                        "id": "FlatExtraLoading",
                        "caption": "Flat Extra Loading",

                    }
                ],
                "reason": {
                    "assessmentFactors": [
                        {
                            "id": "NicotineDependence",
                            "label": "NicotineDependence (en)"
                        }
                    ]
                }

        ]
    },

Properties

category-ID[0..∗] (ArrayAssessment category)
A ratings object may contain an arbitrary number of assessment categories. Names of these categories are used as keys in the map, and the values are assessment category objects containing non-empty array of ratings.

If ratings are available that are not assigned to a assessment category or that are assigned to multiple categories, an additional key/value pair with the special key Global is used to collect all of these ratings. Note, that this category does not specify some kind of aggregated overall rating. It simply acts as a container for ratings that can not be uniquely assigned to a assessment category.

See below for more detailed information regarding a single rating.

Assessment category

An assessment category is a named and translated container for a list of rating objects.

Properties

id (String)
The unique id (name) of the assessment category

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

caption (String)
The translation of the assessment category.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

ratings (ArrayRating)
A non-empty array of ratings that are available for this assessment category.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

Rating

A rating regarding some risk identified during a risk assessment. Apart from the id and the caption of the actual decision imposed by this rating, a rating may contain attributes regarding the rating as well as an optional reason that provides information about why this rating was created.

Properties

id (String)
The id of the kind of this rating.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

caption (String)
The translation of the kind this rating.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

attributes (ArrayAttribute)
The optional attributes of the rating. It strongly depends on the rating’s id if and which attributes are supported here.

This property is set by ReFlex/REST only and must not be modified by clients.

reason (ObjectReason)
The optional reason for a rating provides additional information about why this rating was created.

This property is set by ReFlex/REST only and must not be modified by clients.

DSL mapping

The rating representing an Offer shown in the example above maps to the following definition of a result type in ReFlex/DSL.

ResultType Offer {
    attributes EMLoading, optional FlatExtraDuration, optional FlatExtraLoading
}

Attribute

An attribute assigned to a rating during a risk assessment.

Properties

id (String)
The id of the attribute corresponds to the ID of a result attribute defined in the knowledge base that is supported by the corresponding rating.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

caption (String)
The human readable name of this attribute.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

value (String | Number | Boolean)
The actual value of this attribute. The data type of this property depends on the type of the result attribute. See: Supported types of result attributes

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

Supported types of result attributes

We currently support the five types of result attributes. The type of an attribute in the ReFlex/DSL decides which type the value property will have in the JSON response.

Type in Reflex/DSL Type of value JSON
Boolean Boolean
Number Number
String String
Key String
Date String

This property is set by ReFlex/REST only and must not be modified by clients.

Date attribute

Example (JSON)

{
    "id": "DateAttribute",
    "caption": "Caption of DateAttribute",
    "value": "2016-01-01"
}

Properties

value (String)
The value of the date attribute with the given ID.

The date is specified as a string in ISO 8601 format according to the following patterns (depending upon the precision of the date attribute in the knowledge base):

  • year: yyyy (example: 2015)
  • month: yyyy '-' MM (example: 2015-01)
  • day: yyyy '-' MM '-' dd (example: 2015-01-30)
  • hour: yyyy '-' MM '-' dd 'T' HH (('+' | '-') HH ':' mm | 'Z')? (examples: 2015-01-30T12, 2015-01-30T12+01:00)
  • minute: yyyy '-' MM '-' dd 'T' HH ':' mm (('+' | '-') HH ':' mm | 'Z')? (examples: 2015-01-30T12:05, 2015-01-30T12:05+01:00)

Thereby,

  • yyyy is the year of the date (4 digits).
  • MM is the numerical month of within the year (2 digits, ranging from 01 to 12).
  • dd is the numerical day within the month (2 digits, ranging from 01 to 31).
  • HH is the hour within a day (2 digits, ranging from 00 to 23).
  • mm is the minute within an hour (2 digits, ranging from 00 to 59).

This property is set by ReFlex/REST only and must not be modified by clients.

Note, that in case no explicit time zone offset is specified for a date attribute, the date is to be interpreted as being in UTC time zone (see also general notes about time zones).

DSL mapping

In the example shown above, the attribute EMLoading maps to the following definition of a result attribute in ReFlex/DSL.

ResultAttribute EMLoading {
    type Integer
    initialValue 0
}

Reason

The reason for a rating contains information about why a rating was created by providing a list of assessment factors that resulted in the generation of the rating.

Properties

assessmentFactors (ArrayAssessmentFactorIdsAndLabels)
A list containing the IDs and labels (translations) of the assessment factors that caused a rating to be created.

This property is set by ReFlex/REST only and must not be modified by clients. Clients can expect this property to be always present.

AssessmentFactorIdsAndLabels

An element containing the ID and label of an assessment factor.

Example (JSON)

{
    "id": "OverWeight",
    "label": "OverWeight (en)"
}
