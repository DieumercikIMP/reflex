Risk assessment

A risk profile that contains the current state of a risk assessment.

Example (simplified JSON)

{
        "language": "en",
        "completed": false,
        "assignedToCompleted": {...},
        "valid": false,
        "conversationId": "9ef0c41b-816f-4f33-9411-0b2eb4bd4520"
        "meta": {...},
        "links": [{...}, ..., {...}],
        "products": [...],
        "questionnaire": {...},
        "assessmentFactors": [{...}, ..., {...}],
        "results": {...},
        "provided": [{...}, ..., {...}],
        "calculated": [{...}, ..., {...}]
    }

Properties

language (String)
The language used for performing the risk assessment. All data in the risk profile that have translations will be translated in this language. The language is a required property that must be specified by clients. The language is a string that must conform to the following pattern:

<language-code> [ "-" <country-code> [ "-" <dialect> ] ]

Thereby,

  • language-code is the required language code of the language, according to the two-letter ISO 639-1 format (lower case).
  • country-code is the optional country code of the language, according to the two-letter ISO 3166-1 ALPHA-2 format (upper case). The country code may only be specified if a language code has also been specified.
  • dialect is the optional dialect specifier of the language. The dialect may only be specified if a country code has also been specified.

Examples (valid):

  • "en"
  • "en-GB"
  • "en-GB-cockney"

Examples (invalid):

  • "GB"
  • "en-cockney"
completed (Boolean)
A flag, indicating if the risk assessment is completed. A risk assessment is considered to be completed if all required questions have been validly answered and there are no validation errors for products (i.e. the valid flag is true).

Note:
Even when the completed flag is set to true, clients may continue answer questions or modify answers given to questions. This may then cause the flag to switch from true to false.

This flag is set by ReFlex/REST only and must not be modified by clients. Clients can expect this flag to be always present.

assignedToCompleted (Map<String, Boolean>)
A map, showing for each assignedTo tag value the overall completion status of questions tagged for the corresponding value. Each key of the map corresponds to a value of an assignedTo tag. Each value is either true or false. For the computation of the value the same method is used as for the completed flag.

Example: “assignedToCompleted”: { “Applicant”: true, “3PS”: false }

valid (Boolean)
A flag, indicating if the risk profile is valid. The risk profile is considered to be valid if there is no product or answered question with a validation error.

This flag is set by ReFlex/REST only and must not be modified by clients. Clients can expect this flag to be always present.

conversationId (String)
A unique identifier of the conversation between RAS REST and the client.

The conversationId can be used by clients to track the state of a risk assessment along subsequent requests. The conversationId will be generated by RAS REST if it is not present in the risk assessment sent with the request. The conversationId is guaranteed to be stable in subsequent request/response cycles. Stable means that once the conversationId is set, RAS REST will never change the value of this attribute. The value of this attribute should not be modified by the client.

meta (ObjectMetaData)
The meta data applied to the risk profile.

Meta data in the "ras" namespace always contain a "knowledgebase" object, having the following properties:

  • id: The ID of the knowledge base for which a risk assessment has been performed.
  • version: The version of the knowledge base.
  • cedent: The cedent ID of the knowledge base.

In addition, the meta data in the "ras" namespace always contains a property "version":

  • version: The version of ReFlex/REST that has performed the risk assessment.
links (ArrayLink)
The optional hypermedia links provided by the REST resource that processed / provided the risk profile.
products (ArrayProduct)
The selected products for which a risk assessment is to be performed.
questionnaire (ObjectQuestionnaire)
The questionnaire, containing questions to be asked and answered during a risk assessment, as well as elements used for structuring the questions.
assessmentFactors (ArrayAssessmentFactor)
The optional assessment factors that were identified during a risk assessment.
results (ObjectAssessmentResult)
The results of the risk assessment.
provided (ArrayProvidedData)
Data provided by external sources that is relevant for a risk assessment and should be automatically processed.

The provided data is used to fill in necessary product information, add assessment factors, and answer unanswered questions in the risk profile’s questionnaire if possible. By this means it is e.g. possible for clients to provide information such as birth date, smoker attributes, weight, and other product information before RAS REST shows the corresponding questions.

calculated (ArrayCalculatedElement)
Data that has been automatically calculated during a risk assessment. This is an optional property that will not be present if no data has been calculated. Note, that it may only be present if the risk profile is valid.

The elements contained in the calculated data represent values that are calculated based on elements defined in a knowledge base. Typically, the values will be calculated from answers given to questions contained in the risk assessment’s questionnaire.
