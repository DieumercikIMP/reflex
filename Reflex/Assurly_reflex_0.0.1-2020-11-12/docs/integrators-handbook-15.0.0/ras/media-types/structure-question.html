Structure question

A question that consists of a structure of nested questions.

The type specifier for a structure question is struct. For information about type specifiers see description of media type questionnaire element.

Example (simplified JSON)

{
    "type": "struct",
    "id": "InsuredPerson.Address",
    "scope": "ASSESSMENTFACTOR:InsuredPerson",
    "isNew": true,
    "caption": "Address",
    "constraints": {
        "required": true
    },
    "meta": {...},
    "tags": {...},
    "hasReflexive": false,
    "answerState": "valid",
    "index": "1",
    "assignedTo": "Applicant",
    
    "elements": [{
        "type": "text",
        "id": "InsuredPerson.Street",
        ...,
        "answer": "Rotherstr. 19"
    },
    {
        "type": "text",
        "id": "InsuredPerson.City",
        ...,
        "answer": "Berlin"
    },
    {
        "type": "singleselection",
        "id": "InsuredPerson.Country",
        ...,
        "options": [
            { "id": "DEU", "label": "Germany" },
            { "id": "AUT", "label": "Austria" },
            { "id": "CHE", "label": "Switzerland" }
        ],
        "answer": { "id": "DEU", "label": "Germany" }
    }]
}

Properties

Note: Described here are only those properties that are directly related to a structure question. For a description of general question properties, see the description of media type question.

Note: A structure question cannot be directly answered, it therefore does not have an answer property. Instead, in order to answer a structure questions, its nested questions provided by the elements property (see below) must be answered.

elements (ArrayQuestion)
The nested member questions that this structure contains.

Nested member questions can be of any question type, esp. a nested question can in turn be a structure. Therefore, structures can - at least theoretically - be nested to any depth. This hierarchical structure must not be confused with the structure imposed by the questions property that all questionnaire elements may provide. The questions property expresses logical dependencies between reflexive questions, whereas the elements property expresses structural containment for a single structure.

All nested member questions will have the same scope as the structure in which they are contained. The index of all member questions will always be "0" (this may be changed in later releases). Further, the member questions do not have an isNew attribute.

This property is set by ReFlex/REST only. Clients must not modify the structure of nested questions. Nevertheless, clients are allowed to add, remove, or modify the answers given to the nested questions.

DSL mapping

The question shown in the example above maps to the following definition of an assessment variable in ReFlex/DSL if required in the context of the assessment factor InsuredPerson.

AssessmentVariable Address {
    type AddressStructure
    global
}

Thereby, the type AddressStructure is defined as follows:

Structure AddressStructure {
    AssessmentVariable Street {
        type String
    }
    AssessmentVariable City {
        type String
    }
    AssessmentVariable Country {
        type CountrySelection
        singleSelection
    }
}

Where the CountrySelection type is defined as follows:

Selection CountrySelection {
    DEU, AUT, CHE
}

Minimised information set

The minimised information set for a structure question consists of the following properties:

  • type
  • id
  • scope
  • elements Note: For each nested question the corresponding minimized information set may be used.

Example (JSON)

{
    "type": "struct",
    "id": "InsuredPerson.Address",
    "scope": "ASSESSMENTFACTOR:InsuredPerson",
    "elements": [{
        "type": "text",
        "id": "InsuredPerson.Street",
        "scope": "ASSESSMENTFACTOR:InsuredPerson",
        "answer": "Rotherstr. 19"
    },
    {
        "type": "text",
        "id": "InsuredPerson.City",
        "scope": "ASSESSMENTFACTOR:InsuredPerson",
        "answer": "Berlin"
    },
    {
        "type": "singleselection",
        "id": "InsuredPerson.Country",
        "scope": "ASSESSMENTFACTOR:InsuredPerson",
        "options": [
            { "id": "DEU" },
            { "id": "AUT" },
            { "id": "CHE" }
        ],
        "answer": { "id": "DEU" }
    }]
}
