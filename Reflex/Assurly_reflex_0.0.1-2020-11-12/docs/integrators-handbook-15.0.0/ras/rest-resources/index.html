REST API

The RAS-REST-API provides the following resource endpoints:

/rs/ GET
/rs/knowledgebases GET POST
/rs/knowledgebases/{id}/versions/{version} GET DELETE
/rs/knowledgebases/{id}/versions/{version}/riskassessments POST
/rs/knowledgebases/{id}/versions/{version}/translations POST
/rs/knowledgebases/{id}/versions/{version}/productchange POST
/rs/knowledgebases/{id}/versions/{version}/products GET
/rs/knowledgebases/{id}/versions/{version}/assessmentfactors GET

All resources require that user credentials are set via HTTP basic authentication in the REST client, otherwise resource will respond with 401.

Common error responses codes

Code Name Description Content-Type
401 Unauthorized The request to the specific resource is due to authentication restriction not permitted. text/plain; charset=utf-8
404 Not Found Specified resource not found, URL invalid or does not exist. text/plain; charset=utf-8
500 Internal Server Error Request is not processed due to internal server error. text/plain; charset=utf-8

More detailed and structured status details can be requested by accepting the media type application/problem+json explicitly in the Accept request header, see RCF 7807. The list of supported problem types can be found here.

Note: A resource ignores all query parameter that are not explicitly mentioned in its documentation.
