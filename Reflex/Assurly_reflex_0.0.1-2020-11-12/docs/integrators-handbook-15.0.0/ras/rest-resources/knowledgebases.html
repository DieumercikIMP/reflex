Knowledge bases resource

/rs/knowledgebases

GET

Lists all available (i.e. deployed) knowledge bases.

This resource provides information about the deployed knowledge base and its versions. Each of these versions provides hypermedia links that identify the version’s sub-resources. The sub-resources are necessary to perform risk assessments with products and assessment factors. More information about hypermedia links can be found here.

A client can select the version that she needs in order to perform risk assessments. The latest version is labeled with “latest”.

{
    "knowledgeBases": [ {
        "knowledgeBaseId": "kb-id-01",
        "knowledgeBaseName": "kb-name-01",
        "cedentId": "cedent-id-01",
        "cedentName": "cedent-name-01",
        "versions": {
        "latest": {
            "version": "1.16.0.SNAPSHOT-1",
            "infoText": "Example knowledge base for documentation",
            "creationDate": "2015-06-05T00:40:19+0200",
            "links": [ {
                "href": "/rs/knowledgebases/kb-id-01/versions/
                            1-16-0-snapshot-1",
                "rel": [ "self" ],
                "method": "GET"
                },
               {
                "href": "/rs/knowledgebases/kb-id-01/versions/
                            1-16-0-snapshot-1/riskassessments",
                "rel": [ "process-riskassessments" ],
                "method": "POST"
                },
                {
                    "href": "/rs/knowledgebases/kb-id-01/versions/
                            1-16-0-snapshot-1/products",
                 "rel": [ "search-products" ],
                    "method": "GET"
                },
                {
                    "href": "/rs/knowledgebases/kb-id-01/versions/
                            1-16-0-snapshot-1/assessmentfactors",
                    "rel": [ "search-assessmentfactors" ],
                    "method": "GET"
                } ]
            }
        }
    } ]
}

Responses

Code Name Description Content-Type Media Type
200 OK The request was successfully processed. application/json; charset=utf-8 ObjectKnowledgeBaseList

POST

Performs the deployment of a knowledge base. The deployment is performed by writing the supplied compiled knowledge base file to the file system and register the knowledge base with the RAS/Runtime.

Request

The compiled knowledge base file to be deployed has to be transmitted using a multipart form-data request, containing the following parts.

Part Content-Type Description
file application/octet-stream The compiled knowledge base file to be deployed.

Note: The size for uploaded knowledge base files is limited by the value of com.hannoverre.reflex.ras.maxUploadSize configuration property. Default value is 20MB (20971520 bytes).

Note: There are some known issues regarding the deployment of knowledge bases, see “Multiple knowledge bases”.

Note: The deployment and undeployment of knowledge bases via a REST endpoint is considered to be an internal endpoint, i.e. it is not necessary in an production environment. This endpoint is disabled by default and has to be activated in the configuration.

Note: See usage scenario for more information on how to deploy a knowledge base via the REST endpoint.

Example:

POST http://.../rs/knowledgebases HTTP/1.1
Content-Type: multipart/form-data; boundary="----=_Part_0_784824575.1440421749497"
Content-Length: 2105764

------=_Part_0_784824575.1440421749497
Content-Type: application/octet-stream
Content-Transfer-Encoding: binary
Content-Disposition: form-data; name="file"; filename="knowledgebase.KnowledgeBase-1.17.4.SNAPSHOT-20150807-1302.jar"

(content skipped)

Responses

Code Name Description Content-Type Media Type
201 Created The deployment has been successfully performed. application/json; charset=utf-8 ObjectDeploymentResult
400 Bad Request The data passed to this resource is invalid. This may be because the request is not a multipart request, does not contain a part named “file”, has an invalid content-type, the transfered file is not a valid knowledge base file, this version of the knowledge base is already deployed, or single cedent and single knowledge base mode is activated and another knowledge base is already deployed. text/plain; charset=utf-8
403 Forbidden The compiled knowledge base could not be written, due to a security restriction. text/plain; charset=utf-8

/rs/knowledgebases/{id}/versions/{version}

Path Parameters

id The (URL-encoded) ID of a knowledge base.
version The encoded version of a knowledge base. An encoded version must conform to the following pattern: [0-9]+-[0-9]+-[0-9]+(-(SNAPSHOT-[0-9]+-[0-9]+)|RELEASE)? The keyword ‘latest’ can be used alternatively to select the latest currently deployed version: /rs/knowledgebases/{id}/versions/latest

GET

Retrieves detailed information of a deployed knowledge base.

The information that can be retrieved for all deployed knowledge base versions can also be read for a single knowledge base version. The returned JSON data has the same structure but holds only the data of the version that is identified by the corresponding URI.

This resource can be used to check, whether a particular version is deployed.

Responses

Code Name Description Content-Type Media Type
200 OK The request was successfully processed. application/json; charset=utf-8 ObjectKnowledgeBase
404 Not Found The knowledge base base with the specified ID and version does not exist. text/plain; charset=utf-8

DELETE

Performs the undeployment of a knowledge base. The undeployment is performed by deleting the compiled knowledge base file from the file system and deregister the knowledge base from the RAS/Runtime.

Note: The deployment and undeployment of knowledge bases via a REST endpoint is considered to be an internal endpoint, i.e. it is not necessary in an production environment. This endpoint is disabled by default and has to be activated in the configuration.

Note: See usage scenario for more information on how to undeploy a knowledge base via the REST endpoint.

Responses

Code Name Description Content-Type Media Type
200 OK The undeployment has been successfully performed. application/json; charset=utf-8 ObjectDeploymentResult
400 Bad Request The data (i.e. one of the path parameters) passed to this resource is invalid. This is either because the ID or the version of the knowledge base is syntactically incorrect. text/plain; charset=utf-8
403 Forbidden The compiled knowledge base could not be deleted, due to a security restriction. text/plain; charset=utf-8
404 Not Found The knowledge base base with the specified ID and version does not exist. text/plain; charset=utf-8
