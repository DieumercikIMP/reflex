Deploy a knowledge base

All knowledge bases that already exist in the configured deployment directory on startup will be deployed automatically.
The deployment directory must not be altered during runtime.

Knowledge bases are always expected to be jar-files.

Additional knowledge bases can be deployed via the REST API to the knowledge bases resource with the knowledge base as “multipart/form-data” for the field “file”. No further parameters are required:

POST /rs/knowledgebases

The deploy will happen synchronous, meaning that the response of the call is final.
Due to the synchronous nature, the deploy might need substantial time to complete, depending on the size of the knowledge base.

An essential prerequisite for deploying knowledge bases via REST API is, that the configured run mode is specified as INTERNAL to
enable the corresponding REST endpoint.
Without this configuration change, the corresponding REST endpoint is disabled by default.
Also make sure that the configuration parameter maxUploadSize is set appropriately.

The response contains at its body metadata about the deployed knowledge base ("knowledgeBase": {...}) as well as the links to the deployed knowledge base.

Example: Deploy the version “2.0.0” of knowledge base “standard” for cedent “cedentId”.

{
   "operation" : "deploy",
   "knowledgeBase" : {
      "id" : "standard",
      "cedent" : "cedentId",
      "version" : "2.0.0"
   },
   "links" : [
      {
         "method" : "GET",
         "rel" : [
            "knowledgebase-status"
         ],
         "href" : "/rs/knowledgebases/standard/versions/2-0-0"
      },
      {
         "method" : "GET",
         "rel" : [
            "knowledgebases-list"
         ],
         "href" : "/rs/knowledgebases"
      },
      {
         "method" : "POST",
         "rel" : [
            "knowledgebase-deploy",
            "self"
         ],
         "href" : "/rs/knowledgebases"
      }
   ]
}

The corresponding curl command for the above example is

curl --basic --user <user> --form "file=@<path-to-Knowledge-base.jar>" localhost:8080/<context>/rs/knowledgebases

where <user>, <path-to-Knowledge-base.jar>, and <context> have to be replaced by your actual values.

If the knowledge base was compiled for a different RAS version, then RAS ReST will respond with an “Incompatible knowledge base” error.

To receive this error message in a better parsable JSON form, the POST request must accept application/problem+json media type. The curl command must be extended with -H parameter:

curl -H "Accept: application/problem+json" --basic --user <user> --form "file=@<path-to-Knowledge-base.jar>" localhost:8080/<context>/rs/knowledgebases
