Using the latest knowledge base version

If a client wants to use the latest deployed knowledge base version, the corresponding URIs have to be used. How can clients find out these URIs?

  • Find out URI via API
  • Use keyword ‘latest’ in URI

Find out URI via API

Clients can find out which versions are deployed and which of them is the lasted by calling the knowledgebases resource:

GET /rs/knowledgebases
{
  "knowledgeBases": [ {
    ...
    "versions": {
      "latest": {
        "version": "1.16.2",
        ...
      },
      "1-16-2": {
        "version": "1.16.2",
        ...
      },
      "1-16-1": {
        "version": "1.16.1",
        ...
      },
      "1-16-0": {
        "version": "1.16.0",
        ...
      }
    }
  ]
}

The resource returns a JSON document containing the map versions that provides all available versions and the links to them. The latest version always has the name latest. This map entry is always available if at least one single version is deployed.

Use keyword ‘latest’ in URI

If the client knows about the URI structure, a shortcut can be used to identify the latest version. If for instance a client performs a risk assessments with:

POST /rs/knowledgebases/kb-id/versions/1-16-0/riskassessments

then, the concrete version id could be replaced by the keyword ‘latest’:

POST /rs/knowledgebases/kb-id/versions/latest/riskassessments

This URI-alias always identifies the latest deployed version. The concrete version that is identified by this URI-alias might change over time. The URI-alias is not a unique identifier:

  • If the latest version is undeployed, an older version will step in.
  • If a newer version is deployed, it will automatically become the ‘latest’ version.

Comment: The links that are provided by RAS REST always use URIs with concrete version ids. The keyword ‘latest’ can only be used by clients that know the URI structure.
