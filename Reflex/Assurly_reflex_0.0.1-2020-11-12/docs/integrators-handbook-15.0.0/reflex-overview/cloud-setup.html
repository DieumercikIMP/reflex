Recommendations (based on AWS)

For a maximum of security and reliability please consider the following recommendations for hosting ReFlex on AWS:

  • For a high availability of your installation it’s best practice to use at least two nodes in two different availability zones (AZ) in one region of AWS. Even if one AZ is having trouble and going offline the redundancy of at least one other AZ will provide resilience.
  • To limit and control access to your installation all availability zones of one region should be located behind an Elastic Load Balancer (ELB) with a Web Application Firewall (WAF). In each availability zone there should be two subnets: one public subnet with an ELB and one private subnet. For scalability an auto scaling group should be spanned across the private subnets containing the EC2 nodes with the RAS deployed.
  • All traffic from the internet to this Virtual Private Cloud (VPC) and inside the VPC must be SSL encrypted.
  • The recommended size of the EC2 nodes is similar to the general minimum hardware requirements a CPU with 4 cores and preferably 16 GB RAM, e.g. a m4.xlarge EC2 node.
  • To guarantee that all nodes in a cluster use the same configuration and deploy the same ReFlex knowledge base the use of a Elastic File System (EFS) attached to all nodes is recommended.
  • The incoming traffic can be routed round-robin between the nodes as the RAS is stateless, e.g. the traffic can be routed to any availability zone and there to any node (horizontal scaling).

Software installation

Each node in the cloud can be installed as described in the installation guide and the recommended systems. The software installation for nodes hosted in the cloud does not differ from nodes hosted on premise.
