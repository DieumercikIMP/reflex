ReFlex overview

ReFlex is a modular automated underwriting system. It is not a standalone application, rather it has to be integrated into the client’s application landscape. The required ReFlex components are hosted on the client’s site. The components can be installed by following the installation guide. There are different scenarios involving a different number of ReFlex components and different forms of integration into the client’s application. More details can be found in the integration scenarios chapter and the architectural overview.

ReFlex components

The package delivered by Hannover Re includes a client specific knowledge base and a set of standard ReFlex components. For a rough overview see the image below.

The Risk Assessment Service (RAS) is the runtime environment of the knowledge base (KB). RAS is able to drive the assessment based on the KB and offers (remote) interfaces for other components (like UI). It delivers assessment data to the rules engine and transfers processed data back which contain intermediate results and new questions. RAS and the KB are the only mandatory ReFlex components. All other components are optional and may or may not be included in a delivery package depending on the client’s needs (component documentation).

The Customer Experience Platform (CEP) itself is a framework that is divided into CEP-Services and the CEP-Questionnaire. With the CEP it is straight forward to create a web application that interacts with the RAS (component documentation).

  • The CEP-Questionnaire is the UI component that can be integrated into a client’s web portal.
  • The CEP-Services provide a number of REST API services to manage insurance applications and handle the communication with the RAS.

The ReFlex Standalone Portal (RSP) provides the core ReFlex features by combining RAS and CEP to an out-of-the-box-stack to be used with minimal integration and customization efforts. Additional features like product selection page or simple workflow deliver added value (component documentation).

The Document Creation Service (DCS) uses data provided by the RAS and templates to create documents in different formats, e.g. XML and PDF (component documentation).

Third Party Data Services (3PS) are a framework that integrates with external data providers to aid in insurance risk assessment. (integration scenarios / component documentation)

The Management Information System (MI) is a tool for data analysis including an interactive web interface. (integration scenarios)

AppQueue is a thin client that will anonymise and transfer completed applications from the client’s system to the MI at Hannover Re.(integration scenarios)

The Underwriting Workbench (UWB) is a component for the manual underwriting of incoming applications and assessment of incoming claims. It is a stand-alone, web-based application which can be tailored to the insurer‘s needs via many configuration options and offers technical interfaces for integration into existing systems. (component documentation)

The Underwriter’s Workbench is connected to the Internal Workbench (IWB) which is the Hannover Re internal tool for underwriters. The transfer of cases enables the collaboration between the underwriters at Hannover Re and our clients.
