Minimum hardware requirements

ReFlex is designed for high performance and scalability. In order to ensure this, the hardware used should meet the minimum requirements.

An overview for the RAS integration scenario is listed in the following table:

Hardware Explanation Minimum requirement
Storage space for installation Minimum installation size / runtime size for a ReFlex installation (files & logs). 1GB
Memory RAM (heap size) with 1 KB Heap size for the AppServer. ~4GB
Memory RAM (heap size) for n KB’s Approximate number of additional RAM for each KB.
In addition, the number of concurrently used KB’s has to be stored in the RAS property ‘com.hannoverre.reflex.ras.knowledgeBaseCacheSize’, otherwise, RAS will always remove the KB’s from the memory. 		+ ~2GB x every additional KB
CPU 4-cores

Please consider that in case of 3PS usage additional RAM may be required.


Additionally for the CEP integration scenario:

Hardware Explanation Minimum requirement
Storage space per assessment This amount of KB’s should be roughly calculated for each assessment (for autosave, Suspend & Resume).
Your assessment JSON may contain 3PS data as well as attachments from other sources. This data can increase the average size of an assessment by a factor of 2 and even higher. In order to get numbers for an appropriate DB storage setup, the client project has to determine the average JSON file size.
The 3PS-connector-sizing-data has to be provided by the local office to the customers. 		On average 200 kilobytes per assessment
