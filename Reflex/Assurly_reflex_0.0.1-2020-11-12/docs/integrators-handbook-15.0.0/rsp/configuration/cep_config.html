CEP Configuration

CEP configuration files that are mentioned in this chapter:

$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/config/cep-configuration.js

CEP Properties Inside the RSP Configuration

The following properties inside the RSP configuration are mandatory. For further information please check the integrators documentation of the CEP component.

  • com.hannoverre.reflex.cep.security.keystore
  • com.hannoverre.reflex.cep.security.jwsAlgorithmWhiteList
  • com.hannoverre.reflex.cep.security.allowedClockSkew
  • com.hannoverre.reflex.cep.security.signing.kid
  • com.hannoverre.reflex.cep.security.signing.notBeforeTimeOffset
  • com.hannoverre.reflex.cep.security.signing.algorithm
  • com.hannoverre.reflex.cep.security.signing.issuer
  • com.hannoverre.reflex.cep.security.encryption.kid

To exchange tokens between the RSP and the CEP the following properties have to have the identical values as configured inside the CEP properties file.

  • com.hannoverre.reflex.cep.service.integration.signing.tokenScopes
  • com.hannoverre.reflex.cep.service.integration.signing.expirationTime

Avoiding cookies to be attached to cross-site requests. available options are: Strict, Lax, None https://tools.ietf.org/html/draft-ietf-httpbis-cookie-same-site-00 Suggested option is Strict * com.hannoverre.reflex.rsp.services.authentication.sameSiteSetting=Strict

CEP configuration.js

The behaviour of the CEP is configured through the configuration file $REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/config/cep-configuration.js

This section explains CEP configuration that are relevant to run the RSP. For further information please check also the CEP front end configuration.

CEP Auto Save and Auto Submit Functionality of the Questionnaire

autoSubmit

There are two options to control the auto submit feature:

  • autoSubmit: always
  • autoSubmit: reflexive

On autoSubmit: always the CEP will send the current state of questionnaire with the backend every time a question is answered, on autoSubmit: reflexive the CEP will send the current state of questionnaire only if a reflexive question is answered. On submit the questionnaire will be calculated in the backend and new questions will be appended. If the answer isn’t returned from the backend, the frontend will be blocked for any input.

autoSave

To store the current state of the questionnaire automatically the CEP offers the auto save feature. To check every second for a change to the questionnaire and save it to the backend the following configuration is needed:

  • autoSaveEnabled: true
  • autoSaveInterval: 1000
