RSP Front End Customization

RSP customization files mentioned in this chapter:

$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/translations/${cedentId}_${locale}.json
$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/images/${cedentId}-logo-left.png
$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/styles/${cedentId}.css
$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/config/configuration.js

To support a customization of the RSP for different partners, a partner ID is used to identify the cedent.

The initialization takes place on the welcome page and a cookie is used to make sure that on all pages the correct configuration is used.

The default partner ID is 00000.

Configuration

For an alternative cedent configuration, you have to add the files above with the id of the partner to add in the filename.

Inside the configuration.js file, the parameter defaultPartnerId defines the default partner id if the application couldn’t determine the partner ID.

$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/config/configuration.js

defaultPartnerId: 00000

The application stores a cookie named partnerId with the value of the partner ID to ensure that the correct customization is used on all pages.

When the welcome page is called without the partnerId, the application tries to extract the partnerId from the cookie. If no cookie exits, the application will use the default value that is configured in the file configuration.js.

To refer to a partner explicitly, you have to add the partner ID as a path like /rsp/#/welcome/<partnerId>, for example:

  • http://<your.domain>/rsp/#/welcome/00000
  • http://<your.domain>/rsp/#/welcome/12345

Error page

If a request to the RSP or CEP backend returns with a HTTP error Code 500 the RSP is redirecting to the RSP error page. The text of the title and the body can be adapted with the message keys rspErrorPageTitle and rspErrorPageBody from the translation files (see above).
