RSP Configuration

RSP configuration files that are mentioned in this chapter:

$REFLEX_CONFIG_ROOTDIR/rsp/config.properties
$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/config/configuration.js

Authentication Scenarios Supported

RSP supports two authentication scenarios:

  • The client’s authentication page shown inside RSP (using the embedded authentication page).
  • The client’s authentication page shown outside RSP. As a consequence the embedded authentication page won’t be displayed to the user.

To select the scenario, the boolean property externalAuth has to be set accordingly in the file /reflex-config/rsp/cedentConfig/config/configuration.js.

Authentication Inside RSP (externalAuth=false)

The client’s authentication page will be embedded into RSP (through an iframe).

Authentication Outside RSP (externalAuth=true)

The client’s authentication page will be outside the RSP. Once it calls the endpoint rsp/auth/login/verify, the user will be redirected to RSP. Once he logs out (endpoint rsp/auth/logout/verify), he will be redirected again to the external authentication page.

Signing Page Configuration

RSP offers the possibility to embed a signing page into its worflow. To be able to use it, the authentication interface has to be implemented by the integrator. In case the integrator uses the signing page, once the assessment is signed it will change to the state SIGNED before being processed.

To display the signing page, the property showSignPage has to be set to ‘true’ into file $REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/config/configuration.js.

Assessment Result Page

The assessment result page can be configured to either show a rating result for each product and for each rider a unique rating result, or to only show a rating result for each product, hiding the results of the riders.
The option for this switch can be found in the configuration file: $REFLEX_CONFIG_ROOTDIR/rsp/config.properties. It is labeled com.hannoverre.reflex.rsp.result.showRiderRatings. The value can be set to either true, which enables the display of rider results, or false, which hides the rider results.
The rating results of the riders are always contained in the JSON file, that is generated in the process. This switch only affects, which ratings are displayed to the end user.
