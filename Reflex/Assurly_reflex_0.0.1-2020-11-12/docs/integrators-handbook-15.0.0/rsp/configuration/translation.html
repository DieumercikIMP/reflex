Translation and Localization

RSP translation and localization files that are mentioned in this chapter:

$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/config/angular-locale_${locale}.js
$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/translations/${cedentId}_${locale}.json
$REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/translations/cep.js

Angular Translate

This is the AngularJS library used for translation https://github.com/angular-translate/angular-translate. It is configured to load the translation files from a URL, in this case pointing to the RSP resource endpoint /rsp/resource/translations/{partnerId}/{languageId}.

Translation files are located at $REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/translations:

  • {partnerId}_{languageId}.json: RSP translations
  • cep.js: CEP translations file. It contains translations for CEP questionnaire (errors, placeholders, dates, etc. as defined by the IHB in cep/integration/frontend/customization/translation). This configuration is retrieved in the index.html file, in the script section pointing to the RSP resource endpoint: /rsp/resource/cep-translations.

Note: When entering the RSP all supported languages are determined by the found translation files from the cedent configuration. Supporting other languages inside the RSP as provided by the product will raise an error in the questionnaire. In case the browser of the user is set up to an unsupported language the RSP is loading the default language that is configured in the cedents configuration.js file.

When text has to be displayed and translated in the template RSP uses the translate filter:

<div class="panel-heading">
    {{'rspInitialInformationPrepareTitle'| translate }}
</div>

Angular Dynamic Local and Angular i18n

These are the AngularJS libraries used for internationalization. Angular-dynamic-local is configured to use a loader pointing to the RSP resource endpoint: /rsp/resource/configuration/angular-locale_{{locale}}.js. You can obtain those files from the angular-i18n module located in the node_modules folder of the project or directly from https://github.com/angular/angular.js/tree/master/src/ngLocale.

Localization files are placed at $REFLEX_CONFIG_ROOTDIR/rsp/cedentConfig/config:

  • angular-locale_{{locale}}.js

By default, to apply the locale (when changing the language) the page needs to be refreshed. In order to avoid it and instantly apply the locale, you can use the filter inside the translation file, and perform just the translation in the template, like shown below.

In the template:

{{ 'rspInitialInformationCreated' | translate:{created: assessmentWrapper.created} }}

In RSP translations file:

"rspInitialInformationCreated": "Created: {{ created | date : 'shortDate' }}"

This way the dates will also be refreshed with the corresponding locale when the user changes the language.
