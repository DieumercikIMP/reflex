Housekeeping API

If you don’t want or need the housekeeping provided by the RSP, you can use the interface to implement your own database housekeeping.
To disable the integrated housekeeping go to your REFLEX_CONFIG_ROOTDIR find the properties file at reflex-config/rsp/config.properties and set the property com.hannoverre.reflex.rsp.housekeeping.housekeepingActive to false.
If you are looking for further information about the integrated housekeeping refer to the chapter housekeeping.

Assessments

To clean up assessments with the housekeeping API you can send a DELETE Request to the path /rsp/admin/assessments with the following parameters:

Parameter Description
before Will clean up all assessments before the given date.
processed-before Will clean up only assessments with the state processed before the given timestamp.

Example: http://<yourhost>/rsp/admin/assessments?before=2018-12-01T12:00&amp;processed-before=2019-01-01T12:00

In the example above all unfinished assessments before December 1st, 2018 12:00 a.m. and all processed assessments before January 1st 12 a.m. will be cleaned up.

As a result, the RSP will return a report as JSON.

Response:

{
    "succeededAssessments": 2,
    "succeededUsers": 0,
    "assessmentErrorInfo": [{
        "message": "example error message",
        "assessmentId": "assessment id",
        "userContextId": 12345
    }],
    "total": 3,
    "failed": 1
}

Users

To clean up users with the housekeeping API you can send a DELETE Request to the path /rsp/admin with the following parameters:

Parameter Description
before Will clean up all assessments before the given date.

Example: http://<host>/rsp/admin/users?before=2018-12-01

In this example all users that were created before December 1st, 2018 and are not referenced by any assessments will be cleaned up.

Response:

{
    "succeededAssessments": 0,
    "succeededUsers": 2,
    "userErrorInfo": [{
        "message": "example error message",
        "userContextId": 12345
    }],
    "total": 3,
    "failed": 1
}
