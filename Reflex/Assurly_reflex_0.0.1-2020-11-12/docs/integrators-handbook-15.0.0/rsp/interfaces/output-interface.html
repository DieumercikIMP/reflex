Output Interface

To avoid compatibility issues with surrounding systems the RSP has no built-in data transfer mechanism. Instead, as a final step in the RSP backend, a call to the OutputService interface is done. The RSP core does not provide an implementation of this interface, instead the implementation is expected to be loaded at runtime. This implementation is expected to take care of all data transfer to surrounding systems.

The interface is small and contains two methods.

interface OutputService {
    void setConfig(Map<String, String> config) throws Exception;
    void processData(final long userContextId, final String assessmentJson, final byte[] internalPDF, final byte[] externalPDF) throws Exception;
}

The implementation class has to implement the interface above and its fully qualified class name must be provided in <reflex-config>/rsp/config.properties.

com.hannoverre.reflex.rsp.output.implClass=com.hannoverre.reflex.rsp.output.OutputServiceImpl

The implementation class should be packaged as a .jar file and then be placed in <tomcat-home>/lib before starting tomcat.

Right after the implementation is loaded, the setConfig method is called which reads the com.hannoverre.reflex.rsp.output.implconfig property from <reflex-config>/rsp/config.properties. This enables an integrator to make the implementation configurable. Note that this config is not a single value but rather a key/value map.

com.hannoverre.reflex.rsp.output.implconfig={key1:"value1", key2:"value2", ...}

After loading and configuration of the implementation, the processData method is called with the following arguments:

  • userContextId - The internal database id of the user context.
  • assessmentJson - The final risk assessment as JSON.
  • internalPDF - byte array with the internal PDF (includes e.g. detailed assessment results).
  • externalPDF - byte array with the external PDF (the customer’s PDF).

The integrator is free to decide where and how to deliver data or a subset of it. It can be sent using REST, SOAP, FTP, MQ, etc.
