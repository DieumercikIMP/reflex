Resources

The REST adapter provides the following resource endpoints:

GET /check:

Get a fast and simple response for heartbeat checks. This endpoint is not secured by any authentication.
Example: curl -H "Accept: text/plain" -H "Accept-charset: UTF-8" -X GET http://localhost:8080/com.hannoverre.reflex.tps.rest.adapter/check

Responses

Code Name Description Content-Type
200 OK The request was successfully processed. text/plain; charset=utf-8

GET /version:

Get the version of the deployed adapter set at compile time. This endpoint is not secured by any authentication.
Example: curl -H "Accept: text/plain" -H "Accept-charset: UTF-8" -X GET http://localhost:8080/com.hannoverre.reflex.tps.rest.adapter/version

Responses

Code Name Description Content-Type
200 OK The request was successfully processed. text/plain; charset=utf-8

GET /requirements:

Get a list of all mandatory assessment-factors and variables required to call the provider. This endpoint is secured by HTTP basic authentication.
Example: curl -H "Accept: application/json" -H "Accept-charset: UTF-8" -X GET http://tps:password@localhost:8080/com.hannoverre.reflex.tps.rest.adapter/requirements

Responses

Code Name Description Content-Type
200 OK The request was successfully processed. application/json, application/xml, text/plain; charset=utf-8

GET /deliveries:

Get a list of all assessment-factors and variables processed by the connector from provider content. This endpoint is secured by HTTP basic authentication.
Example: curl -H "Accept: application/json" -H "Accept-charset: UTF-8" -X GET http://tps:password@localhost:8080/com.hannoverre.reflex.tps.rest.adapter/deliveries

Responses

Code Name Description Content-Type
200 OK The request was successfully processed. application/json, application/xml; charset=utf-8

POST /acquireAndAnalyze:

Acquire data from the external provider and analyze it. This endpoint is secured by HTTP basic authentication.

Responses

Code Name Description Content-Type
200 OK The request was successfully processed. application/json; charset=utf-8
400 Bad Request The data passed to this resource is not valid, e.g. the data is syntactically invalid or required values are missing. text/plain; charset=utf-8
502 Bad Gateway The adapter received an invalid response from the connector. text/plain; charset=utf-8

POST /analyze:

Analyze the raw data in the request without contacting the provider. This endpoint is secured by HTTP basic authentication.

Responses

Code Name Description Content-Type
200 OK The request was successfully processed. application/json; charset=utf-8
400 Bad Request The data passed to this resource is not valid, e.g. the data is syntactically invalid or required values are missing. text/plain; charset=utf-8
502 Bad Gateway The adapter received an invalid response from the connector. text/plain; charset=utf-8

Accessing the raw data from a provider

Sometimes it might be necessary to look at the raw data returned by the provider, e.g. for reviewing a result. The raw data could include more data than what is mapped by 3PS. If a third party system was asked its data is included in the assessment. In the application.json it’s contained in the section “provided”.

There might and will be multiple provided-data blocks. The ones that were created by 3PS have the “type” “ThirdPartyService”. Each block has an “id” that is a short name of the provider, here in the example it’s “mib”.

"source" : {
    "id" : "mib",
    "type" : "ThirdPartyService"
}

If you identified the right block of provided data you find the raw response in the value of the “3ps-raw” element. The data in Base64 encoded to not compromise the validity of the JSON. You can decode the data with any tool or online service. The result will be the raw response that 3PS received from the provider, usually in XML format.

Error codes

Common error responses codes

Code Name Description Content-Type
401 Unauthorized The request to the specific resource is due to authentication restriction not permitted. text/plain; charset=utf-8
404 Not Found Specified resource not found, URL invalid or does not exist. text/plain; charset=utf-8
405 Method Not Allowed The request to the specific resource uses a wrong HTTP method, e.g. POST instead of GET. text/plain; charset=utf-8
500 Internal Server Error Request is not processed due to internal server error. text/plain; charset=utf-8
502 Bad Gateway The adapter received an invalid response from the connector. text/plain; charset=utf-8
