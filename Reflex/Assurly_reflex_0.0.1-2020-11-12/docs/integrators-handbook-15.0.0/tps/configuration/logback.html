Logging

To configure the logging behavior of the 3PS REST adapter place a logback-spring.xml in the configuration directory inside the REFLEX_CONFIG_ROOTDIR directory. For details on how to configure Logback please refer to the official documentation for Logback.

GELF appender

The 3PS REST adapter supports the use of Logstash/GELF to send log messages a Graylog server. The sample logback-spring.xml shown below contains a commented out Logstash/GELF appender that can be configured for this purpose.

Please refer to the documentation of the Logstash/GELF appender for more information on customisation options.

Sample logback-spring.xml

<configuration>
    <appender name="tpsFileAppender" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <encoder>
            <pattern>%d{ISO8601} %-5p [%t] %c - %m%n</pattern>
        </encoder>
        <file>${REFLEX_LOG_ROOTDIR}/tps.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <!-- daily rollover -->
            <fileNamePattern>${REFLEX_LOG_ROOTDIR}/tps.%d{yyyy-MM-dd}.log</fileNamePattern>

            <!-- keep 30 days' worth of history capped at 3GB total size -->
            <maxHistory>30</maxHistory>
            <totalSizeCap>3GB</totalSizeCap>
        </rollingPolicy>
    </appender>

    <!--<appender name="tpsGelfAppender" class="biz.paluch.logging.gelf.logback.GelfLogbackAppender">-->
    <!--    <host>udp:localhost</host>-->
    <!--    <port>12201</port>-->
    <!--</appender>-->

    <!-- Must be set to error according to reflex logging conventions -->
    <root level="error">
        <appender-ref ref="tpsFileAppender"/>
    </root>
</configuration>
