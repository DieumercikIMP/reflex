CEP update

This page lists the possibilities that might be considered if a new CEP has been delivered but your RAS and KB have not been updated.

# kind of application that should be assessed CEP used to finish answering process CEP patch release CEP minor release CEP major release
1 newly created newly delivered fully supported fully supported fully supported; it might be required to adjust the KB
2 already existing newly delivered fully supported fully supported fully supported; it might be required to patch the existing application JSON files or even adjust the KB
3 already existing old CEP should still be used no valid use case because it would mean that no update will be done see patch release see patch release

The CEP, in general, is not required for a medical assessment of existing or new applications. It simply provides additional functionality on top of the RAS middleware. Because of that, even a new version of the CEP can be regarded as compatible with the existing RAS. In any other case, CEP and RAS would be shipped together. Since the CEP is forwarding the cases to the RAS middleware, it will be able to do this if new cases are created with no restrictions as mentioned in the first usage scenario described in the table above. Only in case a major update was delivered, could it be required to adjust the integration of the questionnaire or add features to the knowledge base. Detailed information about those needed adjustments can be found in the migration notes of the specific version in the appendix.

Should it be necessary to handle existing cases inside the new CEP, it might be required to update the JSON structure of those cases before sending them to the CEP or even adjust the knowledge base. Apart from this potential migration of application JSON files, this use case is supported in the same way the first one is. Detailed migration notes will also be found inside the appendix.

For this kind of delivery the third use case is not considered to exist, because if there should be the requirement to still use the existing CEP it would basically mean that the new CEP will not be used by the client.
