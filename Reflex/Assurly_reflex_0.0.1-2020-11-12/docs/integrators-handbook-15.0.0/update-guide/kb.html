Knowledge base update

This page lists the update scenarios that might exist if a new knowledge base has been delivered. The table below is based on the kind of applications that should be handled inside the integrating system:

  1. Those thall will be created after the new knowledge base was installed.
  2. Those you already have been created with the previous version and are supposed to be assessed with the new knowledge base.
# kind of application that should be assessed knowledge base used for assessment support
1 newly created newly delivered fully supported
2 already existing newly delivered not supported
3 already existing old KB that was used to create the case fully supported; in a CEP integration scenario the addressing of the correct KB is handled by CEP, in the RAS integration scenario the integrating client system has to make sure that the correct is addressed

In this scenario, a new knowledge base has been delivered. It is assumed that this knowledge base is compatible with the RAS that is already been installed in the client’s environment.

The first scenario does not require any specific attention by the integrator. The creation of future cases and their assessment with the new knowledge base would be the default behaviour. In contrast, the assessment with the new knowledge base of cases that have been created in the past is not supported by ReFlex. For those applications, only the third scenario can be applied.

Existing applications that have been saved inside the integrated environment have to be assessed with the knowledge base they have been created with in the first place. Because of that, it is required that the still required knowledge bases are not uninstalled during the installation of the new knowledge base. If the client is using the RAS integration scenario he has to assure that those legacy applications are sent to the matching knowledge base. The RAS middleware provides a link to use each deployed knowledge base as well as one link to use the latest one for convenience purposes. Inside the client’s integrating system it has to be assured that existing applications are only assessed by the means of the specific link for the corresponding knowledge base.

In case the client uses the CEP integration scenario no further activities are required. The CEP is making sure that the correct knowledge base is used.
