Restrict access to internal DCS documents

The DCS (Document Creation Service) can be configured with different templates as described in the DCS template configuration section. There might be PDF reports that contain confidential information that should not be visible to the customer like risk assessment details. This article describes how the access to those reports can be restricted.

Deployment of internal DCS templates

For the RAS integration scenario it is always necessary to create documents via the DCS REST API. This approach is also possible in the CEP integration scenario. In addition it is also possible to create documents by using the <cep-pdf-link /> directive which internally creates documents by using the CEP documents endpoint. This directive should only be used to create documents for customers only. Confidential documents have to be deployed and generated in a more secure network zone than the webhosting zone. The figure below outlines the deployment of public and internal DCS templates in a CEP integration scenario. The DCS used to generate customer documents from the CEP (cep-core) must never contain internal document templates. Internal documents will be created in the more secured intranet zone that is dedicated for back office processing. For this purpose a dedicated DCS (dcs-internal) instance is being hosted in the intranet zone. Technically the dcs-internal is a standard DCS component which will be configured using the internal DCS templates.

In case the CEP integration scenario is used without using the <cep-pdf-link />directive, the cep documents endpoint should be deactivated via configuration.
