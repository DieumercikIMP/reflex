Electronic Signatures

Overview

At the end of a standard sales process an assessment result is returned by ReFlex and the end user might have the chance to generate a document to apply for the selected products and riders. At this point, it is either possible that the end user has to print, sign, and send in the document by post. Alternatively, the portal could integrate the services of an electronic signature provider such that the end user can directly review and sign the document online.

Integration scenario

The sequence diagram below describes the path from the generation of an offer document after the completion of an assessment until the completion of the signing ceremony as well as the retrieval of the signed offer document from the electronic signature service provider.

In general, a signing ceremony using an electronic signature service provider consists of the following steps. The actual integration may vary depending on the service provider as well as depending on market regulations and country laws.

Starting the signing ceremony

The signing ceremony is started by the end user via the client’s sales system after the assessment for the selected products and riders has been completed and an offer document has been generated. The sales system then sends a signing request to the electronic signature service provider which at least contains the document to be signed as well as information about the signer.

In order to provide the electronic signature service provider with information where the signature as well as potential other information should be inserted into the document, it may either be possible to provide absolute placement information for the document or to provide anchors inside the document that can then be referenced in the signing request. Below, a code snippet from a sample PDF template for DCS is displayed that shows how such an anchor (in a white font) can be placed in the document.

<fo:table-cell>
    <fo:block>
        <fo:block>
            Signature of policy holder
        </fo:block>
        <fo:block color="#FFFFFF">
            <xsl:text>signer1sig</xsl:text>
        </fo:block>
    </fo:block>
</fo:table-cell>

Subsequently, the electronic signature service provider has to provide access to the end user (signer) to perform the signing ceremony. Here, different service providers may offer different solutions for providing access to the signer. Two common solutions are a signing invitation containing an URL to the service provider that is sent to the signer by e-mail or an embedded solution where the signing process is embedded in the sales system.

Authenticating the signer

After the start of the signing ceremony, the end user has to authenticate himself before he can review and sign the document. Here again, different electronic signature service providers offer different authentication methods. Some of the most common methods include e-mail authentication, SMS authentication, and knowledge based authentication.

Reviewing and signing the document

Once the end user has authenticated himself successfully, the electronic signature service provider will present the document to the end user to be reviewed and ultimately signed. Once the end user has signed the document, he generally has the opportunity to download the signed document and the signing ceremony is completed.

Retrieving the signed document

After the end user has signed a document, the last step generally is for the client’s sales system to retrieve the signed document from the electronic signature service provider. Typically, the sales system is either informed by an event notification from the service provider that a document has been signed or the sales system is polling the service provider to check whether new documents have been signed.
