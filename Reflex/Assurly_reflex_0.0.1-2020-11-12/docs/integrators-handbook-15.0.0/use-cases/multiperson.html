Multi person assessments

Although ReFlex natively does not support multi person assessments, it is possible to perform this kind of assessment with some additional integration work.

This section provides information and tips to perform multi person assessments.

General concepts

Multi person assessments allow including several to be insured persons in a single policy. Example: a family life insurance which is paid when one member of the family dies.

A multi person assessment consists of several individual assessments which need to be completed one at a time. After all of them are completed a global decision is obtained and the individual results are aggregated to calculate possible global loadings or postpone durations.

The applicant user should not be able to see any individual assessment result before all the assessments have been completed.

RAS and KB aspects

The cedent application will receive the multi person attribute as a product constraint when retrieving the product information via the RAS API, as explained in this product example, and if set to true it will execute the multi person assessment logic.

CEP integration

If the CEP integration model is used, each assessment has to be started separately as indicated in the CEP integration mechanism section.

The front end application must know the number of persons to be included in the policy (this data may be introduced by the applicant) and perform a loop of assessments until the last one is completed. All the assessments will be stored in the CEP persistence for active applications (as explained in the CEP persistence section), so they can be retrieved after the multi person assessment completion.

The cedent application has to calculate the global decision and aggregate the possible loadings and postpone durations.

Thanks to the Suspend and Resume CEP feature, the applicant may suspend the multi person application and resume it later on (even from a different device). In this case the applicant would be able to continue the particular assessment where it was left without the need to repeat the already completed assessments.

Global decision and results aggregation

The result of each individual assessment must be read from the results : products : ratings and results : products : riders properties in the JSON file, as explained in the Product result section.

These results must be aggregated to provide a global decision with the corresponding loadings, postpone duration, etc. A priority should be established to calculate the global results.

DCS

A specific template must be created to obtain the multi person application PDF document.

The individual assessment JSON documents can be concatenated and sent to the DCS using the document POST endpoint. The template will then transform the custom JSON file into a single PDF document containing all the individual assessments information and the global decision results.
