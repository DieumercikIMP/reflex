Application configuration (IWB)

See Application Configuration for general properties.

In order to start the application in IWB mode the configuration property spring.profiles.active=IWB needs to be set. This can be done by either adding the property to the application.properties or by setting an environment variable.

The following properties are only relevant if the Workbench is deployed as IWB (for internal use at Hannover Re). You may override any of the shown configurations in the following file:

  • $REFLEX_CONFIG_ROOTDIR/iwb/application-IWB.properties
##########################################################################################
##                                                                                      ##
##                                   IWB Properties                                     ##
##                                                                                      ##
##########################################################################################

##########################################################################################
##  OWS - Sanction Screening                                                            ##
##########################################################################################

# Description:
# This property has no effect if the active profile is UWB.
# Sanction screening is always disabled for the UWB.
com.hannoverre.workbench.iwb.ows.enabled=false

# Description:
# URL of the sanction screening endpoint
# Mutability: Start
# Example: https://sanctionscreening.hannover-re.grp/ows
# com.hannoverre.workbench.iwb.ows.url=
# com.hannoverre.workbench.iwb.ows.username=
# com.hannoverre.workbench.iwb.ows.password=


##########################################################################################
##  Currency Database                                                                   ##
##########################################################################################

# Description:
# JDBC URL of the currency conversion database
# Mutability: Start
# Example: jdbc:sqlserver://iwbdb:1433
# com.hannoverre.workbench.iwb.currency.datasource.url=
# com.hannoverre.workbench.iwb.currency.datasource.username=
# com.hannoverre.workbench.iwb.currency.datasource.password=

# Description:
# This property controls the maximum number of milliseconds that a client
# will wait for a connection from the pool. If this time is exceeded without
# a connection becoming available, a SQLException will be thrown. Lowest
# acceptable connection timeout is 250 ms.
# Mutability: Start
# Example: 10000
com.hannoverre.workbench.iwb.currency.datasource.connectionTimeout=10000

# Description:
# This property controls whether the pool will "fail fast" if the pool cannot be
# seeded with an initial connection successfully. Any positive number is taken to
# be the number of milliseconds to attempt to acquire an initial connection; the
# application thread will be blocked during this period. If a connection cannot
# be acquired before this timeout occurs, an exception will be thrown. This
# timeout is applied after the connectionTimeout period. If the value is zero
# (0), HikariCP will attempt to obtain and validate a connection. If a connection
# is obtained, but fails validation, an exception will be thrown and the pool not
# started. However, if a connection cannot be obtained, the pool will start, but
# later efforts to obtain a connection may fail. A value less than zero will
# bypass any initial connection attempt, and the pool will start immediately
# while trying to obtain connections in the background. Consequently, later
# efforts to obtain a connection may fail.
# Mutability: Start
# Example: 30000
com.hannoverre.workbench.iwb.currency.datasource.initializationFailTimeout=30000

# Description:
# Fixed delay for updating the currency information
# Mutability: Start
# Example: 3600000
com.hannoverre.workbench.iwb.currency.sync.schedule.fixedDelay=3600000


##########################################################################################
##  TAF Import                                                                          ##
##########################################################################################
# Description:
# Enables or disables the TAF import module.
# Note that TAF Import is only available in the IWB profile.
# Possible values: true, false
# Mutability: Start
com.hannoverre.workbench.iwb.tafimport.enabled=false

# Description:
# Fixed delay for the TAF import process (checking for new mails in the import queue)
# Mutability: Start
# Example: 300000
com.hannoverre.workbench.iwb.tafimport.schedule.fixedDelay=300000

# Description:
# Target office for the TAF imports
# Mutability: Start
# Example: 1
# com.hannoverre.workbench.iwb.tafimport.officeId=1

# Description:
# IMAP mail server for TAF imports
# Mutability: Start
# com.hannoverre.workbench.iwb.tafimport.imaps.host=
# com.hannoverre.workbench.iwb.tafimport.imaps.port=
# com.hannoverre.workbench.iwb.tafimport.imaps.username=
# com.hannoverre.workbench.iwb.tafimport.imaps.password=

# Description:
# If set to "*", all hosts are trusted. If set to a
# whitespace separated list of hosts, those hosts are trusted. Otherwise, trust
# depends on the certificate the server presents.
# Mutability: Start
# Example: my-mail-server.com
com.hannoverre.workbench.iwb.tafimport.imaps.ssl.trust=


##########################################################################################
## RoBP - Register of Business Partner                                                  ##
##########################################################################################

# Description:
# This property has no effect if the active profile is UWB.
# RoBP is always disabled for the UWB.
# Possible values: true, false
# Mutability: Start
com.hannoverre.workbench.iwb.robp.enabled=false

# Description:
# This property defines the client type that should be used to fetch business partners.
# Possible values:
# HREJDBC = The HREJDBC client fetches business partners directly from the HRE RoBP database.
# This type should be used for production environments.
# MOCKJSON = The MOCKJSON client fetches business partners from a pre-defined json file.
# This type should only be used for demo environments that have no access to the HRE RoBP database.
# Mutability: Start
com.hannoverre.workbench.iwb.robp.clientType=HREJDBC

# Description
# This property controls how long (milliseconds) the system will wait for a response from the
# business partner client before the request is cancelled. In case of HREJDBC the value must be
# greater than robp.hrejdbc.datasource.connectionTimeout and
# robp.hrejdbc.datasource.initializationFailTimeout
# Mutability: Start
# Example: 15000
com.hannoverre.workbench.iwb.robp.clientTimeout=15000

# Description:
# Client type HREJDBC
# JDBC URL of the HRE RoBP database.
# Mutability: Start
# Example: jdbc:sqlserver://iwbdb:1433
# com.hannoverre.workbench.iwb.robp.hrejdbc.datasource.url=
# com.hannoverre.workbench.iwb.robp.hrejdbc.datasource.username=
# com.hannoverre.workbench.iwb.robp.hrejdbc.datasource.password=

# Description:
# Client type HREJDBC
# This property controls the maximum number of milliseconds that a client
# will wait for a connection from the pool. If this time is exceeded without
# a connection becoming available, a SQLException will be thrown. Lowest
# acceptable connection timeout is 250 ms.
# Mutability: Start
# Example: 10000
com.hannoverre.workbench.iwb.robp.hrejdbc.datasource.connectionTimeout=10000

# Description:
# Client type HREJDBC
# This property controls whether the pool will "fail fast" if the pool cannot be
# seeded with an initial connection successfully. Any positive number is taken to
# be the number of milliseconds to attempt to acquire an initial connection; the
# application thread will be blocked during this period. If a connection cannot
# be acquired before this timeout occurs, an exception will be thrown. This
# timeout is applied after the connectionTimeout period. If the value is zero
# (0), HikariCP will attempt to obtain and validate a connection. If a connection
# is obtained, but fails validation, an exception will be thrown and the pool not
# started. However, if a connection cannot be obtained, the pool will start, but
# later efforts to obtain a connection may fail. A value less than zero will
# bypass any initial connection attempt, and the pool will start immediately
# while trying to obtain connections in the background. Consequently, later
# efforts to obtain a connection may fail.
# Mutability: Start
# Example: 1
com.hannoverre.workbench.iwb.robp.hrejdbc.datasource.initializationFailTimeout=1

# Description:
# This property controls if business partner synchronization is enabled.
# Possible values: true, false
# Mutability: Start
# Example: true
com.hannoverre.workbench.iwb.robp.syncEnabled=true

# Description:
# This property controls how long (milliseconds) the system will wait before
# stopping the business partner synchronization. The value must be greater than
# robp.hrejdbc.datasource.connectionTimeout and robp.hrejdbc.datasource.initializationFailTimeout
# if client type is HREJDBC.
# Mutability: Start
# Example: 20000
com.hannoverre.workbench.iwb.robp.syncTimeout=20000

# Description:
# This property controls when synchronizations are started by the system.
# Note that a synchronization is started automatically on system startup if synchronization
# is enabled.
# The format of the cron expression is seconds/minutes/hours/day of month/month/day of week
# and is always resolved in UTC.
# Cron expression generator - https://www.freeformatter.com/cron-expression-generator-quartz.html
# Mutability: Start
# Example: 0 0 21 ? * * (every day at 21pm)
com.hannoverre.workbench.iwb.robp.syncCron=0 0 21 ? * *
