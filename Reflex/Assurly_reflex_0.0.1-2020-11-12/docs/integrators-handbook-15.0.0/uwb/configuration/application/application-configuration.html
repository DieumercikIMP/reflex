Application configuration

Application wide settings are configured using a properties files. The following listing shows the reference configuration for the UWB backend. You may override any of the shown configurations in the following file:

  • $REFLEX_CONFIG_ROOTDIR/uwb/application.properties
  • (or, for backward compatibility $REFLEX_CONFIG_ROOTDIR/uwb/config.properties)

The following placeholders can be used in property files:

  • ${reflex.config.rootdir} Will evaluate to the environment variable $REFLEX_CONFIG_ROOTDIR.
  • ${reflex.config.dir} Will evaluate to the configuration directory for your context path (relative to $REFLEX_CONFIG_ROOTDIR).

##                                                                                      ##
##                                   UWB Properties                                     ##
##                                                                                      ##

# Mutability: When may a value be changed and when would the change be respected by the system
#   Once: Has to be set before the application is started for the first time. Should never change.
#   Start: Can be changed before the system is (re-)started.
#   Runtime: Can be changed while the system is running.
#

# Description:
# When false, shows a banner in the frontend stating that this is not a production system.
# Mutability: Start
# Possible values: true, false
com.hannoverre.workbench.production=false

# Description:
# The case numbers of new cases will be prefixed with this property. For example UWB-1234.
# The prefix should be a 2-4 character long trimmed upper case alphabetic string (no numbers).
# Mutability: Once
# Example: UWB
# com.hannoverre.workbench.casePrefix=UWB



##########################################################################################
##  Application Database                                                                ##
##########################################################################################

# Description:
# JDBC URL of the application database
# Mutability: Start
# Example: jdbc:sqlserver://yourserver:1433
# com.hannoverre.workbench.datasource.url=
# com.hannoverre.workbench.datasource.username=
# com.hannoverre.workbench.datasource.password=

# Description:
# This property controls the maximum number of milliseconds that a client
# will wait for a connection from the pool. If this time is exceeded without
# a connection becoming available, a SQLException will be thrown. Lowest
# acceptable connection timeout is 250 ms.
# Mutability: Start
# Example: 10000
com.hannoverre.workbench.datasource.connectionTimeout=10000

# Description:
# This property controls whether the pool will "fail fast" if the pool cannot be
# seeded with an initial connection successfully. Any positive number is taken to
# be the number of milliseconds to attempt to acquire an initial connection; the
# application thread will be blocked during this period. If a connection cannot
# be acquired before this timeout occurs, an exception will be thrown. This
# timeout is applied after the connectionTimeout period. If the value is zero
# (0), HikariCP will attempt to obtain and validate a connection. If a connection
# is obtained, but fails validation, an exception will be thrown and the pool not
# started. However, if a connection cannot be obtained, the pool will start, but
# later efforts to obtain a connection may fail. A value less than zero will
# bypass any initial connection attempt, and the pool will start immediately
# while trying to obtain connections in the background. Consequently, later
# efforts to obtain a connection may fail.
# Mutability: Start
# Example: 30000
com.hannoverre.workbench.datasource.initializationFailTimeout=30000

# Mutability: Start
## Example: 1000
com.hannoverre.workbench.searchIndex.searchCountTimeoutMillis=1000

##########################################################################################
## UMAC Security Configuration (OAUTH)                                                  ##
##########################################################################################

# Description:
# The name of the workbench in the OIDC provider
# Mutability: Start
# Example: uwb
# com.hannoverre.workbench.security.clientId=

# Description:
# The name of the workbench user interface application in the OIDC provider (optional)
# Mutability: Start
# Example: uwb-ui
# com.hannoverre.workbench.security.uiClientId=

# Description:
# The attribute name used to access the user's name from the user info response.
# The attribute must always contain the same name for a single user and must be unique.
# The example matches the username field emitted by Keycloak's profile mapper.
# Example: preferred_username
com.hannoverre.workbench.security.userNameAttributeName=preferred_username

# Description:
# The com.hannoverre.workbench.security.provider section contains all properties that are required to
# communicate with your OAuth2 authorization server. All these informations can usually be obtained from the
# authorization servers .well-known/openid-configuration.
# - issuer: The authorizations servers name as URI.
# - jwksUri: The Address of the authorization servers jwks_uri.
# - authorizationEndpoint: The Address of the authorization servers authorization_endpoint.
# - tokenEndpoint: The Address of the authorization servers user token_endpoint.
# - userInfoEndpoint: The Address of the authorization servers user_info_endpoint.
# - endSessionEndpoint (optional): The Address of the authorization servers end_session_endpoint (if supported).
# Examples (for Keycloak, adjust the hostname to your environment):
# com.hannoverre.workbench.security.provider.issuer=https://keycloak/auth/realms/reflex
# com.hannoverre.workbench.security.provider.jwksUri=https://keycloak/auth/realms/reflex/protocol/openid-connect/certs
# com.hannoverre.workbench.security.provider.authorizationEndpoint=https://keycloak/auth/realms/reflex/protocol/openid-connect/auth
# com.hannoverre.workbench.security.provider.tokenEndpoint=https://keycloak/auth/realms/reflex/protocol/openid-connect/token
# com.hannoverre.workbench.security.provider.userInfoEndpoint=https://keycloak/auth/realms/protocol/openid-connect/userinfo
# com.hannoverre.workbench.security.provider.endSessionEndpoint=https://keycloak/auth/realms/reflex/protocol/openid-connect/logout

# clients with the same office or referred office.
# In case of problems, turn this feature off.
# Mutability: Start
# Default: true
