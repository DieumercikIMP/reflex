Mail service configuration

UWB provides some features which require sending emails via an SMTP server, e.g.

  • [Send Message Feature](../../(features/email-templates.html)

If these features are used, the mail server properties have to be configured.

Create the file javamail.properties in directory ${REFLEX_CONFIG_ROOTDIR}/uwb and add at least the following parameters:

# Socket connection timeout value in milliseconds. This timeout is implemented by java.net.Socket. Default is infinite timeout.
mail.smtp.connectiontimeout=30000

# Socket read timeout value in milliseconds. This timeout is implemented by java.net.Socket. Default is infinite timeout.
mail.smtp.timeout=15000

# Socket write timeout value in milliseconds. This timeout is implemented by using a java.util.concurrent.ScheduledExecutorService per connection that schedules a thread to close the socket if the timeout expires. Thus, the overhead of using this timeout is one thread per connection. Default is infinite timeout.
mail.smtp.writetimeout=15000

# Chunk size when sending multipart emails, in bytes. Defaults to 1024 when not set. More info: https://docs.oracle.com/cd/E19957-01/816-6028-10/asd2j.htm
mail.smtp.chunksize=262144

We recommend leaving the values shown above.
