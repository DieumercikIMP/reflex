Permission configuration

The UWB uses Role Based Access Control (RBAC) to ensure that only authorized users can access data or perform actions. The authentication is delegated to an external OAuth2 based Authorization Server. Configuration of this external system is described in the Authorization Server Installation chapter.

Currently the UWB supports a very coarse-grained permission concept. Finer role based access are planned for future versions.

Rules for scoped roles

UWB roles are mostly scoped to an office. Offices serve as a tenant boundary. A user may have roles in one office but different (or no) roles in other offices. An office is identified by its two-letter office code (see also: Office Configuration). In the following examples, the office code TE is used. Some roles may also be scoped by vendor code.

Some roles can also be scoped by the vendor. This is especially useful when an external system, that works on behalf of a vendor is given permission to work on cases only for that vendor. In the following examples, the vendor code PFEFFERMINZIA is used. Note that a vendor scoped role MUST also be office scoped.

Examples:

  • USER.TE
  • ADMIN: Administration of all offices
  • OPS_PROMETHEUS: Always without scope
  • API_CASE_IMPORT.TE.PFEFFERMINZIA: May import cases for the office TE and the vendor PFEFFERMINZIA
  • API_CASE_IMPORT: May import cases for all offices and vendors

User roles

Role Description
ADMIN(.TE) Grants administrations permissions in the UWB Admin Interface.
ADMIN_DELETE_CASES(.TE) Permits a user to delete cases physically (!) from the database.
ADMIN_MANAGE_CLIENT(.TE) Permits a user to manage the office configuration (excluding vendors), i.e. text modules, laboratory values, requirements…
ADMIN_MANAGE_VENDORS(.TE) Permits a user to manage vendors including their contacts and underwriting guidelines.
ADMIN_AUDIT_SAMPLE_REPORT(.TE) Permits a user to generate monthly audit report.
ADMIN_MANAGE_AUTHORITIES(.TE) Permits a user to manage underwriting authorities for office and users. (Note: this functionality has not been implemented yet)
USER(.TE) Required base role for all regular users within an office.
USER_VIEW_CLAIM(.TE) Permits a user to see claims.
USER_MODIFY_CLAIM(.TE) Permits a user to work on claims.
USER_CREATE_CLAIM(.TE) Permits a user to create new claims.
USER_DECIDE_CLAIM(.TE) Permits a user to advance a claim workflow to a signed state (also: closing).
USER_VIEW_APPLICATION(.TE) Permits a user to see applications.
USER_MODIFY_APPLICATION(.TE) Permits a user to work on applications.
USER_CREATE_APPLICATION(.TE)> Permits a user to create new applications.
USER_DECIDE_APPLICATION(.TE) Permits a user to advance an application workflow to a signed state (also: closing).
USER_SEND_MESSAGE(.TE) Permits a user to send messages as emails as part of the case workflow.
USER_CREATE_REPORT(.TE) Permits a user to create monthly reports.
USER_ASSIGN_CASES(.TE) Permits a user to assign cases to another user / underwriter.

External system roles

Role Description
API_CASE_DECISION_EXPORT(.TE) The external system is allowed to export decisions.
API_CASE_IMPORT(.TE(.PFEFFERMINZIA)) The external system is allowed to import cases using the UWB Import API.
API_REFLEX_IMPORT(.TE(.PFEFFERMINZIA)) The external system is allowed to import ReFlex cases using the UWB ReFlex Import API.
OPS_PROMETHEUS The external system is allowed to retrieve monitoring information from the UWB REST API. This Role has no scoping, monitoring is applied globally.
OPS_LOGGERS The external system is allowed to change the loggers settings using the spring boot actuator REST API. See chapter Change log level.
