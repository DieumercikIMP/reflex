Key fact configuration

The assessment summary page for cases (applications as well as claims) contains a section that contains important key facts to provide a brief overview for a case. This section is called “Key facts” and can be customized for each office.

Key fact configuration parameters

The key fact configuration is currently performed via SQL. Each entry in the OFFICE table has the columns:

  • KEYFACTS_CONFIG_APPLICATIONS which contains the key fact configuration document for APPLICATIONS
  • KEYFACTS_CONFIG_CLAIMS which contains the key fact configuration document for CLAIMS

Please note that both of these columns are required to contain valid key fact configurations, regardless if a case category is actually used by an office.

For unsupported case categories, you can always use the following minimal valid key fact configuration document which simply contains no key facts or columns at all:

{
  "sections": []
}

Key fact configuration document structure

A Key fact section contains up to four columns while each column can contain multiple sections with a header and a details link. The configuration is encoded as JSON-document with the following structure. At the root level is a two-dimensional array which describes columns and their sections. Currently only up to four columns are supported. Each column contains an array of sections. Each section has an optional label, and optional Details-link (which often is a relative link pointing to another case view) and an array of rows. Each row has a label and a value that refers to one of the supported displayable Key Fact Fields. In addition it can be defined how many rows are shown in the UI by setting the groupLimit attribute. If it is not set a default value is used. Each section can also have subsections. The UWB-UI does how ever only display the first two layers of sections. Please note that the sub-sections field “sections” is mandatory and usually should contain an empty array ([]).

Example:

{
  "sections": [
    [ 
      {
        "label": "Occupational",
        "detailsLink": "../details/#OCCUPATIONAL",
        "sections": [],
        "rows": [
          {
            "label": "Occupation",
            "value": "OCCUPATION"
          }
        ]
      },
      {
        "label": "Financial",
        "detailsLink": "../details/#FINANCIAL",
        "sections": [],
        "rows": [
          {
            "label": "Income",
            "value": "ANNUAL_INCOME"
          },
          {
            "label": "Net Worth",
            "value": "NET_WORTH"
          },
          {
            "label": "Total Sum of Existing Policies",
            "value": "SUM_EXISTING_POLICIES"
          },
          {
            "label": "Ultimate Total",
            "value": "ULTIMATE_TOTAL"
          },
          {
            "label": "Beneficiary Relationship",
            "value": "BENEFICIARY_RELATIONSHIP"
          },
          {
            "label": "Additional Info",
            "value": "FINANCIAL_ADDITIONAL_INFO"
          },
          {
            "label": "Reason(s) for cover",
            "value": "REASONS_FOR_COVER"
          },
          {
            "label": "Existing Ins.",
            "value": "EXISTING_INSURANCES"
          }
        ]
      }
    ],
    [
      {
        "label": "Medical",
        "detailsLink": "../details/#MEDICAL",
        "sections": [],
        "rows": [
          {
            "label": "BMI",
            "value": "BMI"
          },
          {
            "label": "Height/Weight",
            "value": "HEIGHT_WEIGHT"
          },
          {
            "label": "Smoker",
            "value": "SMOKER"
          },
          {
            "label": "Alcohol",
            "value": "ALCOHOL_CONSUMPTION"
          }
        ]
      }
    ],
    [
      {
        "sections": [],
        "rows": [
          {
            "label": "Impairments",
            "value": "TOP_IMPAIRMENT_LIST",
            "groupLimit": "3"
          },
          {
            "label": "Examinations / Documents",
            "value": "TOP_EXAMINATION_LIST"
          }
        ]
      }

        "label": "Avocation/Geograph.",

      }
    ]
  ]
}

update OFFICES set KEYFACTS_CONFIG_APPLICATIONS = '<key-fact-config>' where ID = <office-id>;

update OFFICES set KEYFACTS_CONFIG_CLAIMS = '<key-fact-config>' where ID = <office-id>;

Key fact fields

The following list contains fields which can be configured as key fact row value.

  • OCCUPATION Displays the insured persons occupation with its remarks (if present).
  • OCCUPATION_FIRST_COVER_RATING Displays the insured persons occupation with its first cover rating.
  • ANNUAL_INCOME Displays the insured persons annual income (if present).
  • NET_WORTH Displays the insured persons net worth (if present).
  • SUM_EXISTING_POLICIES Displays the insured persons sum of existing policies (if present).
  • ULTIMATE_TOTAL Displays the insured persons ultimate total (if present).
  • BENEFICIARY_RELATIONSHIP Displays the insured persons relationship to the beneficiary (if present).
  • FINANCIAL_ADDITIONAL_INFO Displays the additional info field from the financial section (if present).
  • REASONS_FOR_COVER Displays a list of cover reasons (default group limit 6).
  • EXISTING_INSURANCES Displays the number of existing insurances.
  • BMI Displays the BMI of the insured person if both weight and height are present.
  • BMI_FIRST_COVER_RATING Displays the BMI and its ratings of the insured person if both weight and height are present.
  • BLOOD_PRESSURE Displays the blood pressure of the insured person if present.
  • BLOOD_PRESSURE_FIRST_COVER_RATING Displays the blood pressure and its ratings of the insured person if present.
  • HEIGHT_WEIGHT Displays the height and weight of the insured person.
  • SMOKER Displays whether a person is a smoker and details if present.
  • ALCOHOL_CONSUMPTION Displays whether a person drinks alcohol and details if present.
  • TOP_IMPAIRMENT_LIST Displays the impairments with their medication (default group limit 3).
  • TOP_IMPAIRMENT_LIST_FURTHER_DETAILS Displays the impairments with their further details (default group limit 3).
  • TOP_IMPAIRMENT_LIST_FIRST_COVER_RATING Displays the impairments with their first cover rating (default group limit 3).
  • TOP_EXAMINATION_LIST Displays the examinations with their completionDate and remarks (default group limit 3).
  • AVOCATIONAL Displays activities with their remarks (default group limit 3).
  • AVOCATIONAL_FIRST_COVER_RATING Displays activities with their first cover rating (default group limit 3).
  • GEOGRAPHICAL Displays geographical stays with their remarks (default group limit 3).
  • GEOGRAPHICAL_FIRST_COVER_RATING Displays geographical stays with their first cover rating (default group limit 3).
  • LNRC_SCORE Extracts the LNRCScore from an existing LNRC ThreePsResult.
  • LNRC_MVR_PERSON_FOUND Displays whether a person has a positive Motor Vehicle Record (MVR) match. Only applies for cases where a MVR-3PS result is provided.
  • LNRC_MVR_STATE_DLN Displays the Motor Vehicle Record (MVR)’s State and DLNumber fields if present. Only applies for cases where a MVR-3PS result is provided.
  • MIB_IAI_COUNT Displays number of found AIA-Reports. Only applies for cases where a MIB-3PS result is provided.
  • MIB_CODES_COUNT Displays number of found MIB-Codes. Only applies for cases where a MIB-3PS result is provided.

The following fields are only available for claims:

  • TREATY_NAME Displays the treaty for this claim.
  • TREATY_ID Displays the treaty id for this claim.
  • POLICY_ISSUANCE_DATE Displays the policy for this claim.
  • RISK_COMMENCEMENT_DATE Displays the risk commencement date for this claim.
  • REINSTATEMENT Displays whether this claim has a reinstatement and the reinstatement date.
  • CLAIM_EVENT_DATE Displays the event date of this claim (if present).
  • CLAIM_TYPE Displays the event type of this claim (if present).
  • CAUSE_OF_CLAIM Displays the event cause of this claim (if present).
  • POLICY_DURATION_UNTIL_CLAIM Displays the duration between risk commencement date and claim event date.
  • DURATION_SINCE_LAST_UW Displays the duration between the date of reinstatement and the claim event date.
  • TOP_CLAIM_DOCUMENTS_LIST Displays the claim documents (default group limit 3).
