Office configuration

At least one office is required for the UWB to function properly. Office specific configuration includes regional settings such as date and time formats, units, languages and more. A case is always handled in the context of an office. Specifically, data for an office is strictly separated from data for other offices.

Create office with basic configuration

insert into OFFICES (
  ID, CODE, DISPLAY_NAME, LANGUAGE_CODE, DATE_FORMAT, TIME_FORMAT, TIMESTAMP_FORMAT,
  DEFAULT_CURRENCY, DEFAULT_HEIGHT_UNIT, DEFAULT_WEIGHT_UNIT, DEFAULT_ASIAN_BUILD, DEFAULT_WAIST_HIP_UNIT,
  CAN_ACCELERATE_CASES, CAN_CLASSIFY_CASES, MAX_SLA_TIME_IN_HOURS,
  CAPACITYCHECK_MAIL_TO, CAPACITYCHECK_MAIL_CC,
  TIMEZONE_ID, VERSION, FROM_EMAIL, KEYFACTS_CONFIG_APPLICATIONS, KEYFACTS_CONFIG_CLAIMS,
  ASCENT_REGION_ID, ENABLE_CLAIMS, LOCAL_NAME, CITY, ADDRESSLINE,
  THREEPS_LNRC_CONNECTOR_URL_TEMPLATE, THREEPS_MIB_CONNECTOR_URL_TEMPLATE, THREEPS_MIB_UPDATE_CODES_CONNECTOR_URL_TEMPLATE,
  LOCAL_NAME_SUPPORT, PRINT_IMPAIRMENTS_DEFAULT, CAN_SIGN_CASES, CAN_REVIEW_CASES, PAPER_SIZE_DECISION_LETTER,
  UNDERWRITER_DISCLOSEABLE, ENABLE_CAPACITY_CHECK, INCLUDE_PENDING_CASES_IN_MONTHLY_REPORT, POLICY_NUMBER_REQUIRED_CLAIMS)
values (<office-id>, 'TE', 'Test Office', 'en-GB', 'dd/MM/yyyy', 'HH:mm', 'dd/MM/yyyy HH:mm',
        'EUR', 'CENTIMETERS', 'KILOGRAMS', 1, 'CENTIMETERS',
        0, 0, 48,
        'to@no.capacity.check.test', 'cc@no.capacity.check.test',
        'Europe/London', 0, null, 'keyfacts_app', 'keyfacts_claim',
        1, 1, 'Hannover Re Test Office', 'London', '8987 Round Apple Vista, London, XY3M 2BE, Tel. +44 222222-1234, Fax -1235, www.hannoverlifere.co.uk',
        null, null, null,
        0, 'ALL', 1, 0, 'A4',
        0, 0, 0, 0);

Mutability: Specifies under which circumstances a configuration setting may be changed and whether a restart is required to apply the change.

  • Never: Cannot be changed (mostly because it comes from the codebase. If changed in code a database migration might be necessary)
  • Once: Might be set once per application lifecycle but must not be changed afterwards (otherwise, undefined behaviour might occur).
  • Start: Can be changed before the system is (re-)started.
  • Runtime: Can be changed while the system is running (mostly from Admin UI, or directly by the integrator)
Field Description Type Required Mutability
ID Unique technical identifier, e.g. 1 bigint Once
Code Two letter shorthand code e.g. DE to uniquely identify an office. It is used in the case reference number in the format of {case-prefix}-{code}-{number} eg. HRE-DE-000123. nvarchar(2) Once
Display Name Full name of the office used in the UI. nvarchar(40) Start
Language Code Language Code in the format xx-YY used for i18n, e.g. de-DE. The language code is used for internationalisation and localization. The currently fully supported languages are limited to en-GB and en-US. nvarchar(10) Start
Date Format Specifies how dates are formatted, e.g. dd.MM.yyyy nvarchar(16) Start
Time Format Specifies how time of date is formatted, e.g. dd.MM.yyyy HH:mm nvarchar(16) Start
Timestamp Format Specifies how timestamps are formatted, e.g. HH:mm nvarchar(16) Start
Default Currency The default currency ISO code, e.g. EUR nvarchar(3) Start
Default Height Unit The default unit of height inputs (CENTIMETERS, FEETS_INCHES) nvarchar(20) Start
Default Weight Unit The default unit of weight inputs (KILOGRAMS, POUNDS, STONES_POUNDS) nvarchar(20) Start
Default Asian Build Determines if asian build (medical section) is pre-selected for new cases bit Start
Default Waist Hip Unit The default unit of waist and hip inputs (CENTIMETERS, INCHES) nvarchar(20) Start
Can Accelerate Cases Always false in this context bit Once
Can Classify Cases Enables the option to classify a case with regard to the associated underwriting effort (easy, difficult, etc…) bit Start
Maximum SLA Time Specifies the maximum SLA time in hours for every vendor/channel and product (if not overwritten in vendor/channel configuration) int Start
Capacity Check Mail To Address List of the To recipients for generated capacity check mails (use semicolon to separate mail addresses if using MS Outlook) nvarchar(255) Start
Capacity Check Mail CC Address List of the CC recipients for generated capacity check mails (use semicolon to separate mail addresses if using MS Outlook) nvarchar(255) Start
Timezone ID Specifies the timezone in which the office is located. All values (e.g. Europe/Berlin) supported by the JVM are allowed. nvarchar(30) Start
Version Technical field used for optimistic locking. Always 0. int Once
From Email Specifies the From mail address for all office-specific mail features (like Send Messages). Should be a mailbox accessible by the underwriting team. nvarchar(160) Start
Key facts Config Applications For the Key facts documentation refer to Key Facts nvarchar(max) Runtime
Key facts Config Claims For the Key facts documentation refer to Key Facts nvarchar(max) Runtime
Enable Claims Enables the claims feature for this office bit Start
Local Name The local name for the office. Used for document creation nvarchar(120) Start
City The city of the office. Used for document creation nvarchar(120) Start
Address Line The address of the office. Used for document creation nvarchar(512) Start
Local Name Support Enables local name support for the office. The local name support currently only affects insured persons. When it is activated an additional local name (e.g. in chinese letters) can be captured for the insured person. bit Start
Print Impairments Default Specifies how impairments are included in the decision letter by default (ALL, NONE, SELECTED) nvarchar(10) Start
Can Sign Cases Specifies whether the signing workflow (second signature) is enabled or not. Must not be enabled together with the review workflow (“Can Review Cases”). bit Start
Can Review Cases Specifies whether the review workflow is enabled or not. Must not be enabled together with the signing workflow (“Can Sign Cases”). bit Start
Paper Size Decision Letter Specifies the paper size for decision letters. Possible values are A4 and US_LETTER. nvarchar(10) Runtime
Ascent Region ID Specifies the corresponding Ascent Region by their ID. Every office belongs to a specific Ascent Region. int Runtime
Underwriter disclosable in Reports Specifies whether case owner (underwriter) shall appear in the workload and case report bit Runtime
Enable Capacity Check Enables the capacity check feature for this office bit Runtime
Include Pending Cases from monthly Report Include the pending cases in the monthly report bit Runtime
Policy number required for claims Specifies whether the policy number is a mandatory field when working with claims. bit Runtime

Add supported languages to an office

For a basic office configuration, at least one supported language is required.

The supported languages for an office can be set with:

-- insert into LANGUAGE_CODES (LANGUAGE_CODE) values ('en-GB');
insert into OFFICES_LANGUAGE_CODES (OFFICE_ID, LANGUAGE_CODE) values (<office-id>, 'en-GB');

The supported languages are used for the document creation configuration (e.g. decision letters). For a list of the supported languages see Language Code in the table above.
