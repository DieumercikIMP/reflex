Vendor/channel configuration

At least one vendor/channel is required for the UWB to function properly. A vendor/channel object in UWB may represent a broker, a vendor or sales channel which generates incoming cases (insurance applications or claims). If you would like to split your cases by vendor or channel, you can implement this behaviour by setting up multiple vendors/channels in UWB.

Create vendor with basic configuration

Vendors can be created via the Admin Portal. An integrator tool for setting up an initial office configuration including a vendor/channel is planned for a future release of UWB.

Pre-requisites

The admin user needs the following roles to be able to manage vendors:

  • ADMIN
  • and ADMIN_MANAGE_VENDORS

See page Permission configuration for more details.

Steps to create new vendor via the Admin Portal

  • Open the UWB Admin Portal.
  • Select menu item “Vendor / Channel”
  • Press button “Add Vendor / Channel”
  • Fill in at least all mandatory attributes for the new vendor, mainly
    • Name - this is the vendor name used in the UWB user interface.
    • Reference - this is the internal reference code used in the technical APIs (REST endpoints) to reference the vendor.
  • Press “Save” to create new vendor.
