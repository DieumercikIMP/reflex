Ascent configuration

Ascent is the Hannover Re Underwriting Manual and provides rating information about numerous occupations, impairments, avocational activities and countries. Ascent also offers some calculators to determine the rating for standard risks, e.g. for BMI ratings, Blood Pressure ratings, etc.

The UWB closely integrates with Ascent and the UWB user may open Ascent rating content and access Ascent calculators directly from within UWB. In addition to the settings which have to be done via the global application configuration, a few office specific-settings have to be done via the database as well with regard to the Ascent integration:

Ascent region id

The region ID is used to determine which region-specific manual / version of Ascent shall be used. When accessing an Ascent API the region id is queried from the selected vendor/channel of a case. If this vendor/channel has no region ID configured, the region ID of the office will be used.

To configure an Ascent region ID for a vendor/channel, the following statement can be used:

update CEDENTS set ASCENT_REGION_ID = <ascent-region-id> where ID = <vendor/channel-id>;

To configure an Ascent region ID for an office the following statement can be used:

update OFFICES set ASCENT_REGION_ID = <ascent-region-id> where ID = <office-id>;

Since an office-specific region id is mandatory when adding an office to the DB, a region ID is always set. This even applies if the Ascent integration is disabled.

Ascent product id

The product id is used to map an UWB cover type to an Ascent product. This is required to apply the received ratings (from various Ascent APIs) to the corresponding cover. There is no one-to-one relationship between Ascent products and UWB cover types so there may be cover types for which no corresponding Ascent product exists. The mapping can be configured using the Admin Portal.
