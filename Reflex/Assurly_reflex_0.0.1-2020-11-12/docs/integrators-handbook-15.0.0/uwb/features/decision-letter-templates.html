Decision letter templates

Apache FreeMarker templates can be used to create email templates or decision letter templates.

Table TEMPLATES

All templates can be configured inside the database for each office in the table named TEMPLATES.

Column Description Required
ID Primary Key backed up by the sequence TEMPLATES__SEQ.
OFFICE_ID The ID of the office to which this template belongs. Office IDs can be found in the table OFFICES.
NAME The template name, i.e. “My Template”.
LANGUAGE_CODE The language code which describes the the localization of the content, e.g. formatting of numbers and dates will be done according to the associated locale. The language code may consist of a language and optionally a region/country. The language part must be a valid ISO 639-1 code and the region/country part a valid ISO-3166 code separated by a hyphen, e.g. “en” or “en-GB”. The language code part can be omitted if this template shall be used for all languages. Templates with a more specific language/region code will override templates with a less specific language code.
TEMPLATE_NAME The internal template name is constructed as a concatenation of the name, “_” and the language code where the hyphen has been replaced by an underscore, e.g. “My Template_en_GB”.
TEMPLATE_CONTENT The content of the template. This column contains the FreeMarker template text.
CREATION_DATE The creation date/time for documentation purposes.
LAST_UPDATE The date / time of the last update of this template for documentation purposes. New templates should have a LAST_UPDATE value equals to the CREATION_DATE value.
VERSION The version number of this template. New templates should be inserted with a version 0.

Special templates

Some template names are reserved for special purposes.

Name Purpose Fallback
decision_letter Template for the PDF decision letter with the underwriting result for applications. English decision letter file decision_letter.ftl
decision_letter_text Template for the Text decision letter. English version in file decision_letter_text.ftl
claims_decision_letter Template for the PDF decision letter for claims. English version in file claims_decision_letter.ftl
case_html_export Template for exporting a case to HTML. English version in file case_html_export.ftl
case_text_export Template for exporting a case as plain text. English version in file case_text_export.ftl

The FreeMarker template language

FreeMarker is a powerful template language and around for quite a bit of time. You can find an excellent documentation of the language under the link https://freemarker.apache.org/docs/index.html.

A very short reference:

FreeMarker Syntax Description
${var} Insert the value of var at this place.
${var!“nothing”} Insert the value of var at this place or the word nothing if var is missing.
[#if test]
… do something
[/#if] 		Check if test is true and then execute more template language.
[#list seq as v]
.. do for each item
[/#list] 		Loop construct, iterate over seq and use v as variable for each item.

PDF output

To create PDF output a FreeMarker template is required to create a FO-XML output which then will be converted to a PDF. A quick introduction can be found at https://xmlgraphics.apache.org/fop/quickstartguide.html.

Available data

The following list shows the available variables which can be used inside the FreeMarker templates. Variables in brackets may be missing.

  • today - format 'dow mon dd hh:mm:ss zzz yyyy', e.g. 'Mon Mar 02 10:44:59 CET 2020'
  • currentUser - currently logged-in user
    • userName - log-in / username
    • displayName - full name
    • emailAddress - email address
  • iwbNb - UWB case number
  • office - office configuration
    • name - office name
    • localName - localized (e.g. translated) office name
    • city - office address city
    • addressline - office address street and additional address details
    • language - office language and country, e.g. 'en' or 'en-GB'
    • paperSize - paper size for PDF generation, can be 'A4' or 'US_LETTER'
    • currency - default currency (ISO 4217 three-letter ('Alpha-3') currency code)
  • caseCategory - indicates whether the case is an 'APPLICATION' or a 'CLAIM'
  • caseType - further type information depending on the case category, e.g. 'CLAIM_OPINION'
  • [cedentPolicyNb] - application / claim reference number
  • creationDate - timestamp when case was imported/captured in UWB
  • lastUpdate - timestamp when case was last updated
  • caseReceiptDate - date when application/claim was received (e.g. by email)
  • cedent - insurance company, vendor or broker details
    • name - full name of the insurance company, vendor or broker details
    • [localName] - localized name of the insurance company, vendor or broker details as printed on PDF
    • [displayName] - short name of the insurance company, vendor or broker details as shown in UWB
    • [addressline] - street and additional address details
    • [postcode] - address postcode
    • [city] - address city
    • [phoneNumber] - insurance company's contact phone number
    • [faxNumber] - insurance company's fax number
    • [country] - address country
    • [decision_letter_template] - used for the HR logo / footer only
    • printInsuredPersonsName - indicates whether to print full insured person details ('FULL_NAME'), only initials ('INITIALS_ONLY') or do not print name at all ('DO_NOT_PRINT')
  • [cedentContact] - Name of the contact person for the current application or claim case
  • [cedentContactLocalName] - Localized name of the contact person for the current application or claim case
  • [owner]
    • userName - log-in / username
    • displayName - full name
    • emailAddress - email address
  • participants[0..n] - additional parties involved in the insurance application / claim
    • role - participant's role, e.g. 'BENEFICIARY', 'POLICY_HOLDER', 'SHAREHOLDER', 'PARENT', 'OTHER'
    • name - participant's person name or company name (depends on type)
    • [roleDetails] - additional details about the participant's role, only if role = 'OTHER'
    • [furtherDetails] - additional information about the participant (free text)
    • [country] - residence country
    • [type] - 'Person' or 'Company'
    • [gender] - 'MALE' or 'FEMALE', only if type = 'Person'
    • [birthday] - only if type = 'Person'
    • [relation] - relationship to the insured, e.g. 'SPOUSE, 'CHILD', 'PARENT', 'SIBLING', 'ESTATE', 'TRUST', 'BUSINESS', 'BUSINESS_PARTNER', 'CREDITOR', 'OTHER', only if type = 'Person'
  • tags - list of tags assigned to the current case (available tags depend on the configuration)
  • personCount - number of insured persons for this application or claim case
  • persons[1..n] - insured persons in this application or claim case
    • policyInForce - 'true' if the case has been marked as policy in force, 'false' otherwise
    • [policyInForceDate] - date when policy was put in force
    • [firstname] - insured person's first name
    • [localFirstname] - insured person's localised first name
    • [middlename] - insured person's middle name
    • [localMiddlename] - insured person's localised middle name
    • lastname - insured person's surname
    • localLastname - insured person's localised surname
    • fullname - insured person's full name in format 'lastname, firstname middlename'
    • localFullname - insured person's localized full name in format 'localLastname, localFirstname localMiddlename'
    • birthday - insured person's date of birth
    • gender - insured person's gender, either 'MALE' or 'FEMALE'
    • [nationality] - insured person's nationality
    • [personalIdNumber] - insured person's person identification number, e.g. social security number, passport id, bank id, etc.
    • [classification] - classification of underwriting effort involved with the application / claim, one of 'SIMPLE', 'EASY', 'MEDIUM', 'DIFFICULT', 'VERY_DIFFICULT', 'JUMBO'
    • [signer] - user who signed/approved the underwriting / claims assessment decision
      • userName - log-in / username
      • displayName - full name
      • emailAddress - email address
    • [secondSigner] - user who 2nd signed/approved the underwriting / claims assessment decision (4-eye-principal)
      • userName - log-in / username
      • displayName - full name
      • emailAddress - email address
    • [reviewer] - user who reviewed the underwriting / claims assessment decision
      • userName - log-in / username
      • displayName - full name
      • emailAddress - email address
    • attachments[0..n] - application / claim documents
      • uploadDate - timestamp when case document was added in UWB
      • fileName - name of case document file
      • [uploader] - log-in / user name of user who uploaded the case document file
      • [description] - custom description entered for the uploaded case document
      • [issueDate] - custom date entered for the uploaded case document
    • printImpairments - specifies whether to print all ('ALL'), no ('NONE') or only ('SELECTED') selected impairments on the PDF decision letter
    • [coverLetterText] - free cover letter text entered for the PDF decision letter
    • textModules[0..n] - text blocks printed on the decision letter for the current case
      • category - currently always set to 'DECISION_LETTER'
      • position - determines the position in case multiple text modules / blocks are printed
      • label - short name to identify text module
      • content - actual text module phrase
      • language - text module language
    • cover[0..n] - selected insurance covers
      • coverType - name of the associated insurance type
      • coverTypeLocalName - localiuzed name of the associated insurance type as printed on the PDF decision letter
      • coverTypeDisplayName - short name of the associated insurance type as shown in the UWB user interface
      • authorityType - risk type, one of 'LIFE', 'PERSONAL_ACCIDENT', 'DISABILITY', 'CRITICAL_ILLNESS', 'LONG_TERM_CARE', 'HEALTH', 'ENHANCED_ANNUITIES', 'CDI'
      • inceptionDate - requested start date for the risk coverage
      • [benefitType] - payment type for the benefit, one of 'LUMP_SUM', 'YEARLY_PAYMENT', 'MONTHLY_PAYMENT', 'DAILY_PAYMENT', 'OTHER'
      • [benefitSum] - associated insurance sum / regular payment
      • [benefitSumInOfficeCurrency] - insurance sum converted to the default currency for the office
      • currency - insurance sum currency
      • [durationType] - determines how the policy duration is specified, one of 'IN_YEARS', 'IN_MONTH', 'IN_DAYS', 'END_AGE', 'WHOLE_LIFE'
      • [durationPeriod] - associated duration period (e.g. number of years/months/days, end age)
      • [totalSumAtRisk] - calculated total initial sum insured
      • [remarks] - remarks for the cover
      • [remarksForCedent] - remarks for the cover to print on the PDF decision letter
      • decisions[0..n] - underwriting decision with regard to current cover
        • [MEDICAL] | [FINANCIAL] | [AVOCATIONAL] | [GEOGRAPHICAL] | [OCCUPATIONAL] | [OVERALL]
          • categoryName - decision category, one of 'MEDICAL', 'FINANCIAL', 'AVOCATIONAL', 'GEOGRAPHICAL', 'OCCUPATIONAL' and 'OVERALL'
          • category - translated category name
          • decisionTypeName - name of decisionType, one of 'STANDARD', STANDARD_BOR'DERLINE, 'OFFER', 'POSTPONE', 'DECLINE', 'NOT_QUALIFIED', 'CLOSE', 'REQUEST', 'OTHER'
          • decisionType - translated name of decisionType name
          • descisionAttributeSetType Attribute Set enum, supported values depend on the custom decision configuration
          • descisionAttributeSetTypeName Attribute Set Name, supported values depend on the custom decision configuration
          The following attributes may be filled depending on the custom decision configuration and the selected descisionAttributeSetType:
          • [loading] - extra mortality or premium loading in percent
          • [totalLading] - total mortality or total premium in percent
          • [flatExtra] - flat extra loading in per mille of sum assured
          • [tempFlatExtra] - temporary flat extra loading in per mille of sum assured
          • [tempFlatExtraDuration] - duration for temporary flat extra
          • [tempFlatExtraDurationUnit] - duration period for temporary flat extra, one of 'MONTH', 'YEARS'
          • [exclusionClause] - textual exclusion clause
          • [restrictionClause] - textual restriction clause
          • [duration] - postpone duration
          • [durationUnit] - postpone duration period, one of 'MONTH', 'YEARS'
          • [untilEvent] - textual description of postpone until event
          • [resultText] - textual description of associated decision
          • [male] - ELE/MPLE values for enhanced annuity products (male)
            • [underwritingLevel] - underwriting level
            • [eleInYears] - ELE (estimated life expectancy) in years
            • [eleInMonths]- ELE (estimated life expectancy) in months
            • [mpleInYears] - MPLE (maximum probable life expectancy) in years
            • [mpleInMonths] - MPLE (maximum probable life expectancy) in months
          • [female] - ELE/MPLE values for enhanced annuity products (female)
            • [underwritingLevel] - underwriting level
            • [eleInYears] - ELE (estimated life expectancy) in years
            • [eleInMonths]- ELE (estimated life expectancy) in months
            • [mpleInYears] - MPLE (maximum probable life expectancy) in years
            • [mpleInMonths] - MPLE (maximum probable life expectancy) in months
          • [curveCode] - mortality curve for enhanced annuity ratings
          • [underwritingLevel] - underwriting level for immediate care annuity (CDI) ratings
          • [impairedAle] - ALE (average probable life expectancy) for immediate care annuity (CDI) ratings
          • [impairedMple] - MPLE (maximum probable life expectancy) for immediate care annuity (CDI) ratings
          • [remarks] - textual remarks for the decision
          • [claimDetails] - textual description of claim decision
    • assessment
      • [occupational]
        • occupation - name of occupation
        • [remarks] - further details for occupation
      • [medical]
        • [height] - insured person's height
        • heightUnit - insured person's height unit, one of 'CENTIMETERS', 'FEET_INCHES'
        • [weight] - insured person's weight
        • weightUnit - insured person's weight unit, one of 'KILOGRAMS', 'POUNDS', 'STONES_POUNDS'
        • [asianBuild] - indicates whether the insured person is of asian build or not
        • [waist] - insured person's waist size
        • [hip] - insured person's hip size
        • waistHipUnit - insured person's waist and hip unit, one of 'CENTIMETERS', 'INCHES'
        • [bloodSystole] - systolic blood pressure value
        • [bloodDiastole] - diastolic blood pressure value
        • [bloodNbMedications] - number of medications taken to treat elevated blood pressure
        • [smoker] - smoker indicator (yes/no)
        • [smokingBehavior] - textual description of smoking behaviour
        • [drinksAlcohol] - drinks alcohol indicator (yes/no)
        • [alcoholBehavior] - textual description of alcohol consumption
        • [laboratoryValues] - further remarks regarding the laboratory value results
        • [familyHistory] - textual description of history certain diseases in the insured person's family
        • [impairments[0..n]]
          • impairmentLabel - name of impairments / medical condition
          • [medications] - textual description regarding the medication taken to treat the associated impairment
          • [remarks] - textual description with further details regarding the associated impairment
          • [calculatorResult]
            • calculatorType - 'CONTENT' or Ascent calculator type, e.g. 'BMI', 'BP', 'IR', 'RENAL', 'DM', 'REWARD', 'CAD', 'LFT'
            • calculatorState - state of calculation - one of 'OK', 'FAILED', 'OUTDATED'
          • [ratings[0..n]]
            • coverType - effective cover type display name
            • [rating] - rating decision, one of 'STANDARD', 'LOADING', 'FLAT_EXTRA', 'DECLINE', 'IC', 'POSTPONE', 'RMD', 'REFER_TO_HR', 'RFC', 'CMD', 'MEDICAL_REPORT', 'EXCLUDE', 'OTHER'
            • [loading] - extra mortality or premium loading in percent for rating decision = 'LOADING'
            • [flatExtra] - flat extra loading in per mille for rating decision = 'FLAT_EXTRA'
            • [permanentLoading] - optional premium loading in percent for rating decision = 'FLAT_EXTRA'
            • [duration] - optional duration for rating decision = 'FLAT_EXTRA'
            • [durationType] - optional duration period for rating decision = 'FLAT_EXTRA', one of 'MONTHS', 'YEARS'
            • [other] - additional detalis for rating decision = 'OTHER'
          • [ratingDetails] - additional details regarding the associated impairment rating
          • selected - indicates whether the associated impairment shall be printed on the decision letter or not
        • [examinations[1..n]]
          • name - name of examination / medical requirement
          • [completionDate] - date when associated examination was conducted or medical requirement created
          • [remarks] - further details regarding the associated examination
        • [labValues[1..n]]
          • label - name of laboratory value
          • defaultValue - whether the laboratory values is marked as default or not
          • [unit] - optional unit for the (numeric) laboratory value
          • type - type of laboratory values, one of 'NUMERIC', 'SELECTION', 'TEXT'
          • [referenceRange] - reference range for the (numeric) laboratory value (e.g. '> 55', '< 100' or '55 - 100')
          • value - measured laboratory value result
      • financial
        • [annualIncome] - insured person's annual income
        • [incomeCurrency] - currency for insured person's annual income
        • [netWorth] - insured person's net worth
        • [netWorthCurrency] - currency for insured person's net worth
        • [sumExistingPolicies] - total sum of existing policies for insured person
        • [sumExistingPoliciesCurrency] - currency for total sum of existing policies for insured person
        • [ultimateTotal] - ultimate total sum assured for insured person including the current policy
        • [ultimateTotalCurrency] - currency for ultimate total sum assured
        • [beneficiaryRelationship] - insured person's relationship to beneficiary, one of 'SPOUSE', 'CHILD', 'PARENT', 'SIBLING', 'ESTATE', 'TRUST', 'BUSINESS', 'BUSINESS_PARTNER', 'CREDITOR', 'OTHER'
        • [coverReasons[1..n]]
          • cover reason name, e.g. 'Personal Protection'
        • [additionalInfo] - further details for financial assessment
        • existingInsurances[0..n]
          • coverTypeLocalName - insurance cover's effective name
          • concurrentApplication - denotes whether the existing insurance application is one of several concurrrent applications
          • [insurer] - name of insurance company which issued the associated policy
          • [remarks] - additional details for the existing policy
          • [toBeReplaced] - denotes whether the current application will replace the associated insurance coverage
          • [durationTypeEnum] - translated name for duration period (see below)
          • [durationType] - duration period, one of 'IN_YEARS', 'IN_MONTH', 'IN_DAYS', 'END_AGE', 'WHOLE_LIFE'
          • [durationPeriod] - policy duration for existing insurance
          • [benefitType] - type of sum assured, one of 'LUMP_SUM', 'YEARLY_PAYMENT', 'MONTHLY_PAYMENT', 'DAILY_PAYMENT', 'OTHER'
          • [benefitSum] - sum assured for existing insurance
          • sumCurrency - currency for sum assured
      • avocational[0..n]
        • activity - name of avocational activity
        • [remarks] - further details regarding avocational activity
      • geographical[0..n]
        • country - name of country for foreign travel / residential risk
        • business - denotes whether the insured person travels for business reasons
        • moreThan6Month - denotes whether the insured person travels for more than 6 months
        • [remarks] - further details regarding foreign travel / residence
    • alternativeDecision - denotes whether a market decision exists which overrides the underwriting decision
    • isAvocationalIrrelevantForAssessment - denotes if avocational activities are irrelevant for the assessment and therefore there is no avocational rating / underwriting decision
    • isFinancialIrrelevantForAssessment - denotes if financial information are irrelevant for the assessment and therefore there is no financial rating / underwriting decision
    • isGeographicalIrrelevantForAssessment - denotes if information about travel/residence are irrelevant for the assessment and therefore there is no geographical rating / underwriting decision
    • isMedicalIrrelevantForAssessment - denotes if medical information are irrelevant for the assessment and therefore there is no medical rating / underwriting decision
    • isOccupationalIrrelevantForAssessment - denotes if occupational information are irrelevant for the assessment and therefore there is no occupational rating / underwriting decision
    • remarksForCedent - the remarks for the insurance company or the market decision if 'alternativeDecision' is set
