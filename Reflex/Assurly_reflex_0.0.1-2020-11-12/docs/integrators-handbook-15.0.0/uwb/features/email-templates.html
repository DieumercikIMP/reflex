Email templates

The UWB System can send emails to pre-configured recipients directly out of a case when this feature is configured for the vendor/channel of the associated case. To configure an email two templates need to be registered, one for the subject and one for the body of the email. These templates can be reused across vendor/channels. Additionally, an email configuration for the vendor/channel needs to be set up.

Templates for emails create plain text for:

  • Email subject
  • Email body

The email templates can be configured for each vendor/channel of an office inside the table CEDENT_EMAIL_CONFIGURATIONS, after inserting a template for the subject and the body.

Table CEDENT_EMAIL_CONFIGURATIONS

Column Description Required
ID Primary Key backed up by the sequence CEDENT_EMAIL_CONFIGURATIONS__SEQ.
CEDENT_ID The ID of the vendor/channel, to which this configuration applies.
NAME Name of the email configuration as you will see it at the User Interface.
SUBJECT_TEMPLATE The name of the subject template, has to correspond to the NAME column of the TEMPLATES table.
BODY_TEMPLATE The name of the body template, has to correspond to the NAME column of the TEMPLATES table.
EMAIL_TO email address of the receiver or list of email addresses separated by comma.
EMAIL_CC email address of carbon copy receiver or list of email addresses separated by comma.
VERSION The version number of this configuration, new ones should be inserted with version 0.

Sample email configuration

In order to provide a new email configuration you can use an SQL script like the following one, which would insert an email configuration for the vendor/channel with ID 1. It is assumed that there are already two templates inside the TEMPLATES table with the names “subject” and “body”. Because every vendor/channel exists only in one office, the email configuration cannot be directly shared between offices.

insert into CEDENT_EMAIL_CONFIGURATIONS (ID, CEDENT_ID, NAME, SUBJECT_TEMPLATE_NAME, BODY_TEMPLATE_NAME, EMAIL_TO, EMAIL_CC, VERSION)
values (next value for dbo.cedent_email_configurations__seq, <vendor/channel-id>, 'My new email', 'subject', 'body', 'rec@vendor.test', null, 0);

insert into TEMPLATES (ID, OFFICE_ID, NAME, LANGUAGE_CODE, TEMPLATE_NAME, TEMPLATE_CONTENT, CREATION_DATE, LAST_UPDATE, VERSION) values
  (next value for dbo.templates__seq, <office-id>, 'subject', null, 'subject', 'My subject template for case ${iwbNb}', GETDATE(), GETDATE(), 0),
  (next value for dbo.templates__seq, <office-id>, 'body', null, 'body', '[#ftl outputFormat="plainText" stripWhitespace=true encoding="UTF-8"]
[#setting dateFormat="yyyy-MM-dd"]
[#assign person = persons[0]]
[#assign cover = persons[0].covers[0]]
Dear colleagues,

could you kindly register the following applicant:

  Reference Number:  ${(cedentPolicyNb)!"unknown"}

  Name, First Name:  ${person.lastname}${(", " + person.firstname)!}
  Gender:  ${person.gender}
  Date of Birth:  ${person.birthday?date}  (YYYY-MM-DD)
  Nationality:  ${(person.nationality)!"unknown"}

  Type of Business and sum applied:
  ${cover.coverType} - ${(cover.benefitSum)!"unknown amount"} ${(cover.currency)!} ${(cover.benefitType)!}

Kind regards
  ${(currentUser.displayName)!}

This email was automatically created by the ReFlex Underwriting Workbench (UWB)!
', GETDATE(), GETDATE(), 0);
