Import API

The UWB provides an application programming interface (API), enabling third party applications to interact with the UWB application via a technical interface.

The UWB API is secured by OAuth2 and requires the client application to provide a valid JSON Web Token (JWT) to be granted access. This token must be issued by the same authorization server that is used for user authentication.

The required permissions and their fallback semantics are documented in the Permission Configuration chapter.

The following sections illustrates how to setup Keycloak to issue a valid JWT for API access. If you choose another identity server / solution for authorization, please consult the documentation of the selected solution to set it up accordingly.

Clients

Select the existing UWB client and add the following import roles:

  • case import - API_CASE_IMPORT
  • link case - API_CASE_IMPORT_LINK

Create a new client with the following attributes.

Attribute Value
Client ID A unique client id (e.g. importapi)
Client Protocol openid-connect

Settings

Attribute Value
Access Type confidential
Standard Flow Enabled off
Direct Access Grants off
Service Accounts Enabled on

Credentials

Attribute Value
Client Authenticator Client id and Secret

Service account roles

Attribute Value
Client Roles Existing UWB client (e.g. uwb)
Assigned Roles API_CASE_IMPORT and/or API_CASE_IMPORT_LINK
