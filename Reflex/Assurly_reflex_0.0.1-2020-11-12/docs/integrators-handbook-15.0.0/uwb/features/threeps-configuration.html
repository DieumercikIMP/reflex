3PS integration into UWB

The primary goal of the third party data services (or 3PS) is to integrate external data providers as a source for information that is available for the RAS as part of the automatic underwriting process. ReFlex UWB also integrates some selected 3PS into the manual underwriting process. At the moment, UWB is able to retrieve and process third-party-data from the following providers via the according ReFlex 3PS Connectors.

  • MIB: The MIB group provides services designed to protect insurers, policyholders, and applicants from attempts to conceal or omit information material to underwriting life & health insurance. This service is only available in the US and Canadian markets. The information provided by the MIB group can be retrieved and used from within the UWB via the ReFlex 3PS MIB Connector.
  • LNRC: LexisNexis is a provider for miscellaneous risk management services. UWB retrieves motor vehicle records (MVR) and/or risk classifier (RC) information about insurance applicants via the ReFlex 3PS LNRC Connector.

Configuration

To enable the 3PS feature in UWB system in order to retrieve / send data from 3PS you have to configure a connector URL for each 3PS connector. These connector URLs can be configured office-wide or specifically for each vendor/channel. The system checks for a vendor/channel specific configuration and if there is no url for this vendor/channel, the system will use the office configuration instead.

For example the LNRC connector can be configured office-wide inside the table OFFICES.

update OFFICES set THREEPS_LNRC_CONNECTOR_URL_TEMPLATE = 'http://tps:password@iwb-3ps-adapter:8080/rest-adapter/acquireAndAnalyze' where ID = <office-id>;

For example the LNRC connector can be configured for each vendor/channel of an office inside the table CEDENTS.

update CEDENTS set THREEPS_LNRC_CONNECTOR_URL_TEMPLATE = 'http://tps:password@iwb-3ps-adapter:8080/rest-adapter/acquireAndAnalyze' where ID = <channel/cedent-id>;
