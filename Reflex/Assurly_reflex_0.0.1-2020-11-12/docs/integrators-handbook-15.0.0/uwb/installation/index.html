UWB installation guide

Minimum hardware requirements

The below requirements assume a maximum number of approximately 100 UWB users (Underwriters) with approximately 50 concurrent users.

Hardware Explanation Minimum requirement
Storage space for installation Minimum installation size / runtime size for a ReFlex UWB installation (files & logs). This does not include the required storage space for the database and the authorization server. 2 GB
Memory RAM (heap size) Heap size for the AppServer. 4 GB
Database storage space per case (data without attachments) The database storage required for an application or claim case mainly depends on the textual description added by the Underwriters. 100 KB
Database storage space per case (attachments) Highly depends on the integration scenario and the business scenarios. As a very rough estimate, one could assume an average number of 10 attachments per case with 1 MB each. However, the database size should be monitored closely to be able to increase the available storage as needed. 10 MB
CPU UWB does not perform heavy computations, therefore a standard, modern CPU will be sufficient. Standard multi-core CPU.

Quick start guide / installation overview

The following general steps need to be performed to set up a new UWB application (the procedure to update an existing installation is different):

  • Install / setup a supported database system (see section Recommended Systems in the ReFlex Overview chapter of this handbook), e.g. MS SQL Server, PostgresSQL) with default settings.
  • Create a database for UWB on the database system.
  • Setup authorization server for Identity and Access Management (e.g. Keycloak).
  • Configure authorization server as described in chapter Authorization Server Installation.
  • Install ReFlex UWB on a supported application server and Java version (see section Recommended Systems in the ReFlex Overview chapter of this handbook).
  • Harden Application Server, see section “Application Server Hardening” below.
  • Adjust default application configuration in application properties as described in chapter Configuration Documentation.
  • Start ReFlex UWB. This will automatically create the required database schema (tables, sequences, constraints …).
  • Setup initial office configuration in the database as described in chapter Configuration Documentation.
  • Open UWB in a supported browser, e.g. if installed on the local system with default configuration, open localhost:8180/uwb

Additional installation notes

Verify installation

The Health Actuator endpoint can be used to verify that the application is up and running.

HTTP GET: http://localhost/uwb/api/actuator/health

{"status":"UP"}

Authentication

Please refer to the chapter Authorization Server Installation.

Logging

The delivered artifact contains a sample logback-spring.xml file which can be found in directory REFLEX_CONFIG_ROOTDIR/uwb after installation. This file can be adjusted according to your requirements. For details on how to configure logback please refer to the official documentation for logback.

The logging for UWB aims to follow the ReFlex logging rules regarding the following log levels:

  • Info: An expected event occurs that is of interest to developers and administrators, e.g. life cycle information of different components.
  • Warning: An event occurred indicating irregular circumstances but the application is still able to process the request and can provide a meaningful response or providing a reasonable solution.
  • Error: The application cannot process a single request due to an unexpected event. The application cannot answer the request with a meaningful response, i.e. it responds with an error message.

Application server hardening

We strongly recommend to harden your application server. This includes but is not limited to

  • enabling TLS
  • enforce HTTPS
  • run the app server as non-privileged user
