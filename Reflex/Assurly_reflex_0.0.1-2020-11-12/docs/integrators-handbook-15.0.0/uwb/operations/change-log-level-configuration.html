Change log level

The log level can be changed during runtime via a REST API endpoint provided by the Underwriting Workbench.

All UWB API endpoints are secured by OAuth2 and require the client application to provide a valid JSON Web Token (JWT) to be granted access. This token must be issued by the same authorization server that is used for user authentication.

The following sections illustrates how to setup Keycloak to issue a valid JWT for accessing the REST API endpoint to change the log level. If you choose another identity server / solution for authorization, please consult the documentation of the selected solution to set it up accordingly.

Setup client in Keycloak for change log level endpoint

Create a new client and add the following client-role from the UWB-client:

  • change logging - OPS_LOGGERS

In Keycloak you have to open Service Account Roles on the new client and choose uwb under Client Roles and add OPS_LOGGERS there.

Create a new client with the following attributes.

Attribute Value
Client ID A unique client id (e.g. service-account-change-logging)
Client Protocol openid-connect

In Keycloak also turn off the following authorization flows: Standard Flow, Implicit Flow, Direct Access Grants, Authorization. Only Service Accounts must be enabled.

Credentials

Attribute Value
Client Authenticator Client id and Secret

Service account roles

Attribute Value
Client Roles Existing UWB client (e.g. uwb)
Assigned Roles OPS_LOGGERS

Sample request

When using Keycloak you can get an access token with the following request:

curl --data "grant_type=client_credentials" http://${server:port}/auth/realms/reflex/protocol/openid-connect/token -H "Authorization: Basic ${basicAuth Header}"

Send a request to the UWB logging endpoint http://${server:port}/iwb/api/actuator/loggers/${class name} with the following content and an authorization header:

curl --data '{"configuredLevel":"DEBUG"}' http://${server:port}/iwb/api/actuator/loggers/com.hannoverre.iwb -H "Authorization: Bearer ${paste access token here}" -H "Content-Type: application/json"

The last URL path segment is the logger to be changed, e.g. com.hannoverre.iwb. The value of the configuredLevel key in the request body (parameter --data) is the new logging level.

The changed log level is only applied until the server is restarted and is not saved permanently.
